XML.load

myXML.load(a)

 

Where myXML is an instance of the XML object, and a is a string specifying the location from which the XML document is being loaded from.

 

Compatible with Flash Player 5 and later. You can load the XML document from a location on your hard drive, or from a Web server, and the path can be absolute or relative. Refer to related entries XML.loaded, XML.onLoad, XML.sendAndLoad, XML.getBytesLoaded, XML.getBytesTotal, and XML.status.

Description

The XML.load() method is used to load an XML document from a specified location. The XML object instance has its content replaced with the newly loaded data. Whether or not the loading process is complete is monitored using the XML.loaded property.

Examples and practical uses

You will probably use this method frequently when you work with XML and Flash. After XML data has completed loading (or not loading) into the Flash movie, the XML.onLoad event handler is called. The successful or unsuccessful transmission can be ascertained using this handler. The contents of the XML document to load are not parsed until after the load is completed. You can use the onLoad handler so processing initiates after the XML has been loaded.

 

To load a document called xmldoc.xml into the existing myXML_xml instance, and error check based on the outcome, you would issue the following ActionScript:

 

myXML_xml = new XML();

myXML_xml.ignoreWhite = true;

myXML_xml.onLoad = function(success) {

   if (success) {

      trace("loaded! a smashing success");

      processXML();

   } else {

      trace("loading didn't go so well");

   }

};

myXML_xml.load("xmldoc.xml");

 

This code uses xml.load in conjunction with the XML.onLoad handler, and if the data has been loaded, the XML will be processed using the processXML function. For a working example, refer to the xml.load.fla and xml.load.swf, using the xmldoc.xml file, .

Tips and precautions

It is important to check that your movie has loaded and parsed before manipulating the new data within the XML object instance. You can do this either by using the XML.loaded or XML.onLoad callback event handler to make sure it is completely loaded and parsed. However, these methods will not check whether or not it has been successfully parsed (in that it is a well-formed document). In order to do some checking for this, you will need to use the XML.status property.

 

For security reasons, the location you are loading data from must be in the same subdomain as the wherever the current movie resides. That said, you can bypass these security measures using a server-side language (such as ColdFusion, PHP or ASP, to name a few). For more information on loading movies from a location outside the subdomain, refer to Chapter 19.

 

On a final note, the instance you are loading the XML data into must be of a top level XML class.
