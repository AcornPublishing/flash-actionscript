XML.nextSibling

myXMLNode.nextSibling

 

Where myXMLNode is a node within an instance of the XML object.

 

This property is compatible with Flash 5 Player and above.The XML.nextSibling property references the next child of the object’s child list of a given node, and is used to navigate XMLdocument structures. Refer to related entries on XML.childNodes, XML.nodeName, XML.nodeValue, XML.previousSibling, XML.firstChild, and XML.lastChild. These entries are all used to traverse XML structure hierarchies.

Description

The XML.nextSibling property examines the XML object node's parent’s child list. It returns the name of the next sibling in this list. If there is no next sibling, a null is returned. This is a read-only property.

Examples and practical uses

This property is used to access and return references to a particular node. If the next sibling does not exist, a null is returned so you must be careful to scope your hierarchies correctly.

 

If you constructed the XML represented in

 

myXML_xml = new XML("<cat><name>whiskers</name><age>5</age></cat>");

 

you would have a very simple hierarchy that looked like the following:

 

<cat>

   <name>whiskers</name>

   <age>5</age>

<cat>

 

As you can see, the <name> and <age> nodes are on the same level, and therefore are sibling nodes. You use nextSibling to traverse the structure, from one child node to the next sibling (which is on the same level).

 

myXML_xml = new XML("<cat><name>whiskers</name><age>5</age></cat>");

trace(myXML_xml.firstChild.firstChild.nextSibling);

 

Which as you saw above, traced the <age>5</age> node. This is because this node is the next sibling. If you only wanted to extract the nodeName, then you would use the following code:

 

trace(myXML_xml.firstChild.firstChild.nextSibling.nodeName);

 

If you needed to extract the value of 5 from this node, you would then use the following notation:

 

trace(myXML_xml.firstChild.firstChild.nextSibling.firstChild.nodeName);

 

Refer to the example xml.nextsibling.fla and xml.nextsibling.swf for a look at this property in use.

Tips and precautions

The XML.nextSibling property cannot be used to set the value of (or manipulate) sibling nodes in any way. It can only be used to access the names and values of the nodes. It can be used in conjunction with other methods to change these node properties. As usual, you might also find it useful to store the value of the XML.nextSibling in a variable by querying the property.
