onClipEvent

onClipEvent(event){

 code;

}

 

Where event is one of several events and code is the statement(s) to be executed when the event is detected.

 

This method is compatible with Flash 5 and later.

 

The onClipEvent() method runs an event-triggered set of statements. Flash MX and later allow you to assign functions as an event handler instead of using the onClipEvent() method. The newer coding approach is advisable for an event handler. With Flash MX and later, there is a move away from using the onClipEvent() method in favor of the MovieClip, Mouse, and Key methods of the same names.

Description

The onClipEvent() method is an event handler used to monitor a movie clip for a particular event. The onClipEvent() method is always attached to a movie clip instance. If an event is detected, the statements inside the code block are executed. The events that can be monitored and the triggering conditions are:

 

·       data – triggered when data is received in a loadVariable or loadMovie action

·       enterFrame – triggered when the playhead enters a new frame

·       keyDown – triggered when a keyboard key is depressed

·       keyUp – triggered when a keyboard key is released

·       load – triggered when a movie clip is instantiated and appears on the timeline

·       mouseDown – triggered when a mouse button is depressed

·       mouseMove – triggered when the mouse is moved

·       mouseUp – triggered when a mouse button is released

·       unload – triggered when a movie clip is removed from the timeline

 

Whenever one of these events is detected, if an onClipEvent() method has been defined for that event the code inside that code block is executed.

Examples and practical uses

The onClipEvent() method is used to watch a movie clip for events. The event to be watched for is specified in parentheses, followed by the code to be executed when the event occurs in a code block. For example, to watch for the event MouseUp (which is triggered whenever a mouse button has been pressed and released) you would write:

 

onClipEvent(mouseUp){

 trace("The button has been released");

}

 

You can add as many lines of code as you want in the body of the onClipEvent() event handler. You can define an event handler for each event in the list above. There are many other examples of using the onClipEvent() method throughout this book.

Tips and precautions

With Flash 5 and previous releases, movie clip events could only be accessed using the onClipEvent() method, With Flash MX and later, there are specific event handlers for these events, including onData, onDragOut, onDragOver, onEnterFrame, onKeyDown, onKeyUp, onLoad, onMouseDown, onMouseMove, onMouseUp, and onUnload. The details of these event handlers can be found in the relevant object entries (such as the Mouse object for onMouseDown, onMouseMove, and onMouseUp, or the Key object for onKeyDown and onKeyUp ).The MovieClip object contains entries for the rest of the event handlers. With Flash MX and later, there is a move away from using the onClipEvent() method in favor of the MovieClip, Mouse, and Key methods of the same names. In other words, instead of using:

 

onClipEvent(onEnterFrame){

 // do something

}

 

Flash MX and later should be using:

 

myMovieClip(onEnterFrame){

 // do something

}

 

The onClipEvent() method is used as an event handler with many other actions. Event handlers can be defined for as many (or as few) events as you want to watch for in a movie. The event handlers apply for the entire movie.

 

When using the onClipEvent(data) method the event is triggered when the SWF frame has completed loading, not when the data arrives. If there are several packets of data arriving in one frame, the event is triggered only once after the frame has loaded.
