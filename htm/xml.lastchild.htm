XML.lastChild

myXMLNode.lastChild

 

Where myXML is an instance of the XML object.

 

Compatible with Flash Player 5 and later. The XML.lastChild property references the last child of the object, and is one of the properties used to traverse the XML data hierarchy. Refer to related entries on XML.firstChild, XML.childNodes, XML.parentNode, XMLpreviousSibling, and XML.nextSibling.

Description

The XML.lastChild property examines the XML object’s parent’s child list and returns the name of the last child in the list. It is used in practically the same way as the related properties listed above, such as XML.firstChild and XML.nextSibling. If the specified node does not have children, null is returned. This is a read-only property.

myXML_xml = new XML("<cat><name>whiskers</name><age>5</age></cat>");

trace(myXML_xml.lastChild);

 

 

<cat><name>whiskers</name><age>5</age></cat>

 

The entire XML structure is traced, because the <cat> node is both the first and last node.

 

 

myXML_xml = new XML("<cat/>");

trace(myXML_xml.firstChild.lastChild);

 

 

null

 

There is no node to return.

 

Examples and practical uses

You might want to use XML.lastChild to navigate to the end of the end of a list of child nodes. For example, you can select a particular node and store it in a variable (mynode), and then return the lastChild of the specified node to the Output window:

 

myXML_xml = new XML("<cat><name>whiskers</name><age>5</age></cat>");

thenode = myXML_xml.firstChild.firstChild;

trace(thenode.lastChild);

 

Would return whiskers to the Output window.

 

This is another tool in your belt for navigating XML structures. It might be a quicker, easier way to reach and return a reference to the node you require for your movie.

 

Refer to the example xml.lastchild.fla and xml.lastchild.swf for an example of this property in use.

Tips and precautions

The XML.lastChild property cannot be set, only read. Therefore, just like any other of the related properties, it cannot be used to manipulate children (such as setting values for the child node in question). It can only access them.
