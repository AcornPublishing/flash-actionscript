LoadVars.loaded

instancename.loaded

 

where instancename is an instance of the LoadVars object

 

Compatible with Flash Player 6 and later.

 

The LoadVars.loaded property indicates if a load operation is underway or completed.

Description

The LoadVars.loaded property normally has a value of undefined. When a.load() or a.sendAndLoad() operation has begun, the property has the value set to false. After the operation has completed, the property is set to true.

 

 

Code

 

 

Additional explanation

 

Notes

 

Xferflag=myMovieVars.loaded;

 

 

 

If a load or sendAndLoad has begun, Xferflag is false. If the load or sendAndLoad is complete, Xferflag is true. If no transfer is underway, Xferflag is undefined.

 

 

If a transfer has begun but fails without success, the property is still set to false

 

 

Examples and practical uses

The LoadVars.loaded() property can be used to track when a data transfer has been started, (set to false) when it is underway (set to false), and when it has completed (set to true). If a transfer is not completed successfully for any reason, but it had started, the property is still set to false. If no transfer is underway, the property has an undefined value. Usually the result (which will be true, false, or undefined) is assigned to a variable for later comparison.

 

To use this method it has to be associated with an instance of the LoadVars object, and then can be called for that instance like this:

 

myMovieVars = new LoadVars();

// start a data transfer

myMovieVars.load("myData.dat");

// check whether the transfer has started or is completed

Xferflag=myMovieVars.loaded;

 

This method is similar to the XML.loaded() method used by the XML object.

Tips and precautions

To determine if a load operation has begin or not, you can use this method with a simple conditional clause:

 

myMovieVars = new LoadVars();

myMovieVars.load("data.dat");

// the load could be a movieclip using loadMovie

// check to see if the load has started; false if started

if (!myMovieVars.loaded){

   trace("The transfer has started...");

   // code to show progress screen

}

else {

   // check to see how many bytes were transferred

   transferred=myMovieVars.getBytesLoaded();

   // do something with the results

}

 

There is a slight issue with using the !myMovieVars.loaded construct as it is shown in this code. If myMovieVars.loaded is false, then the first block of code is executed properly. However if the value of myMovieVars.loaded is undefined, as can happen when no transfer is underway, then it is evaluated to true. This would cause the else block to be executed, and may cause errors. To prevent this problem, a nested if... could be used to check specifically for a true value or to rule out an undefined value prior to calling other statements.

 

The loaded() property can only be referenced from an instance of the LoadVars object.
