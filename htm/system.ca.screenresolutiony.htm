System.capabilities.screenResolutionY

a = System.capabilities.screenResolutionY;

 

a will be returned as the resolution of the users screen in the y direction in absolute pixels.

 

This property is compatible with Flash 6 or later. This property is read only.

Description

The resolution of a users screen is a measure of how many pixels can be displayed. The screen resolution in the y (height) direction is held in the property screenResolutionY.

 

You should note that this is not an indication of the number of pixels available to the Flash player, because the browser window will usually occupy only a proportion of the screen. This property is therefore best used in conjunction with a number of other properties and objects to build up a true picture of how the user is viewing your site. See the Examples and practical uses section for more details.

Examples and practical uses

When looking at the screenResolutionY, the following properties and methods will give you a fuller picture of how the user is viewing your site:

 

·       System.properties.screenResolutionX, System.properties.screenResolutionY :
These properties give you the screen resolution of the full screen. The browser window holding your content will generally be smaller than this area. You can force the browser to occupy the full screen with fscommand, but the use of this feature is somewhat frowned upon as Very Bad Design.

·       System.properties.screenDPI :
This property is useful in differentiating between a standard monitor and a custom screen. If this property returns 72, then the screen is a standard personal computer (see also the entry for System.properties.screenDPI, which gives some warnings about what a "standard monitor" actually is in practice; "standard" can be a misleading term!).

If this property is any other value, then the screen may be a custom screen. Such screens are usually smaller than a standard monitor, and will have custom pixel densities depending on the technology they use. You should be aware of the screenDPI property in such cases, because it can affect readability of text displayed on the screen.

·       Stage.width, Stage.height, Stage.scaleMode :
The Stage.width and Stage.height properties refer to the Flash stage size. They have two modes of operation, and this is defined by the read/write property Stage.scaleMode. If you set Stage.scaleMode to the string value "exactFit ", then the pixel size will be in "scaled pixels" (see also the entry for Button._quality, which describes these as "virtual pixels", and explains their graphic relationship to true pixels). Scaled pixels are not related to true pixels, and cannot be compared with System.properties.screenResolutionX or System.properties.screenResolutionY, which are true pixels.

If you set Stage.scaleMode to the string value "noscale ", then the values returned in Stage.width and Stage.height will be in true pixels, but you also lose the ability to scale your SWF to fit the browser window. If you want to see the browser window dimensions in exact pixels, but also want to scale your content to fit in the browser window, a good trick is to only set scaleMode to "noscale " whilst you are reading Stage.width and Stage.height, and set it back to "exactFit " the rest of the time. If you set-reset it within the same frame, there will be no change. Run the following script with Test Movie, and you will see that opening/closing/resizing the bandwidth profiler will change the values returned:

_root.onEnterFrame = function() {
   Stage.scaleMode = "noScale";
   trace(Stage.width);
   trace(Stage.height);
   Stage.scaleMode = "exactFit";
};

In contrast, the following script will always return the same values, because it is scaling the SWF to fit the available space, and this results in the same number of pixels at all times (they are simply scaled). On a default flash SWF (550x400), this script will always return 550 and 400 as the width and height, irrespective of the actual browser size in true screen pixels.

Stage.scaleMode = "exactFit";
_root.onEnterFrame = function() {
   trace(Stage.width);
   trace(Stage.height);
};

See the example system.capabilities.fla and system.capabilities.swf .

Tips and precautions

As seen above, this property is best used in conjunction with a number of other features, and is much less useful when used on its own.

 

Unlike most other properties of the System.capabilities object, this property does refer to the specific user for the standard PC/Mac Flash player. It will reflect the current configuration of the individual user.
