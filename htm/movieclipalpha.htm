MovieClip._alpha

instanceName._alpha = a;

b = my_mc._alpha;

 

Where instanceNameis the instance name of a movie clip, and ais the value you want to assign to the clip alpha value. bis a variable that is equated to the current value of the movie clipalpha value. The alpha value range is from 0 to 100.

 

This property is compatible with Flash 5 or later.

Description

The _alpha property of a movie clip defines the clip’s alpha setting, and this can range between 0 (fully transparent) and 100% (fully solid) per pixel. The _alpha property is read/write.

 

 

Code

 

 

Movie clip

 

Notes

 

my_mc._alpha = 0;

 

 

 

Movie clip is fully transparent.

 

my_mc._alpha = 25;

 

 

 

Movie clip is approximately 25% opaque.

 

my_mc._alpha = 50;

 

 

 

Movie clip is approximately 50% opaque.

 

 

my_mc._alpha = 100;

 

 

 

Movie clip is solid and fully opaque.

 

 

my_mc._alpha = 1000;

 

 

A little known trick; super high values of alpha setting provide a high contrast effect if your graphics are composed of gradient and radial fills that have varying alpha. This occurs because the alpha value is per pixel. See also Tips and precautions section.

 

 

my_mc._alpha = -50;

 

 

Negative values of _alpha are rendered as if they were 0%.

 

Examples and practical uses

The alpha setting can be used to produce fade in/out effects.

Tips and precautions

Although _alpha is varied between 0 and100, it internally varies between 0 and 255 (because it operates internally using 8 bit alpha resolution). This means that any alpha value you set will be approximate; the actual alpha setting displayed will be to the nearest 0 to 255 alpha level. For example:

 

my_mc.onEnterFrame = function() {

    this._alpha--;

    trace(this._alpha);

    if (this._alpha<0) {

        this.onEnterFrame = 100

    }

};

 

The first few iterations will give you:

 

Actual                     Expected

98.828125        99.000

97.65625         98.000

96.484375        97.000

95.3125          96.000

94.140625        95.000

92.96875         94.000

91.796875        93.000

 

This occurs because alpha cannot exist in 'in-between' values such as 99.000. This means that you cannot assume that it will take 100 decrement subtractions to get from 100% to 0% for alpha if you decrement alpha directly.

 

Also, alpha is calculated per pixel rather than per object. This means that if you set the alpha of a clip to 100%, there may still be pixels that are not totally opaque if your graphics are made up of fills that do not have a 100% full value alpha. You can see this via the following short example:

 

 

Create a circle which has a fill color that has an alpha of 50%. Select the circle and hit F8, making it a movie clip called clip. Give it an instance name of my_mc.

 

Add a new layer and on frame 1 of it add the following:

 

my_mc._alpha = 100;

 

Notice that this does not give you a movie clip with 100% alpha. Because the full scale alpha value of your graphic is 50%, you have to take the alpha setting of my_mcto 200% before all the pixels in it are at 100%.

 

Having large areas of pixels in a presentation can cause major slowdowns, because displaying transparency is computationally expensive (areas of >100% alpha – those which are transparent) have to be drawn at least twice, whereas areas of 100% alpha are drawn only once). To reduce the effects of this you should consider:

 

·       Keeping areas with alpha settings less than 100% small to reduce the need for Flash to have to draw large areas of pixels more than once.

·       If you want to be sure that Flash will set _alpha to a specified value, make sure that value is exactly divisible by 100/256, so that it corresponds exactly with the internal 0-255 transparency levels. Notice that all the Actual value column figures in the table above are exact multiples of 100/256.

·       If you want to set a button’s alpha to 100% (thus making it totally opaque again and Flash has to draw it only once), you should explicitly set it to 100%. For example, the following will not return your alpha to 100% (it will give you a final alpha of 99.609375, so Flash will still perform multiple redraws, even though the movie clip may appear to be at 100% alpha by visual inspection ):

 

my_mc._alpha = 74.2;

my_mc._alpha += 25.8;

trace(my_mc._alpha);

 

…whereas this will, because it is explicit:

 

my_mc._alpha = 100;

 

 

·       If you actually want to disable a button and make it invisible, use the following code;

 

my_mc._visible = false;

 

Setting invisible via alpha is not recommended, given that Flash will draw the movie clip at 0% alpha, as opposed to setting visible, where the clip is not drawn at all. See also MovieClip._visible.
