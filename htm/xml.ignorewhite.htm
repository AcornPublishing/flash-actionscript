XML.ignoreWhite

myXML.ignoreWhite = a;

 

Where myXML is an instance of the XML object, and a is either true or false.

 

Compatible with Flash 5 Player (build 41) and above. Thisproperty is used to store a Boolean value. You set the property to either true or false: true will remove white spaces from your XML document, and false will leave them in. It is important to note that you will have to manually strip whitespace (or use established AS #include files from the Flash community!) from earlier versions of the Flash 5 player. Refer to related content in Chapters 18 and 19 about whitespace.

Description

The XML.ignoreWhite property is a Boolean indicating whether nodes that only contain white space are discarded during parsing. The default setting is false, which will leave the white-space text nodes in the document. When set to true only text nodes that contain only white space are discarded.

Examples and practical uses

The XML.ignoreWhite property. should be added to your ActionScript so you are able to process loaded XML data. If you do not strip white spaces from XML documents using this property, all spaces will be treated as empty nodes, including the firstChild! After you strip white spaces, you can process your structures as usual. For example, if you load (using XML.load or XML.sendAndLoad methods) the following file into a Flash movie:

 

<mylist>

   <vegetables>

      <carrots>3</carrots>

      <corn>6</corn>

   </vegetables>

   <meat>

      <beef>1lb</beef>

      <chicken>2lb</chicken>

   </meat>

</mylist>

 

It will appear as the following if you set XML.ignoreWhite to true.

 

<mylist><vegetables><carrots>3</carrots><corn>6</corn></vegetables><meat><beef>1lb</beef><chicken>2lb</chicken></meat></mylist>

 

If you do not set ignoreWhite to true, and then display the structure in an Output window or text field, it will appear like the original document. Refer to the example xml.ignorewhite.fla and xml.ignorewhite.swf and the corresponding XML file xmldoc.xml for an elaboration on this code.

Tips and precautions

The default value for XML.ignoreWhite is false. If you do not set this property in your Flash movie, whitespace will be left in your XML document. Even though white spaces help you read and understand a document, you will probably want to remove white space if you intend to process XML data within your movie from any data source containing white spaces (like the <mylist> tree above). If you want your Flash movies using XML to work with players earlier than release 41, you will need to manually strip white spaces. You can do this manually by removing all returns and spaces from the XML. As this would probably be very tedious, you might want to use an established routine from www.moock.org/asdg/codedepot or http://chattyfig.figleaf.com(XMLNitro).

 

This property can only be used on top level XML class nodes, and will globally affect your entire XML object instance.
