XML.parentNode

myXMLNode.parentNode

 

Where myXMLNode is an instance of the XML object.

 

Compatible with the Flash 5 Player and above. The XML.parentNode property references the parent nodeof a given node in an XML object instance. This property is used in the same way as the related entires: XML.childNodes, XML.nextSibling, XML.previousSibling, XML.lastChildand XML.firstChild.

Description

The XML.nodeValue property is used to reference the parent node of an XML object. If there is no parent node, null is returned.

myXML_xml = new XML("<cat><catname>whiskers</catname><age>5</age></cat>");

whiskers = myXML_xml.firstChild.childNodes[0].childNodes[0];

trace(whiskers.parentNode.nodeName);

 

 

catname

 

Returns catname because it is the node name of the parent node of the whiskers text node.

 

 

myXML_xml = new XML("<cat><catname>whiskers</catname><age>5</age></cat>");

trace(myXML_xml.parentNode);

 

 

null

 

A root node cannot have a parent node.

 

Examples and practical uses

The XML.nodeValue property is references an XML object’s parent node. If there is no parent node, a null is returned instead. The XML.nodeValue property is a read-only value and cannot be set. In the following code, we load a document into the Flash movie, and then when it is loaded, call a function based on the onLoad handler.

 

myXML_xml = new XML();

myXML_xml.ignoreWhite = true;

myXML_xml.load("xmldoc.xml");

myXML_xml.onLoad = function() {

   var newXML = new XML();

   newXML = myXML_xml.firstChild.childNodes[0].firstChild;

   trace(newXML);

   trace(newXML.parentNode);

   trace(newXML.parentNode.nodeName);

};

 

To see the code working, look at the example xml.parentnode.fla and xml.parentnode.swf, which use xmldoc.xml.

Tips and precautions

XML.parentNode is a read-only property, so it cannot be used to manipulate values. It is important to remember scope so you can traverse XML hierarchies without difficulty, and node types when extracting node data.
