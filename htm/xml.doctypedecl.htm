XML.docTypeDecl

myXML.docTypeDecl

 

Where myXML is an instance of the XML object.

 

Compatible with Flash 5 Player and above. The XML.docTypeDecl property can be used to set and retrieve the DOCTYPE declaration, although it is not used for XML validation of any sort. Also see the related entry on XML.xmlDecl.

Description

The XML.docTypeDecl property is used to set and retrieve information about an XML document’s DOCTYPE declaration. If no declaration was found, the XML.docTypeDecl property has a value of undefined and no output is returned to the Flash movie.

Examples and practical uses

If any DOCTYPE tag of the XML object instance exists, the docTypeDecl property will return a string. If there is no such declaration, the value of the XML.docTypeDecl property is undefined. This property can take the tag, and then use it to help externally validate the XML documents. It can also indicate to us the kind of XML document being loaded into the movie, because DTDs are sometimes standardized, and one DTD is often used widely across certain groups for uniform validation purposes.

 

As you have seen in the above code, you can trace an exisiting DOCTYPE using the docTyleDecl property. However, you can also set a new value for the XML.docTypeDecl property, by using a standard assignment:

 

<myXML_xml = new XML();

myXML_xml.docTypeDecl = "<!DOCTYPE greeting SYSTEM '"Welcome.dtd'">";

 

For an example of this property in use, see the example xml.doctypedecl.fla and xml.doctypedecl.swf .

Tips and precautions

The XML.docTypeDecl property does not validate any values in the DOCTYPE declaration. Flash does not perform any DTD validation, but instead simply parses the XML document.

 

Also, the XML object instance must be a top-level node of the XML object.
