MovieClip._focusrect

myMovieClip._focusrect;

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip._focusrect property indicates whether a movie clip that has keyboard focus has a yellow rectangle around it.

Description

The MovieClip._focusrect property returns a Boolean that indicates whether a movie has a yellow rectangle around it when keyboard focus is active. If set to true, the yellow rectangle appears; if set to false, no yellow rectangle is shown. The movie clip below left is shown with input focus and the rectangle enabled.

Notes

 

 

myMovieClip._focusrect= false;

 

 

Prevents the movie clip from having a yellow rectangle around it when it has keyboard focus.

 

Examples and practical uses

The MovieClip._focusrect property is a Boolean value that indicates whether a movie clip that has keyboard focus has a yellow rectangle around it or not. By default, a movie clip has a yellow rectangle around it when it has input focus.

Tips and precautions

The MovieClip._focusrect property tells you whether a movie clip has a yellow rectangle around it when keyboard focus is active. The yellow rectangle is a convention many users are familiar with, and can help act as an aid for users. However, you may want to disable the yellow rectangle for cosmetic reasons on certain movie clips:

 

·       Invisible buttons. If you are using an invisible button movie clip, you don’t want the user to see it is there!

·       A movie clip that is below (or partly obscured by) other movie clips. This may lead to confusion form the user.

·       A movie clip that is animating. Users would normally only expect buttons or button-like movie clips to have the rectangle around them.

 

In general though, if you don’t want a movie clip to show that it has input focus, you don’t want it to have input focus either, so this method is little used.

 

Input focus is very difficult to maintain for anything other than a text field in any case, and you are advised not to rely on it for keyboard inputs. Getting the user to specifically use the Tab key to give a movie clip is hard enough, but they only have to click on something else for the focus to be lost. Focus has also been seen to be lost simply by the user moving the mouse!
