Number.MIN_VALUE

Number.MIN_VALUE

 

Where Number is the Number class.

 

This property is compatible with Flash 5 and later.

 

The Number.MIN_VALUE property is the smallest number that can be represented.

Description

The Number.MIN_VALUE property is a value representing the smallest number that can be stored in a Flash number data type. This is a double precision number adhering to the IEEE-754 standards and is equal to approximately 5.0 * 10^-324 or 5.0E-324. The Number.MIN_VALUE property is a real number and can have values added and multiplied by it, but cannot be subtracted from or divided by, as those results would exceed the double-precision representation.

Notes

 

myNum = Number.MIN_VALUE;

 

 

Assigns the smallest possible number to myNum (based on the data type used for double precision numbers).

 

 

Flash uses the IEEE-754 standard to determine the minimum value.

 

 

myNum = Number.MIN_VALUE-10;

 

 

Since Number.MAX_VALUE is the smallest number possible, you can’t subtract from it without exceeding the capabilities of the double precision number type.

 

 

 

 

Examples and practical uses

The Number.MIN_VALUE property represents a valid numeric value that cannot have any value smaller, but it can still be manipulated by Flash. You can assign the Number.MIN_VALUE property to a variable and then manipulate the value like this:

 

myNum = Number.MIN_VALUE;

myNum *= 5;

 

The Number.MIN_VALUE property can be used to check that a value is representable by Flash, using a simple conditional:

 

if (myVar > Number.MIN_VALUE){

 // code to continue

}

Tips and precautions

The Number.MIN_VALUE property is the minimum allowable value that can be handled by Flash’s number data type. See also Number.MAX_VALUE.
