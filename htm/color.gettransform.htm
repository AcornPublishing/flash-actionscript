Color.getTransform

a = myColor.getTransform();

 

Where a is the variable that you want to equate to the color transform object that was used as the argument of the last color transformation using the color object myColor.

 

This method is compatible with Flash 5 or later. In Flash 5, the Color object can be applied to movie clips only. In Flash 6, it can be applied to movie clips, buttons, or dynamic/input text fields. See the entry for Color to see how you can work around this via a wrapper.

 

NB – This method is very rarely used.

Description

This method will hold the value of the color object's current color transform object. This value will be equal to the object that created the last color transformation, assuming that at least one color transformation has taken place. If the last color transformation was made manually (i.e. via the Color dropdown menu in the Property inspector, the corresponding color transform object for that transform will be returned.

 

See also Color.setTransform, which describes the color transform object.

 

 

Code

 

 

Notes

 

my_color = new Color(my_mc);

myColorTrans = {ra:70, rb:77, ga:70, gb:77, ba:70, bb:0, aa:100, ab:0};

my_color.setTransform(myColorTrans);

//

//  read last transform object...

a = my_color.getTransform();

for (prop in a) {

   message = prop+"= "+a[prop];

   trace(message);

}

 

 

This will trace the values of myColorTrans,the color transform object that is used earlier in the script to transform the target my_mc.

 

 

my_color = new Color(my_mc);

my_color.setRGB(0xFFFFFF);

//

//  read last transform object...

a = my_color.getTransform();

for (prop in a) {

   message = prop+"= "+a[prop];

   trace(message);

}

 

 

Because the Color.setRGB is really just a simplified form of Color.setTransform, the script will still return a color transform object. In this case it will have properties as shown:

 

ab= 0

aa= 100

bb= 0xFF

ba= 0

gb= 0xFF

ga= 0

rb= 0xFF

ra= 0

 

 

my_color = new Color(my_mc);

//

//  read last transform object...

a = my_color.getTransform();

for (prop in a) {

   message = prop+"= "+a[prop];

   trace(message);

}

 

 

This code makes no color transformations prior to requesting the object used to form the last transformation. This will not raise an erroneous response, given that the object returned is one that would make no change to a target's default color, and this object will have properties as shown:

 

ab= 0

aa= 100

bb= 0

ba= 100

gb= 0

ga= 100

rb= 0

ra= 100

 

Examples and practical uses

Given that you will know what the properties of the color transform object that formed the last transform actually are because your code will have created it (you have to define a separate object that defines the color transformation information when using Color.setTransform ; see the entry for this action for further details), there is little point in asking Flash what you already know!

 

There is only one occasion when you might need this method, and this use should be reserved for the really lazy coder:

 

·         If you have manually created the color change you want to apply later in the runtime SWF, you can leave the manually transformed clip/button/text field on the stage and apply a getTransform on its color object to retrieve the color transform object required to create your manual color change without having to work it out for yourself. You can then use this object in subsequent transformations.

 

See also the example FLA, color.gettransform.fla and color.gettransform.swf .

Tips and precautions

Ignoring the very lazy, there is no need to use this method, given that your code will already contain a copy of the color transform object. If you need to recall the state of this at any time, you are advised to save it in another object:

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:200, ga:100, gb:50, ba:100, bb:50, aa:100, ab:0};

my_color.setTransform(myColorTrans);

//

// save the last color transform object…

myLastTrans = myColorTrans;

 

If you need to convince yourself that myLastTrans is equal to myColorTrans,this code added onto the end of this script should do the trick:

 

for (prop in myLastTrans) {

   message = prop+"= "+myLastTrans[prop];

   trace(message);

}
