getVersion

getVersion()

 

Compatible with Flash 5 and later. This function is used to check for the version of the Flash Player 5 or later installed on a users system. It cannot be used to check for the presence of a Flash player. Related entries include System.capabilities.version.

Description

This function returns a string indicating the installed Flash Player version, and platform of the host computer.

Examples and practical uses

This function is used to check for the version of a Flash Player installed on an end users computer. It cannot check to see if one is already installed. This function is typically used to sniff the computer and redirect based on the returned string.

Programming

In the example getversion.fla and getversion.swf , we can grab the server string using getVersion. Then we will split the string using the split method. This will create substrings and put them into an array. Therefore, we can sniff the major version number of the Flash Player, so a detection script can be created to redirect end users. The following code is used to detect and split the server string:

 

myVar = getVersion().split(" ");

OS = myVar[0];

versions = myVar[1].split(",");

major = versions[0]

minor = versions[2];

 

Using this, major will return the version number of the Flash Player on the end users system if they have Flash Player 5 or Flash Player 6 installed, e.g. if the user has Flash Player 6 installed, the major variable will be 6.

Tips and precautions

This function can only be used to check for the Flash Player 5 or later versions. To check for versions of the Flash Player 6 or later, you can also use System.Capabilities.version property. When using sniffers or detection scripts, it is important to remember that they are not always reliable.
