MovieClip.onEnterFrame

myMovieClip.onEnterFrame

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip.onEnterFrame event handler is invoked continually as each frame in a movie is entered.

Description

The MovieClip.onEnterFrameevent handler is invoked as the playback head enters each frame in a movie. Any actions associated with the event handler are invoked as each frame is entered.

Examples and practical uses

The MovieClip.onEnterFrameevent handler is invoked when the playback head enters a movie frame. The actions in the event handler are invoked before any other actions in the frame are processed. Whatever function has been defined as part of the event handler definition is launched on receipt of data. Obviously, the MovieClip.onEnterFrameevent handler is executed at the frame rate of the movie clip.

 

To define the MovieClip.onEnterFrame event handler, the function is written like this:

 

MyMovieClip.onEnterFrame = function {

 // code to run every frame

}

 

The MovieClip.onEnterFrame event is one of the two most used events in flash (the other being Button.onRelease). The onEnterFrame script is important because it allows you to run a script every frame to create animation. Many of the examples for other entries use thes event.

 

There are two variations of the event handler definition:

 

1.       Anonymous callback

 

myClip.onEnterFrame = function(){

    // do stuff

}

 

The function is defined at the same time it is attached to the event, so you don’t have to give it a name. Flash will assign the function a name internally, but you do not know what this name is, hence anonymous. The advantage of doing this is that it is quick and compact.

 

2.       Named function as callback

 

function doStuff() {

    // do stuff

}

myClip.onEnterFrame = doStuff;

 

Here, we give the callback a name. This means that we can refer to it again:

 

myOtherClip.onEnterFrame = doStuff;

 

In general:

 

·       Use anonymous callbacks when your code is short or when you know that you will only use the function once.

·       Use named functions when you want to attach them to more than one movie clip.

Tips and precautions

The movie clip myClipmust exist on the timeline when you define the event - event handler linkage:

 

myClip.onEnterFrame = function(){

    //do stuff

}

 

… will fail if myClipis not yet seen on the timeline.
