TextField.getFontList

myTextField .getFontList()

 

Where myTextField is a TextField instance.

 

Compatible with Flash Player 6 and later .

 

The TextField.getFontList() method returns an array of all fonts on the Flash Player host system

Description

The TextField.getFontList() method examines all the fonts on the host machine of a Flash player and returns the list of fonts as an array. All fonts included in SWF files are included in the list. The array is returned using strings.

Examples and practical uses

The TextField.getFontList() method is used to allow you to detect and choose fonts from a Flash player. When called, the TextField.getFontList() method returns an array of strings that contains all the fonts on the Flash player's system, as well as any loaded SWF files. This array is stored in an array variable:

 

myFonts = TextField.getFontList();

 

See also the example textfield.getfontlist.fla and textfield.getfontlist.swf . This file will list all fonts on your system.

Tips and precautions

The TextField.getFontList() method is a way of checking existing fonts on a player to ensure the correct fonts will be displayed in a movie. Since the fonts loaded on a machine may vary widely, this allows you to check for specific fonts you wish to use in a movie.

 

You can use the information retrieved by the getFontList to select a particular font to use in all or some of your text fields. This is discussed in the entry for TextField .

 

Using the fonts on the users machine saves you having to download them as part of your SWF. A nice bit of Flash candy is to let the user choose the font that they want to view a site, as selected from a combo box or list box component. Useless, but apt to impress, well until everyone works out how it is done and it becomes yet another overused Flash trick!
