Math.floor

a = Math.floor(b);

 

Where a is the variable that you want to equate to the floor of b.

 

This method is compatible with Flash 5 or later.

Description

This method will return the value of b rounded down to the nearest whole number that is less than, or equal to b.

 

 

Code

 

 

Notes

 

a = Math.floor(0);

 

b = Math.floor(1.1);

 

c = Math.floor(-1.1);

 

d = Math.floor(8.999999);

 

 

 

In the first line, a will be set to 0.

 

In the second line, b will be set to 1, because the next whole integer less than 1.1 is 1.

 

In the third example, c will be set to ­2, because the next whole integer less than ­1.1 is ­2.

 

In the fourth example, d is set to 8, because even though the fractional portion is very close to 9, Math.floor will still round down to the next lesser whole number.

 

 

e = Math.floor ("text");

 

If anything other than a number is passed into the function, then Flash will return NaN; which stands for "Not a Number".

 

Examples and practical uses

To see an example of the Math.floor method used to make objects snap into place, look at the example math.floor.fla and math.floor.swf . We can use the Math.floor method to ensure that numbers fall within a certain range.

 

For example, imagine we have a beautifully tiled ballroom, and each tile is 1 metre by 1 metre in size. Let’s say that to make the ballroom live up to its name, I throw a ball onto the floor, and it rolls to a stop on the tile that is 5.57 metres from the north wall, and 8.72 metres from the west wall. Somewhat like this bird’s eye view:

 

 

Now, let’s say that I want to use those two measurements to determine quickly which tile the ball is on. Let’s assume that the position of the ball is stored in ballxpos and ballypos, where ballxpos is 8.72, and ballypos is 5.57. Now, let’s use Math.floor:

 

tilex = Math.floor(ballxpos);

tiley = Math.floor(ballypos);

 

Then we’re going to be left with tilex being 8, and tiley being 5, because what Math.floor did was chop off the fractional portion, and leave us with those two floored numbers. We have determined that the ball is resting on tile (8,5) (assuming that the upper-left tile on this floor is (0,0). That seems pretty straightforward. What if each tile was something crazy like 8.7 metres square, rather than a nice tidy 1. How would we determine which tile the ball was on then? Like so:

 

tilex = Math.floor(ballxpos / tileWidth);

tiley = Math.floor(ballypos / tileHeight);

 

Or, more simply:

 

tilex = Math.floor(ballxpos / 8.7);

tiley = Math.floor(ballypos / 8.7);

 

To see Math.floor plotted on a graph see the example math.floorplot.fla and math.floorplot.swf .

Tips and precautions

Remember, this function does not round to the nearest whole number, it rounds to the next greatest whole number. The Math.round function will round to the nearest whole number.

 

In Flash 4, the methods of the Math object are emulated, and may therefore not always be as accurate as in Flash 5 and above.
