XMLSocket.connect

myXMLSocket.connect("a", b)

 

Where myXMLSocket is an XMLSocket instance, a is the IP address or domain name of the server you are connecting to, and b is the TCP port number to be listened to. You can also use localhost in place of a.

 

Compatible with Flash Player 5 and later. The XMLSocket.connect() method is used to create a socket connection. Refer to related entries on XMLSocket.onConnect, andXMLSocket.close.

Description

The XMLSocket.connect() method creates a socket connection to a specified server. A return of true indicates the socket was properly created, while a false indicates the connection attempt failed. The server can be identified by an IP address, a domain name, or the word null in order to indicate the same server that the movie resides on. The TCP port number must also be specified, and must be 1024 or higher for security reasons.

Examples and practical uses

The XMLSocket.connect() method is used to create a socket connection to a server. For example, the command:

 

mysocket = new XMLSocket();

mysocket.connect(null, 12345);

 

will try to establish a connection with the same server the SWF movie was loaded from, using TCP port 12345. If the connection is created successfully, the method will return true while a failed connection attempt will return a false. After connect returns true, the onConnect handler is invoked. If a connection is not made, the onConnect handler is not invoked. You can specify an IP address or a domain name as part of the XMLSocket.connect() method

 

You can embed a conditional check of the connection attempt in your movie by using the return from myXMLSocket.connect():

 

myXMLSocket = new XMLSocket();

if (myXMLSocket.connect("www.mydomain.com", 12345)) {

   trace("The connection attempt was successful.");

} else {

   trace("The connection attempt failed.");

}

 

Refer to the example xmlsocket.connect.fla and xmlsocket.connect.swf rom for more information.

Tips and precautions

The XMLSocket.connect() method will fail if the server to connect to is not in the same subdomain as the movie currently loaded. This only applies to applications being played within a browser. Also, connections can sometimes take a lengthy time to establish, so it may be wise to indicate a connection is processed while using connect().
