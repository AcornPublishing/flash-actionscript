Sound.attachSound

mySound.attachSound(“myButtonSound”)

 

Where the sound from the Library with a linkage identifier of myButtonSound will be attached to the mySound instance of the Sound object.

 

This method is compatible with Flash 5 and above. There are no known issues with any version of Flash that supports this object.

 

The attachSound method is used to attach a sound from the Library at runtime. If you want to use this method you need to import a sound into your Library while in authoring mode. Any sounds that you wish to use via the attachSound method need to have unique linkage identifier names. See also Sound.start, Sound.loadSound.

Description

This method attaches a sound from the library to the specified Sound object using a linkage identifier.

Using a variable for the identifier parameter. The value of userDefinedSound should be a string that matches the linkage identifier from one of the sounds in the Library.

 

Examples and practical uses

The attachSound method is used to dynamically attach a sound to an instance of the Sound object. Using this method you can replace sounds simply be calling the method again on the same instance of the Sound object and supplying a different linkage identifier. The example soundattachsound.fla / soundattachsound.swf shows you how to dynamically attach and replace sounds using the attachSound method.

Tips and precautions

You can only attach one sound at a time using the attachSound method. For example, in the following code, the sound with the linkage identifier “sound2” will play, as it was the last sound to be attached with the previous sound being removed from the Sound object.

 

sound1 = new Sound();

sound1.attachSound("sound1");

sound1.attachSound("sound2");

sound1.start();
