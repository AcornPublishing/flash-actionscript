XMLSocket.onXML

myXMLSocket.onXML(a)

 

Where myXMLSocket is an XMLSocket instance and a is an XML object instance that will contain  the new parsed XML data.

 

Compatible with Flash 5 Player and above. The XMLSocket.onXML() method is invoked when XML data is received. The new XML data will be contained within the a argument. Refer to the related entry: XMLSocket.onData().

Description

The XMLSocket.onXML() event handler is used with a "callback" function. The handler is invoked by the Flash Player when an XML document has been received over an open socket connection. All XML messages are terminated with a zero byte, and when one is received, the handler is invoked.

thesocket = new XMLSocket();

thesocket.connect("localhost", 12345);

thesocket.send("<myname>jepo</myname>");

thesocket.onXML = function(xmldata) {

   trace("xml data arrived");

   trace (xmldata);

};

 

 

 

When XML arrives to the movie, the content is traced, as well as an alert that it has arrived.

 

This XML is not parsed by Flash. To parse this information, you would enter functions within the inline function.

 

Examples and practical uses

The XMLSocket.onXML() event handler is used in conjunction with a custom "callback" function, so whenever a server downloads data terminated with a zero byte, a function in your movie is called. The XMLSocket.onXML() handler reads the stream from the server and whenever a zero byte is received it parses the data and invokes the handler. The contents between every zero byte is treated as an XML message, and is passed by the XMLSocket.onData() handler to the XMLSocket.onXML()handler for processing. Refer to the example xmlsocket.onxml.fla and xmlsocket.onxml.swf for more information on this handler.

Tips and precautions

You should remember that you can access new XML data which has arrived at your movie through the XML object. This XML object instance is the a argument in myXMLSocket.onXML(a).
