FStyleFormat.shadow

a.shadow = b;

 

Where a is the name of your style format, and b is a RGB value being assigned to the shadow region of component instances.

 

Compatible with Flash MX. This color value is given to the shadow regions, which help make up the edges of various components. The RGB color value assigned to this property is written using the 0xRRGGBB format. Related entries include those for FStyleFormat.highlight,  FStyleFormat.highlight3D, and FStyleFormat.darkshadow.

Description

This property changes the RGB color value of the shadow property.

 

 

Code

 

 

Shortcut

 

 

Notes

 

myStyle = new FStyleFormat();

myStyle.shadow = 0xFF99CC;

myStyle.applyChanges();

myStyle.addListener (myComponent);

 

 

myStyle = new FStyleFormat({shadow: 0xFF99CC});

myStyle.applyChanges();

myStyle.addListener (myComponent);

 

 

This code will change the shadow region of the instance myComponent to a light blue color.

 

Examples and practical uses

You can use this property to change the shadow color regions in your component instances.

Programming

In the example fstyleformat.shadow.fla and fstyleformat.shadow.swf , we assign a new color value to the background of the component instances myBox and myCombo. Then the text color is changed to white so it is still visible with this changed background color property. These new property values are applied to the style format, and then add listeners for this custom style format to myBox and myCombo.

 

myStyle = new FStyleFormat();

myStyle.background = 0x333399;

myStyle.textColor = 0xffffff;

myStyle.applyChanges();

myStyle.addListener(myBox, myCombo);

Tips and precautions

It is a good idea to consider this color value along with those for the darkshadow, highlight, and highlight3D properties. All of these properties working together will produce a realistic edge, which appears to be depressed when the button is pressed during runtime.

 

When you choose your colors, it is a good idea to choose a dark color for darkshadow, and then continually lighter for highlight,shadow, and the lightest value for highlight3D. When a component is disabled, it will retain these four color values. However, buttons cannot be pressed and the text color will change.
