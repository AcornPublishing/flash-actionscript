gotoAndPlay

gotoAndPlay(a)

 

or

 

gotoAndPlay("a"[,b])

 

Where a is the name or number of a frame or the name of a scene. Scene and frame names are put in double quotation marks. b is an optional frame number.

 

Compatible with Flash 2. This is one of the many actions used to control the playhead in your Flash movies. The playhead is sent to a frame, and it plays the movie from this location forward. Also refer to the related entries on Movieclip.gotoAndPlay, play, nextFrame, Movieclip.play, and gotoAndStop.

Description

This actionis used to send the playhead to a frame, and from that location it will continue to play.

Notes

 

 

Notes

 

on (release){

gotoAndPlay("page2");

}

 

 

 

 

This action placed on a button causes the playhead to move to the frame labeled with page2, and play.

 

 

this.myMC.gotoAndPlay(45);

 

 

 

On the main timeline, this action moves the playhead within a movie clip, myMC, to move to 45 and play.

 

 

myMC.gotoAndPlay(1);

 

Frame 1 of myMC has a stop action.

 

It is probable that the movie will play past the stop action. In this situation, make sure you use gotoAndStop instead.

 

Examples and practical uses

This is possibly one of the most common, yet useful, actions used in ActionScript. This action is one of the primary ways of controlling the playhead in a Flash movie.

Programming

In the example gotoandplay.fla and gotoandplay.swf , this action is used in several different ways. The first gotoAndPlay action is between "intro" and "scene2", on a button, which takes the movie to "scene2", on the first frame labeled "start":

 

on (release){

   _root.gotoAndPlay("scene2","start");

}

 

There are three more examples in the second scene of this FLA. Buttons control three animations: one is a SWF loaded onto level 1, the second is a movie clip placed on the stage, and the third is a SWF loaded into a movie clip placeholder. Each movie clip is being controlled by the main movie and the gotoAndPlay action.

Tips and precautions

If you are using this action to go to a frame which has a stop() action, a common result is for your playhead to continue playing the frame sequence. If you actually want your movie to stop, it is a good idea to use the gotoAndStop action instead.

 

When you are designing movies with the intention of loading them into another movie, be careful when constructing your actions within that movie. It is a good idea to use this, or _parent instead of _root or you may encounter unexpected results from your scoping. Therefore, if you use the action gotoAndPlay within that movie to control it, the code would be similar to the following:

 

this.gotoAndPlay("myLabel");

 

Also, whenever you attempt controlling movies loaded onto levels, make sure that the content is loaded before attempting to control it with these actions.
