XML.nodeValue

myXMLNode.nodeValue

 

Where myXMLNode is an instance of the XML object.

 

This property is compatible with the Flash 5 Player and above. XML.nodeValue returns the node value of an XML object. This is only applicable to type 3 text nodes. Refer to related entries on XML.nodeType and XML.nodeName.

Description

The XML.nodeValue property is used to retrieve the node value of a given type 3 node of an XML object instance. If the object is an XML element the node type is 1, the node value is null. If the object is a type 3 text node, the property returns the value of this node.

myNode = myXML_xml.firstChild.childNodes[0];

whiskers is the node value of the first child node of <cat>.

 

myXML_xml = new XML("<cat>whiskers</cat>");

myNode = myXML_xml.firstChild.childNodes[0];

trace(myXML_xml.nodeValue(myNode));

Examples and practical uses

The XML.nodeValue property is designed to hold the text content of a text node (when the node type is 3). If the property’s object is an XML element the node type is 1, then a null is returned. For example, if you tried to extract the nodeValue of <cat>:

 

myXML_xml = new XML("<cat>whiskers</cat>");

trace(myXML_xml.firstChild.nodeValue);

 

it would return a value of null. However, as you saw in our earlier code, if you traced:

 

myXML_xml.firstChild.childNodes[0]

 

you access the text node, so it would then return whiskers.

 

You can also extract the data using the XML.toString method, as it returns the value in string form. Therefore, the following code will also return whiskers:

 

myXML_xml = new XML("<cat>whiskers</cat>");

trace(myXML_xml.firstChild.childNodes[0].toString());

 

See the example xml.nodevalue.fla and xml.nodevalue.swf .

Tips and precautions

The properties nodeName and nodeValue are used all the time when processing XML structures. As long as you remember what type of nodes you are dealing with, or check them using the nodeType property, you will be able to extract the data you need.
