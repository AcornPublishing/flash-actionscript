FComponent.getPaneWidth

(This method is applicable to FScrollPane.getPaneWidth. )

 

a = myComponent.getPaneWidth();

 

Where the variable a equates to the value of the width of the display area for the scroll pane instance myComponent.

 

This method is compatible with Flash 6.

Description

This method allows you to retrieve the width of a scroll pane’s display area. You would typically need the value returned by this method when you want to resize a pane.

 

 

Code

 

 

Notes

 

a = myScrollPane.getPaneWidth();

 

Variable a is equated to the width value of the scroll pane instance myScrollPane.

 

Examples and practical uses

The getPaneWidth method is normally used in conjunction with two other methods of the scroll pane, FComponent.getPaneHeight and FComponent.setSize (see FComponent.getPaneHeight and FComponent.setSize ). To increase the size of a scroll pane by 10 pixels, you would do this:

 

myScrollPane.setSize(myScrollPane.getPaneHeight()+10, myScrollPane.getPaneWidth()+10);

 

… and to decrease its size by 10 pixels you would do this:

 

myScrollPane.setSize(myScrollPane.getPaneHeight()-10, myScrollPane.getPaneWidth()-10);

 

See also the example fcomponentgetpaneheight.fla / fcomponentgetpaneheight.swf (this example uses the getPaneHeight and the getPaneWidth methods together)

Tips and precautions

When resizing a scroll pane you should:

·       Use the methods getPaneHeight and getPaneWidth to retrieve the current size of the pane.

·       Resize the pane via the FComponent.setSize method.

 

You should not access the _width, _height, _xscale or _yscale properties directly to perform scaling, because this will scale not only the scroll pane content window area, but also:

 

·       The content itself

·       The scrollbars associated with the scroll pane

 

(See the Fcomponent.getPaneHeight entry’s Tips and precautions section for an example demonstrating this point.)

 

This raises an important point when using components: you should always access components through their methods, and never via the low level properties of the underlying movie clip object. Components are built around true Object Orientated Programming (OOP) techniques, and one consequence of this is that you should control them through their methods, and not through the lower level objects (movie clips, etc.) that make them up.
