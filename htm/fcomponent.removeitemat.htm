FComponent.removeItemAt

(This method is applicable to FComboBox.removeItemAt and FListBox.removeItemAt. )

 

a = myComponent.removeItemAt(index);

 

Where myComponent is a combo box or list box instance. The label at index index will be removed, and a will be equated to an object with properties label and data, which will contain the label and data associated with the deleted item (or undefined if they don’t have a value).

 

This method is compatible with Flash 6.

Description

This method will delete the label at the specified index. The combo box or list box will be redrawn to reflect this change.

Notes

 

myListBox.removeItemAt(4);

 

Removes the label at index 4 of the list box myListBox. This will remove the 5th label from the top (because the indexes start at zero).

 

 

component.removeItemAt(component.getSelectedIndex());

 

 

Removes the last selected item of instance component. See also FComponent.getSelectedIndex.

 

You would typically use this code within the component change handler.

 

 

selected = component.removeItemAt(component.getSelectedIndex());

    for (prop in selected) {

          message = prop+" = "+selected[prop]

          trace(message);

    }

 

Removes the last selected item and assigns the label, data and __ID__ properties to variable selected, making it an Object object. The for … in loop traces the values of these properties to the Output window.

 

You would typically use this code within the component change handler.

 

Examples and practical uses

There are many cases where you want to remove the last selected item or items that are no longer relevant from one or more components. For example:

 

·       If you had two combo boxes specifying types of garments and available colors, some garments may not be available in all colors, and you would have to adjust the ‘color’ combo accordingly. Likewise for the garment combo if the user selected a color.

·       If you were using a list or combo as a navigation device, you would want to remove the label corresponding to the current page. For example, if the user selected ‘about’, once the site had moved to the about page, the about label should be removed because you have no reason to click it again.

 

In short, removing and adding items allows you to build context sensitive combo and list boxes.

 

See also the example fcomponentremoveitemat.fla / fcomponentremoveitemat.swf.

Tips and precautions

You can also remove all elements using FComponent.removeAll. If you want to remove a significant amount of options, you may be better off removing them all and re-populating the ones you need with FComponent.addItemAt.
