Color.getRGB

a = myColor.getRGB();

 

Where a is the variable that you want to equate to the current RGB value of the color object myColor.

 

This method is compatible with Flash 5 or later. In Flash 5, the Color object can be applied to movie clips only. In Flash 6, it can be applied to movie clips, buttons, or dynamic/input text fields. See the entry for Color to see how you can work around this via a wrapper.

 

NB – This method is rarely used, and can give confusing information when used. You are better advised to write code that tracks the changes it has made via the color object by storing changes you want to recall latter in variables specially set up for this task.

Description

This method will hold the value of the color object's current RGB value. This value will not always be equal to the current RGB color value of the target associated with the color object. See also Color constructor for a definition of the target.

 

 

Code

 

 

Notes

 

my_color = new Color(my_mc);

my_color.setRGB(0x0000FF);

trace(my_color.getRGB().toString(16));

 

A new color object is created called my_color. A Color.setRGB() method is then used to set the target clip, my_mc to blue. The last line returns the value of this color change (0xFF ).

 

Note the use of String.toString(16) to convert the decimal value to Hex before displaying it. Without the ".toString(16) " part, the value returned would be 255.

 

 

my_color = new Color(my_mc);

trace(my_color.getRGB().toString(16));

 

The getRGB value will be zero if the color object has not yet applied any method that makes color changes. In other words, the getRGB method cannot be relied on as giving the current color of the target.

 

my_color = new Color(my_mc);

myColorTrans = {ra:70, rb:77, ga:70, gb:77, ba:70, bb:0, aa:100, ab:0};

my_color.setTransform(myColorTrans);

trace(my_color.getRGB().toString(16))

 

The getRGB value returns the last RGB value applied vie the last applied Color.setTransform, but because this transformation is one that will leave the target with more than one color, the getRGB value is all but useless. See also Color.SetTransform.

Examples and practical uses

None. See Tips and precautions.

Tips and precautions

The getRGB method may return confusing information unless you know exactly what the last color transformation was. In particular the getRGB value will only return something meaningful (i.e. the color of the target object) if getRGB is used after a Color.setRGB and you have used no Color.setTransform methods and you have added no manual color effects during authoring (via the Color dropdown menu in the Property inspector ). The word "useless" quickly comes to mind!

 

You are much better off tracking any color changes you make via variables:

 

a = 0x0000FF;

my_color = new Color(my_mc);

my_color.setRGB(a);

trace(my_color.getRGB().toString(16));

 

The last line can just as easily be replaced via the more dependable:

 

trace(a);
