Key.INSERT

Key.INSERT

 

Compatible with Flash Player 5 and later. There are no known issues with any versions of Flash that support this method.

 

Key.INSERT is a data constant assigned the ASCII value for the Insert key (ASCII 45).

Description

Key.INSERT holds the ASCII value (ASCII 45) for the Insert key. It cannot be reassigned.

 

 

Code

 

 

Notes

 

_root.onEnterFrame = function() {

   if (Key.isDown(Key.INSERT)) {

      trace("insert key is down");

   }

};

 

The "do something" code will be executed every for every frame the Insert key is down.

Examples and practical uses

As with all the defined constants in the Key object, Key.INSERT holds the ASCII value of a particular key, in this case ASCII 45. It is usually used as part of a condition test:

 

keyListener = new Object();

keyListener.onKeyUp = function() {

   lastKeypress = Key.getCode();

   if (lastKeypress == Key.INSERT) {

      output.text = "You pressed the Insert key.";

   } else {

      output.text = "That was not the Insert key.";

   }

};

Key.addListener(keyListener);

 

As with all Flash constants, the values cannot be changed.

Tips and precautions

Testing all the known Key constants for particular key identities involves using if... conditional statements. Stepping through these constants with nested if...s can be an easy way of quickly checking key press identities.

 

Note that the Flash player has to have browser focus before it can start sensing keyboard presses. This means that in the final presentation, you must make the user click within the SWF file at least once with the mouse before beginning to test for subsequent key presses.

 

Some key presses will not be detected if you test the movie in the Flash environment because Flash will capture them. For example, "backspace" and Enter are captured by Flash for itself. You are therefore much better off viewing your SWFs embedded in HTML (i.e. rather than using Control > Test Movie, use File > Publish Preview > HTML).
