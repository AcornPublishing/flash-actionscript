Math.sin

a = Math.sin(b);

 

Where a is the variable that you want to equate to the sine of b.

 

This method is compatible with Flash 5 or later.

Description

This method will return the sine of the angle b. This angle must be in radians, and the method will return a number from ­1.0 to 1.0.

 

 

Code

 

 

Notes

 

a = Math.sin(0);

 

b = Math.sin(Math.PI);

 

c = Math.sin(-Math.PI);

 

 

 

In the first line, a will be set to 0; the starting position of the sine wave.

 

In the second line, b will be set to 0, which is exactly halfway through the cycle.

 

In the third example, c will also be set to 0, because the wave is identical in the negative direction.

 

 

d = Math.sin ("text");

 

If anything other than a number is passed into the function, then Flash will return NaN; which stands for "Not a Number".

 

Examples and practical uses

The result of this function ends up being a cycling pattern, which begins at 0 (when a 0 is passed in), climbs up to 1, then down across 0 to ­1, then back up to 0 where the cycle repeats. The cycle repeats every 2 PI radians.

 

 

To see Math.sin plotted on a graph see the example math.sinplot.fla and math.sinplot.swf .

 

It’s possible to use this cycling effect to pass a variable to the method, and as long as the variable is smoothly incremented each frame, then Math.sin will return an ever cycling stream of numbers, ranging from ­1.0 to 1.0. Taking those numbers, and multiplying them by N, will mean that Math.sin returns a stream of numbers cycling from ­N to N. We can see this used effectively in the example math.sin.fla and math.sin.swf .

 

If we have an angle in degrees, we can convert it to radians before passing it into the Math.sin function, like so:

 

rad = angle / (180 / Math.PI);

myMC._y = Math.sin(rad);

 

This way, we could do all of our computations with variables that were based on 360 degrees in a circle, and then at the last possible moment, convert that angle into the radians that Math.sin understands.

Tips and precautions

In Flash 4, the methods of the Math object are emulated, and may therefore not always be as accurate as in Flash 5 and above.
