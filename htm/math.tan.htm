Math.tan

a = Math.tan(b);

 

Where a is the variable that you want to equate to the tangent of b.

 

This method is compatible with Flash 5 or later.

Description

This method will return the tangent of the angle b. This angle must be in radians, and the method will return a number from ­Infinity to Infinity.

 

 

Code

 

 

Notes

 

a = Math.tan(0);

 

b = Math.tan(Math.PI);

 

c = Math.tan(Math.PI / 2);

 

 

 

 

In the first line, a will be set to 0; the starting position of the tangent wave.

 

In the second line, b will be set to 0, which is the next point that the tangent wave crosses 0.

 

In the third example, c will essentially be undefined, because at PI/2, tangent technically returns Infinity. See the graph in the Examples and practical uses section.

 

 

c = Math.tan ("text");

 

If anything other than a number is passed into the function, then Flash will return NaN; which stands for "Not a Number".

 

Examples and practical uses

The result of this function ends up being a cycling pattern, which begins at 0 (when a 0 is passed in), climbs up to Iinfinity, then loops back around to ­Infinity, then climbs back up to 0 where the cycle repeats. The cycle repeats every PI radians.

 

 

To see Math.tan plotted on a graph see the example math.tanplot.fla and math.tanplot.swf .

 

If we have an angle in degrees, we can convert it to radians before passing it into the Math.tan function, like so:

 

rad = angle / (180 / Math.PI);

myMC._y = Math.tan(rad);

 

This way, we could do all of our computations with variables that were based on 360 degrees in a circle, and then at the last possible moment, convert that angle into the radians that Math.tan understands. There aren’t many practical applications for the Math.tan method, like there are for Math.sin and Math.cos. Math.tan technically tells us the slope of a line at any angle around a circle, which happens to be known as the "tangent" to the circle.

Tips and precautions

In Flash 4, the methods of the Math object are emulated, and may therefore not always be as accurate as in Flash 5 and above.
