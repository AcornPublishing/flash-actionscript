Color.setTransform

myColor.setTransform(a);

 

Where myColor is a color object, and a is the color transform object that represents the color changes you want to apply during the transform.

 

This method is compatible with Flash 5 or later. In Flash 5, the Color object can be applied to movie clips only. In Flash 6, it can be applied to movie clips, buttons, or dynamic/input text fields. See the entry for Color to see how you can work around this via a wrapper.

Description

This method allows you to apply color, mix, filter, and tint effects toa movie clip, button or text field. See the entry for Color for a fuller discussion on the terms color, mix, filter, and tint.

 

The argument for this method is best expressed in hexadecimal. See also the entry for Color for a fuller discussion and links that explain the use of hexadecimal with color representation.

 

You must construct a color object before using this (or any other) method. See also the entry for Color constructor. The examples in the table below also show you how to create a color object in the first line of each example listing.

 

You must construct a color transform object before using this method. The examples in the table below all create a color transform object called myColorTrans on line 2.

 

The color transform object contains the same information as that entered in the Advanced Effect window. See the entry for Color for more information on how to use this window.

 

 

ra

The red RGB channel (in percent of the original symbol color value) to use in the final color transformed symbol.

 

ga

The green RGB channel (in percent of the original symbol color value) to use in the final color transformed symbol.

 

ba

The blue RGB channel (in percent of the original symbol color value) to use in the final color transformed symbol.

 

aa

The alpha value (in percent of the original symbol) to use in the final color transformed image.

 

rb

The red channel offset (-255 to +255) to add to the red channel of the original symbol.

 

gb

The green channel offset (-255 to +255) to add to the red channel of the original symbol.

 

bb

The blue channel offset (-255 to +255) to add to the red channel of the original symbol.

 

ab

The alpha offset (-255 to +255) to add to the red channel of the original symbol.

 

 

Tip – the range values shown in the table above can be broken when using ActionScript, because they only apply to the Advanced Effect window sliders. We shall see this later in the examples.

 

The "a" values allow you to reduce the percentage of the existingcolor (and alpha) values on the target clip, button, or text-field. The "b" values allow you to specify an absolute figure that is added to the color.

 

Setting up the color transform object is the key part of using the setTransform action. This object must have the eight values ra to ab listed above as its properties:

 

 

The easiest way to set a color transform is to use the {} braces as object initializers (see also the entry for {} (object initializer). The following color transform object would add a blue tinge and reduce the alpha to 70%.

 

myColTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:50, aa:70, ab:20}

 

NB – although it would be normal to expect the absolute color values rb, gb, and bb to be expressed in hex, the values may well be taken from the "advanced effect window", where they are expressed in decimal. Consequently, the values in the color transform object are just as likely to be expressed in decimal.

Examples

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:0, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

 

 

This color transition object actually does nothing, but it is a good starting point for further investigation, because changing the values one at a time will show you their effects. This color transition object is set to the default values you see when you first open the advanced effect window.

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:40, rb:0, ga:100, gb:0, ba:100, bb:0, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

This color transform provides a filtering effect. The red component % (ra ) is filtered or reduced to 40 %. All other values are left at the default. The remaining components begin to dominate, resulting in the image tending towards the color cyan (blue-green).

 

   

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100000000, rb:0, ga:100, gb:0, ba:100, bb:0, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

This color transform provides a strong color amplification effect. The red component is increased dramatically into over-saturation.

 

A little known trick: the advanced effect window % values actually go between 1000 and -1000 should read "-1000" on same line rather than the 100 to -200 implied by the sliders. setTransform can accept numbers much larger than even this, as seen here. This feature can be used to create burn and over saturation effects, and can be useful to create solid "cartoon" color effects, especially when applied to moving video embedded inside a standard movie clip wrapper.

See also the entry for Color.

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:50, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

The blue color offset (bb ) is increased from 0 to 50. This means that every pixel in the movie clip will have 50 points added to its existing blue color component, making the pixels appear more blue and brighter

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:-50, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

This time, the blue color offset (bb ) is reduced by 50 to -50. This means that every pixel in the movie clip will have 50 points subtracted from its existing blue color component, making the pixels appear more yellow and darker.

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:0, aa:50, ab:0};

my_color.setTransform(myColorTrans);

 

The alpha percentage is reduced from 100 to 50 (aa ). This reduces the alpha value of each pixel by 50% of its original value.

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0, ga:100, gb:0, ba:100, bb:0, aa:100, ab:-50};

my_color.setTransform(myColorTrans);

 

The alpha offset is reduced from 0 to -50 (ab ). This reduces the alpha value of each pixel by 50 points of its original value.

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:-100, rb:255, ga:-100, gb:255, ba:-100, bb:255, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

Once you realize how the setTransform method works, you quickly find that it can be used to create some really startling effects, particularly when used on video and animating movie clips.

 

Cool color transformation #1 : negative photograph. Inverse video is also good for suggesting a selection; graphics going to inverse video is a good way of suggesting a group or block selection.

 

 

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:500, rb:-300, ga:500, gb:-300, ba:500, bb:-300, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

Cool color transformation #2 : solid color. Great when used with Flash 6 video to give a "poster" effect

Examples and practical uses

The setTransform method is really the only color method, given that Color.setCol is simply a cut down version of it that requires less parameters. The setTransform method can completely change the appearance of a vector graphic object, and can also be used to good effect when used with bitmaps or video embedded inside a movie clip.

 

See also the example color.settransform.fla and color.settransform.swf .

Tips and precautions

The real use of setTransform is when the parameters are varied in a loop to create animating transitions. Also, remember that the parameters can be set well outside the values suggested by the Macromedia documentation, and that's when the real fun starts!
