FStyleFormat.textSize

a.textSize = b;

 

Where a is the name of your custom style format, and b is an integer specifying the point-size of the text.

 

Compatible with Flash MX. This property should be considered when setting the textFont property, and the size of your component. Related entries include FStyleFormat.textFont, and FComponent.setSize.

Description

This property, within a custom style format, changes the RGB value of a radio dot listening to the custom style.

myStyle = new FStyleFormat();

myStyle.textSize = 8;

myStyle.applyChanges();

myStyle.addListener (myButton);

 

 

myStyle = new FStyleFormat({textSize:8});

myStyle.applyChanges();

myStyle.addListener(myButton);

 

 

This code will change the point size of the text face in the instance myButton to 8.

 

Examples and practical uses

This property is useful when working with custom fonts especially. There are many fonts used in design, such as pixel fonts, which rely on specific text size settings. Therefore, the textFont and textSize properties go hand in hand.

Programming

Several different text sizes can be set simultaneously during runtime. In the example fstyleformat.textsize.fla and fstyleformat.textsize.swf , we see how this can be accomplished. Three new style formats are created:

 

changeSize = new FStyleFormat({textSize: 21});

changeSize.applyChanges();

 

changeSize2 = new FStyleFormat({textSize: 15});

changeSize2.applyChanges();

 

changeSize3 = new FStyleFormat({textSize: 8});

changeSize3.applyChanges();

 

On the stage is a button called button1 which has a change handler called pButton1. Because we are working with a list box component, we need to set the size of the list box instance list1. This will not cause it to remain perfectly the same, but will instead stop it from decreasing each time a listener is added or removed.  The function written for button1 is as follows:

 

//function to add listeners for button 1

function pButton1(){

   changeSize.addListener(list1);

   changeSize2.addListener(combo1);

   changeSize3.addListener(combo2);

   //set size of the ListBox

   list1.setSize(100,101);

}

 

Changing text size in your components may be useful if you need to adjust your movie for those who may need larger text to read your content.

Tips and precautions

When working with list boxes, textSize, addListener, and removeListener you may notice unpredictable results. When changing the textSize property in a combo box, it will resize to fit the point-height of the text. The width (but not the height) of your combo box components can be adjusted with the Free Transform Tool or setSize.

 

When changing the textSize property in a list box, you will notice the size of your component will decrease each time you add or remove a listener. This will repeat each time you do so until it is two lines in height. You can avoid this problem by using FComponent.setSize each time after you add or remove listeners setting your textSize in the component instance.
