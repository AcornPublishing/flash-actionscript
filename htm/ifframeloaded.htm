ifFrameLoaded

ifFrameLoaded(a){

   statement;

}

 

or

 

ifFrameLoaded("a",b){

   statement;

}

 

Where a is the name or number of a frame or scene, and b is a number of a frame.

 

Compatible with Flash 3. This property is deprecated in Flash 5 in favor of the MovieClip._framesloaded action. This action currently works in Flash MX. Related entries include the getBytesLoaded and getBytesTotal methods, and MovieClip._framesloaded.

Description

This action checks to see if a specified frame has loaded. When this frame has loaded, the statement executes.

gotoAndStop("mymovie ", 1);

This action will go to frame 1 of the scene mymovie after frame 15 of mymovie has loaded.

Examples and practical uses

This action is used to download content before the movie starts playback. Typically, it is placed at the beginning of a movie, and loops between two frames while it downloads. The loop will keep checking if the frame you have specified has loaded to the local computer. When it has downloaded the set frame, it executes your statement.

Programming

This action is used to check if a certain number of frames are loaded before the movie progresses. In our example ifframeloaded.fla and ifframeloaded.swf , a movie loops between the first two frames in a movie. On the first frame is the following code:

 

ifFrameLoaded("theend"){

   gotoAndPlay("thebeginning");

}

 

On the second frame is simply an action to return to the first frame. This will loop between frame 1 and 2 until the frame theend is downloaded onto the local machine. When testing your Flash movie in the test environment, try pressing Ctrl+Enter after the movie is in the test player with the bandwidth profiler active. You can simulate a slow download speed and watch the movie loop until it is fully loaded, and skip to the frame labeled thebeginning.

Tips and precautions

Because this action is now deprecated, support in current or future versions of the Flash Player is not known. Using this action is not recommended unless you are aiming for backward compatibility to early versions of Flash. If this is not a concern, it is recommended to use the getBytesLoaded and getBytesTotal methods.
