System.capabilities.hasVideoEncoder

a = System.capabilities.hasVideoEncoder;

 

a will be returned as a Boolean value (PC/Mac) or an array containing the supported video capabilities (wireless or pocket/portable device).

 

This property is compatible with Flash 6 or later. This property is read only.

Description

This property is used to find out the video streams supported by the Flash player. This is Boolean true for the standard Flash player on a typical desktop, but will return an array containing the particular capabilities of the Flash player for wireless or portable devices such as PDAs and pocket PCs.

Examples and practical uses

This property is only useful for non-standard systems, such as PDAs and pocket/mobile devices. Such devices will typically support a reduced set of the normal flash player's features, and may include custom or proprietary capabilities.

 

See the example system.capabilities.fla and system.capabilities.swf .

Tips and precautions

You should note that this value refers to the platform and not the specific user. For example, a PC will always have a Flash player that supports the Sorenson codec (given that Flash 6 is the only Flash player that supports the System object; other earlier versions of the Flash player don't support video, but they don't support the System object either!), and all the standard Flash player sound formats. A particular user may have an integrated PDA and mobile phone that has a Flash player that supports a cut down or proprietary video format.

 

In particular you should note that, this property will not look at other video codecs a PC or Mac may have installed in the operating system, and would not for example realize that the windows media player or QuickTime are installed on a users machine.

 

This property is geared towards versions of the Flash player on non-standard devices, and these will generally be mobile/pocket/wireless devices.
