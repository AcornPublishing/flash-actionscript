FComponent.getPaneHeight

(This method is applicable to FScrollPane.getPaneHeight. )

 

a = myComponent.getPaneHeight();

 

Where the variable a equates to the value of the height of the display area for the scroll pane instance myComponent.

 

This method is compatible with Flash 6.

Description

This method allows you to retrieve the height of a scroll pane’s display area. You would typically need the value returned by this method when you want to resize a pane.

 

 

Code

 

 

Notes

 

a = myScrollPane.getPaneHeight();

 

Variable a is equated to the height value of the scroll pane instance myScrollPane.

 

Examples and practical uses

The getPaneHeight method is normally used in conjunction with two other methods of the scroll pane, FScrollPane.getPaneWidth and FScrollPane.setSize (see FComponent.getPaneWidth and FComponent.setSize ). To increase the size of a scroll pane by 10 pixels, you would do this:

 

myScrollPane.setSize(myScrollPane.getPaneHeight()+10, myScrollPane.getPaneWidth()+10);

 

… and to decrease its size by 10 pixels you would do this:

 

myScrollPane.setSize(myScrollPane.getPaneHeight()-10, myScrollPane.getPaneWidth()-10);

 

See also the example files fcomponentgetpaneheight.fla and fcomponentgetpaneheight.swf.

Tips and precautions

When resizing a scroll pane you should:

 

·       Use the methods getPaneHeight and getPaneWidth to retrieve the current size of the pane.

·       Resize the pane via the FComponent.setSize method.

 

You should not access the _width, _height, _xscale or _yscale properties directly to perform scaling, because this will scale not only the scroll pane content window area, but also:

 

·       The content itself

·       The scrollbars associated with the scroll pane

 

You can see this in the examples below. The first one uses the _width property of our scroll pane instance myScrollPane, and this gives an undesirable stretched appearance. The second version correctly scales the scroll pane window area only, and leaves the scrollbars and the content at their correct dimensions.

 

myScrollPane._width=+400;

 

 

myScrollPane.setSize(myScrollPane.getPaneWidth()+400, mySCrollPane.getPaneHeight());

 

 

This raises an important point when using components: you should always access components through their methods, and never via the low level properties of the underlying movie clip object. Components are built around true Object Orientated Programming (OOP) techniques, and one consequence of this is that you should control them through their methods, and not through the lower level objects (movie clips, etc.) that make them up.
