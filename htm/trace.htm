trace

trace(expression)

 

Where expression is an expression to evaluate, such as a string.

 

Compatible with Flash Player 4 and later.

 

The trace() method evaluates an expression and displays the result in the Output window. This feature should only be used in the authoring environment as a debugging tool.

Description

The trace() method is used to perform an evaluation on an expression and display the result via the Output window. This window should only be used during authoring, and will not appear in the final SWF. Usually, the trace() method is used to display strings or variable values as required during the debugging process during content creation.

Examples and practical uses

The trace() method is used to display results of an expression. The easiest usage is to display a string, like this:

 

trace( "this is a string" );

 

which will display the string. You can embed variables in the expression:

 

trace( "The value of the variable is " + varname );

 

or perform calculations within the expression:

 

trace( "The maximum value is " +  Math.max(var1, var2));

 

Any valid expression can be placed in the trace() method. This means that although there is a limit of one item, to be displayed, you can actually display several values by concatenating several variable values into a single string:

 

a = 20;

b = 30;

c = 40;

message = "a: "+a+"  b: "+b+"  c: "+c;

trace(message);

You can also add formatting as seen below:

 

a = 20;

b = 30;

c = 40;

message = "a: "+a+"\nb: "+b+"\nc: "+c;

trace(message);

 

 

This listing adds the newline control code (\n) to separate the output.

Tips and precautions

The trace() method is useful for debugging a movie script, allowing you to display status messages and variable values as the script progresses. When you publish a movie, you can disable all trace() methods by using the Omit Trace option in the Publish Settings window (this prevents you from having to manually delete all the trace() methods.

 

Although the advanced features of the Debugger window allow you to debug in real time, the trace action still has many uses:

 

·         The Output window allows you to save the output as a file and take hard copies, as well as specifying the level of debugging information. It also allows you to search the output file for particular values or variable names. All these options can be found in the (little used) Options menu. Now you know!

 


 

·         The trace action sometimes provides faster debugging facilities than using the debugger in single step mode (breakpoints).

·         The Debugger window doesn't work well with some debugging applications, particularly where you are trying to debug real time user inputs (such as keyboard entry via the Key object). The trace action and Output window doesn't suffer from this constraint.

·         Certain actions (such as typeof) provide features not available to the Debugger window, and the quickest way to get hold of such information is to trace it.

 

When debugging certain FLAs, you may find that the trace action fills the Output window too quickly for you to see what is going on. As with any debugging, it is easier to find out what is happening if you slow Flash down temporarily by reducing the frame rate to 1-5 fps.
