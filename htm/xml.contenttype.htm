XML.contentType

 

myXML.contentType

 

Where myXML is an instance of the XML object.

 

XML.contentType is compatible with Flash 5 Player (release 41) and above. You would use this property to set the MIME type used by XML objects to send to a server. Refer to related entries on XML.sendAndLoad and XML.send.

Description

The XML.contentType property sets and retrieves the MIME type used when sending data to a server by the XML.send() and XML.sendAndLoad() methods. By default the value is set to application/x-www-form-urlencoded MIME format.

The content type must be expressed as a string.

Examples and practical uses

The XML.contentType property indicates the MIME format used to transfer data to the server. The content header will indicate what kind of data is arriving at the server. The current value can be stored in a variable:

 

myXML_xml = new XML();

myXML_xml.contentType = "text/xml";

myMime = myXML_xml.contentType;

myXML_xml.send("myserver.cfm", "_blank");

trace (myMime);

 

This code traces text/xml in the output window. We are setting the content type for the myXML_xml instance of the XML object to text/xml.

 

There are several different kinds of content descriptors commonly used, such as application/xml and text/xml. Therefore, you can indicate to the server what kind of information it is receiving. This will be useful when you need to send XML formatted data to an application server from Flash, as it will improve the communication between the two applications. For example, you would want to set the contentType to "text/xml" when sending XML to a PHP script so it is not separated into key value pairs.

 

See also the example xml.contenttype.fla and xml.contenttype.swf .

Tips and precautions

It is important to remember that despite using the default settings of this property that your data is not URL encoded. The default value, application/x-www-urlform-encoded , is in place because such a header is widely used and required by several different kinds of application servers (middleware). This default value is the default used for ColdFusion.

 

You should also remember when designing your movies that this property is only available in later releases of the Flash 5 Player, and Flash Player 6. It does not work in version 30 of the Flash 5 Player, and this property can only be invoked on an instance of a top level XML class node.

 

The default MIME format setting of application/x-www-form-urlencoded is used for most CGI scripts. This uses a layout where each variable name is followed by an equal sign, the value assigned to the variable, an ampersand, and then the next variable-value pair until all variables are assigned, like this:

 

Petname=pepper&PetType=cat&PetAge=5&PetColor=tabby

 

Words within an application’s MIME format string must be separated by a plus sign, like this:

 

Petname=Mr+Big&PetType=cat&PetAge=5&PetColor=tabby

 

Avoid using any other MIME format than application/x-www-form-urlencoded unless you are sure you need to; almost all CGI scripts use this format.
