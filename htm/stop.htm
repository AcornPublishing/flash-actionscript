stop

stop();

 

This action is used to stop a timeline at the current frame (the frame in which the stop action is executed).

 

Compatible with Flash 2 and above. There are no known issues with any version of Flash that supports this action.

 

It can be used to stop a timeline from playing (therefore it only relates to halting movie clips and the _root timeline) in which the action is executed on.

 

Related actions include MovieClip.stop (which is the same as stop, except it references a particular movie clip) and Sound.stop (which is similar to stop in that it causes a particular sound to stop playing), play and MovieClip.play, as they are the opposite of stop (causing a timeline to play).

Description

This action is used only to stop an untargeted timeline playing.

stop();

 

Will stop the timeline of the movie clip in which the action resides, being either on a frame or a button.

 


targetClip.onEnterFrame = function () {

 

 if (_root.stopMotion == true) {

 

 stop();

 

 }

Incorrect usage.

 

This code will not stop targetClip ’s timeline (the intended usage), but will instead stop the timeline where this code resides.

Examples and practical uses

The stop action can be used in frames, or on buttons and movie clips (anything with a timeline, or inherent timeline).

Frame control

The following event code will stop the movie clip from playing (in which the buttons resides, this includes the _root timeline) when the button is pressed.

 

on(press) {

 

 stop();

 

}

 

See also the example stoproottimeline.fla / stoproottimeline.swf.

Tips and precautions

Be careful not to confuse stop with MovieClip.stop, or to use stop where you should use MovieClip.stop, because of the similarity between the two it is quite easy to confuse them. stop automatically targets the timeline the action is executed on. For example, the following code demonstrates using stop on a frame in the _root timeline to stop it playing.

 

stop();

 

If the above code were to be placed on a frame inside of a movie clip nested on the _root timeline, it would stop the timeline of the nested movie clip. Similarly, if the code were to be placed inside an on event of a button, it would halt the timeline of the movie clip in which the button resides; see example stopbutton.fla / stopbutton.swf.

 

The following code demonstrates incorrect usage of the stop action in an onEnterFrame event loop. See the example stoponenterframe.fla / stoponenterframe.swf.

 

targetClip.onEnterFrame = function () {

 

 if (_root.stopClipMotion == true) {

 

  stop();

 this.onEnterFrame = undefined;

 

}

 

}

 

The above code will actually stop the timeline in which this code is written (in the case of the example, the _root timeline), which is not was is intended. The intention of the example is stop the timeline of targetClip from playing. The code below fixes the erroneous code above replacing the stop action with the MovieClip.stop method.

 

targetClip.onEnterFrame = function () {

 

 if (_root.stopClipMotion == true) {

 

  this.stop();

 this.onEnterFrame = undefined;

 

}

 

}
