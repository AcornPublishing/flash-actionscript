clearInterval

clearInterval(a);

 

Where a is the identifier of the interval you want to clear (remove).

 

This action is compatible with Flash 6 or later. The clearInterval action is paired with setInterval, which sets (assigns) interval events.

Description

The clearInterval action clears an interval event associated with a previously defined named interval via setInterval. When using interval events for animation, you may also need to use the updateOnEvent action, particularly if your interval is greater than the frame rate of the main timeline (typically every 83.3ms for the default 12 fps timeline). See also the entry for updateOnEvent.

 

The interval event is a major advance in the abilities of ActionScript to control the motion graphics environment. In particular, if you use setInterval /clearInterval together, it can allow the advanced coder to emulate independent timelines with variable frame rates. This has implications in:

 

·         Optimizing events so that they only run as often as needed, and not every frame.

·         Creating animation based event code that updates faster/slower than the normal timeline based onEnterFrame event handler.

 

This issue is illustrated briefly in the following sections via additional FLA examples.

 

 

Code

 

 

Notes

 

myInterval = setInterval(myFunction, 100);

clearInterval(myInterval);

 

Line 1 defines an interval event that will execute the function myFunction() every 100ms, or as close to this as Flash can achieve.

 

Line 2 clears this event

 

 

setInterval(myFunction, 100);

 

Although the setInterval shown here will correctly create an interval event that executes myFunction every 100ms, this line does not assign this event with an identifier.

 

Because the interval is anonymous, you cannot clear this interval using clearInterval.

 

Do not use this syntax if you wish to later use clearInterval.

 

Examples and practical uses

See the example clearinterval.fla and clearinterval.swf . This animates two movie clips. One is animated via an onEnterFrame event script running at the default duration for the timeline (which is 12 fps or every 83.3ms). The second ball is being animated by an interval event that is running at approximately twice this rate (every 40ms). You can add/remove the interval event dynamically via the two buttons.

Tips and precautions

There are a number of things you should be aware of when using setInterval /clearInterval in more advanced applications.

 

·       The exact timing of the interval is approximate and still tied somewhat to the frame rate. Like all event driven code, the setInterval function will only run as often as Flash can service it or the timing you specify, whichever is the easiest to maintain. This is not a fault of Flash, but a feature of event driven code. It means that if you specify an interval event of 1ms, Flash will not be responding anywhere near that fast, but will respond as fast as it can after it has taken care of everything else.

·       If you specify high frame rates and/or lots of animation changes per frame, the timeline will soak up more of the available timing, and this leaves less processing time for setInterval.

 

An undocumented feature is that you can specify multiple setIntervals per function. This is shown in the second example, clearintervaladvanced1.fla and clearintervaladvanced1.swf . This allows you to run the same interval event more than once per interval. In the FLA, adding multiple interval events makes the ball go faster, but because all the added events run at the same instant per interval, the ball becomes less smooth.

 

To make the ball go faster and maintain smoothness, it is better to delete the last interval event and replace it with a single but more frequent one. This is shown in the example, clearintervaladvanced2.fla and clearintervaladvanced2.swf .

 

Notice that the second of these FLAs is effectively emulating fBall_mc being on a timeline that is largely independent of the main timeline, and has its own frame rate, which can be altered dynamically
