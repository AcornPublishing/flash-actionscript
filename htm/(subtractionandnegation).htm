- Subtraction and negation

a = -b;

c = d-e;

 

Where b,d,and e can be any expression that evaluate to a number. a will be returned as the negative value of the expression b (negation). c will be returned as the number value of d-e. If b,d,or e do not evaluate to a number, the returned value held in a or c will be NaN.

 

Compatible with Flash 4 and above. You should be aware that because ActionScript is not typed, if you use Booleans in error, the returned value will assume true = 1 and false = 0.

Description

This operator conforms to the usual rules of mathematical evaluation. If in doubt as to the order of such evaluation, you should use brackets.

a = true-false;

b = "10"-"3";

 

a = 1

b = 7

 

true is converted to 1 and false is converted to 0 before the subtraction is applied. This result is not usually meaningful, and an error has probably occurred if your code subtracts Booleans from each other or numbers.

 

Strings that are numerals only will work with -, but this is not the case for other arithmetic operators (such as multiplication) so use this feature with caution.

 

 

a = "my cat" - "my";

 

 

a = NaN

 

Unlike addition, subtraction cannot be used with strings.

myClip0.onEnterFrame = function(){

   this._x = this._x-(this._x-_xmouse)/10;

}

Tips and precautions

ActionScript is not typed, and will allow you to use - with types that will result in meaningless results (notably strings and Booleans).

 

NB – A typed language takes account of data types (Boolean, string, number) and does not allow you to directly combine dissimilar types, so you cannot add a string and number directly together for example. An untyped language does allow you to combine different types in this way, because it automatically makes the type conversions for you. When adding a number to a string for example, Flash will automatically convert the number to a string and give you a string result:

x = "3"+2;
trace(x);
will give you x = "32".

 

Also, note that subtracting from certain properties will give unexpected results because of the nature of the property. For an example of this, change the code in the motion graphics example above to

 

myClip.onEnterFrame = function() {

   this._alpha = this._alpha - 5;

   trace(this._alpha);

   if (this._alpha == 0) {

      this.onEnterFrame = undefined;

   }

};

 

The iterations will give you:

 

Actual                Expected

94.921875    95.0000

89.84375      90.0000

84.765625    85.0000

...      ...

8.59375       10.0000

3.515625      5.0000

-1.171875    0.0000

 

This occurs because alpha goes between 0 and 100 in 256 discrete steps (alpha goes from 255 to 0 internal to the Flash player), and this property cannot exist in 'in-between' values such as 99.000. This does mean that the property will never equal zero, and the event will never stop. A better way is to look for a value range(this._alpha > 0) and set the final value explicitly as shown:

 

myClip.onEnterFrame = function() {

   this._alpha = this._alpha - 5;

   if (this._alpha < 0) {

      this.onEnterFrame = undefined;

      this._alpha = 0;

   }

};

 

Although the in between values created via subtraction are not as expected, the start and end points of the transition are.
