MovieClip._currentframe

myMovieClip._currentframe

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 4 and later.

 

The MovieClip._currentframe property shows the number of the current frame (the frame where the playhead is located).

Description

The MovieClip._ currentframe property returns the frame number of the frame the playhead is currently positioned over in the timeline, whether the frame is being played or not.

Notes

 

myCurrentFrame = myMovieClip._currentframe;

 

 

 

myMovieClip.gotoAndPlay(_currentframe + 2);

 

Stores the frame number the playhead is over to the variable myCurrentFrame.

 

Moves the playhead two frames and begins playback.

 

 

The playhead position in a frame is not dependent on the playback state of the movie.

 

myMovieClip._currentFrame+=5;

 

The _currentframe property is read only.

 

To advance the timeline relative thethe current position, use methods such as MovieClip.gotoAndPlay().

 

Examples and practical uses

MovieClip._ currentframe lets you retrieve the frame number of the playhead position on the timeline, and use the returned value to move the playhead relative to the current position. This allows you to make generic functions that are not dependent on specific frame numbers, as well as quickly move the playhead based on user inputs. For example, if you have a button that allows a user to skip forward in a movie, the code could look like this:

 

on(release){

 myMovieClip.gotoAndPlay(_currentframe + 10);

}

 

You can also store the current frame number in a variable for use in other statements:

 

myCurrentFrame = myMovieClip._currentframe;

Tips and precautions

Although you usually want to move through a movie to definite frame labels, there are a number of situations where you will want to move to a frame relative to the current frame. This includes when you are using video in Flash and want to add fast forward/rewind buttons.
