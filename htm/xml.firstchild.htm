XML.firstChild

myXMLNode.firstChild

 

Where myXMLNode is an instance of a XML object or XMLnodes object.

 

The XML.firstChild property is compatible with Flash Player 5 and above. This property references the first child of a given instance of an XML or XMLnodes object. It is exactly the same as any reference to XML.childNodes[0]. Refer to related entries XML.appendChild(), XML.insertBefore(), and XML.removeNode() for more information on manipulating children.

Description

The XML.firstChild property examines the XML object instance parent’s child list and returns the name of the first child. If there is no first child, null is returned. If the first child is a text node, the value is set to undefined. This is a read-only property.

trace(myXML_xml.firstChild.childNodes[0].childNodes[0].firstChild);

 

 

null

 

A text node where no first child is present, returns null.

 

 

Examples and practical uses

The XML.firstChild property is a read-only value that indicates the first child in the parent node’s child list. A reference to the first child is returned. Therefore, you can use this property to access particular XML data. It is integral to understand this property, and how to navigate a structure using it, and dot notation hierarchy.

 

Consider the following XML structure:

<mylist>

   <vegetables>

      <carrots>3</carrots>

      <corn>6</corn>

   </vegetables>

   <meat>

      <beef>1lb</beef>

      <chicken>2lb</chicken>

   </meat>

</mylist>

 

If we loaded this XML structure into an XML object instance called myXML_xml, the firstChild.nodeName of myXML_xml would be mylist. If you wanted the firstChild of the <carrots> node, 3 would be returned.

 

trace(myXML_xml.firstChild.firstChild.nodeName);

 

you would have vegetables returned. You would also have vegetables returned if you traced:

 

trace(myXML_xml.firstChild.childNodes[0].nodeName);

 

If you wanted to access 6 within the <corn> node, you would use the following hierarchy:

 

trace(myXML_xml.firstChild.childNodes[0].childNodes[0].nextSibling.firstChild.nodeValue);

 

Notice that we navigate to the child node of carrots, and then access its sibling <corn>. Then the first child of this node is 6, so we navigate to the text node, and then call for its value, which is 6.

 

See the example xml.firstchild.fla and xml.firstchild.swf for further examples of how to navigate this structure using firstChild.

Tips and precautions

The XML.firstChild property cannot be set, only read. If there is no child present of the specified node, null will be returned to the player. It is important to realize that childNodes[0] is the same as firstChild.
