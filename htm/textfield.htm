TextField

Compatible with Flash Player 6 and later .

 

The TextField object handles text fields appearance and behavior. All dynamic and input text fields in a Flash text field are instances of the TextField object.

Description

The TextField object has many methods attached to it that can be invoked to control the actions and appearances of a text field. The TextField object inherits from the Object object, so all methods and properties of the Object object are available to the TextField object. Text fields can be created statically, or dynamically using the TextField.createTextField() method.

 

The TextField object contains these methods:

 

·       TextField.addListener – registers an object that will receive notification broadcasts from the text field events, onChanged and onScroller .

·       TextField.getDepth – returns the level of the text field.

·       TextField.getNewTextFormat – gets a textformat object, a copy of the text fields current text object.

·       TextField.removeListener – removes a listener object from receiving event notification broadcasts from the text field.

·       TextField.setNewTextFormat – sets a new text format for the text field.

·       TextField.replaceSel – replaces the current selection.

·       TextField.setTextFormat – sets the default text format.

 

The TextField object also has several properties associated with it:

 

·       TextField._alpha – the transparency value.

·       TextField.autoSize – controls sizing and alignment.

·       TextField.background – a flag to define if the text field has a background.

·       TextField.backgroundColor – the background color.

·       TextField.border – a flag to define if the text field has a border.

·       TextField.borderColor – the color of the border.

·       TextField.bottomScroll – an integer representing the bottom-most line in a text field (read-only).

·       TextField.embedFonts – a flag defining if the text field uses embedded fonts.

·       TextField._highquality – a integer value used to determine the rendering quality.

·       TextField._height – the height of the text field.

·       TextField.hscroll – the horizontal scroll value.

·       TextField.html – a flag to define if the text field's content contains HTML.

·       TextField.htmlText – the HTML content of the text field.

·       TextField.length – represents number of characters in the text field (read-only).

·       TextField.maxChars – indicates the maximum number of characters.

·       TextField.maxhscroll – represents maximum value of the TextField.hscroll property (read-only).

·       TextField.maxscroll – represents the maximum value of TextField.scroll .

·       TextField.multiline – defines if the text field is a multiline text field.

·       TextField._name – the instance name of the text field.

·       TextField._parent – a reference to the text fields parent movie clip (or level).

·       TextField.password – a flag defining if the text fields contents should be hidden.

·       TextField._quality – defines the rendering quality.

·       TextField.restrict – indicates the set of characters that can be used.

·       TextField._rotation – indicates the rotation of the instance.

·       TextField.scroll – the current scroll position.

·       TextField.selectable – a flag that indicates if the field can be selected.

·       TextField._soundbuftime – indicates the amount of seconds of sound to buffer.

·       TextField.tabEnabled – a flag indicating if the field is included in automatic tab ordering.

·       TextField.tabIndex – indicates the index value of the text field in tab ordering.

·       TextField.text – represents the string in the text field (non HTML text).

·       TextField.textColor – represents the color of the text (a hexadecimal color value, i.e. 0xFFFFFF ).

·       TextField.textHeight – indicates the height of the text (not the text field's bounding box).

·       TextField.textWidth – indicates the width of the text (not the text field's bounding box).

·       TextField.type – defines if the text field is a dynamic or an input text field.

·       TextField._url – represents the URL of the SWF file in which the text field exists (read-only).

·       TextField.variable – the name of the variable associated with the text field.

·       TextField._visible – a flag that defines if the text field is visible.

·       TextField._width – represents the width of the text field.

·       TextField._x – represents the position (in pixels) of the text field on the x-axis.

·       TextField._xmouse – the x coordinate of the mouse relative to the text field.

·       TextField._xscale – indicates the scaling of the text field on the x-axis.

·       TextField._y – represents the position of the text field on the y-axis.

·       TextField._ymouse – the y coordinate of the mouse relative to the text field.

·       TextField._yscale – represents the scaling in the of the text field on the y-axis.

 

The TextField object also has the following events:

 

·       TextField.onChanged – invoked when the contents is changed.

·       TextField.onKillFocus – invoked when focus is removed.

·       TextField.onScroller – invoked when the scroll properties change.

·       TextField.onSetFocus – invoked when a field receives focus.

 

The Movie clip object has one method that relates to text field creation:

·         MovieClip.createTextField – creates a new text field.

Examples and practical uses

The TextField object contains many objects, properties, and event handlers. Each is discussed individually in the following pages.

 

The TextField object behaves differently to the MovieClip and Button objects in a number of crucial ways. These are discussed below, and it is important that you understand the implications of them when controlling text fields via ActionScript.

Relationship between the TextField and TextFormat objects

Some features of the text field's appearance cannot be set via the TextField object alone. You have to instead create an instance of TextFormat object. This object handles text formatting of the text within a text field, and is discussed in its own section in this dictionary. See the entry for TextFormat .

TextField event model

You should note that the TextField object is different from either the Button and the MovieClip objects because you cannot attach events to them using the Flash 5 style, i.e. using on(event) or onClipEvent(event) ; the TextField object is new to Flash 6 and does not need to include them.

 

You have to use the Flash 6 style of coding to make full use of text fields using event driven code, and this style involves knowing how to use functions as event handlers. In general, this document uses the Flash 6 style throughout.

Text field font rendering

Flash has two ways of rendering fonts:

 

·       By rendering the text as system fonts. Although this means the final SWF is smaller and the text is rendered more efficiently (i.e. faster), it does have one side effect; flash does not treat them the same as other graphics. In particular, the text fonts cannot be transformed in any way by altering certain properties (such as alpha and rotation), and is not anti-aliased.

·       By using the font outlines. When you do this, Flash has to save the point data used to make up the font shapes. This means that your final SWF will be slightly larger, but has the advantage that the font is treated as any other vector graphic. You can now apply property based changes, and the font will be anti-aliased.

 

NB – You will not see the effects of this until you test the movie or view it in a browser.

 

To save outline information, you need to select the text field and hit the Character… button on the property inspector. This will bring up the Character Options window:

 

 

This will be set to no characters by default. If you intend to apply any property-based animation to the text field (except position movement via the _x and _y properties), you must embed all the font outlines that you expect to appear in the text field. Failure to embed any outlines means that Flash stops rendering the text field as a graphic, and your property-based animations will fail (with no error message to say why).

 

If you only embed a subset of the full font, only those characters will appear in the final text field. For example, if you embed lower case letters only and then try to display "Friends Of Ed", you will see "reinds f d".

 

To embed your entire font, check All Characters , and to embed selected characters, check Only and then select the subset you want to embed. Embedding a large number of fonts, or embedding a complex font can drastically increase the file size of your fonts. Using complex fonts can also affect performance, because such fonts will take a long time to draw and animate.

 

You are recommended to use simple fonts such as Arial or Trebuchet when embedding fonts, and to only embed those characters that will be actually used.

 

To see this in action, create a new dynamic text field and give it an instance name my_txt . Depending on whether or not you embed the fonts or not, you will see the rendered text (once you test the movie ) to be either not anti-aliased (no outlines exported) or anti-aliased (outlines exported):

 

 

 

You will also see that the following code will only work if you have exported outlines:

 

my_txt._alpha = 50;

my_txt._rotation = 45;

 

and that if you don't save the outlines, and then test your movie, not only do you not see the text fade and rotate, it disappears altogether and no error message why is given. This is a feature of Flash that you should not lose sight off; its one of those head-scratching "what's wrong with this darn thing?" moments that usually comes to light half an hour before the client is due to view the site in person (we've all been there, done that!).

Forcing Flash to embed fonts

As mentioned above, Flash treats fonts that have their outlines embedded differently to those that do not. Another thing that Flash does that is not documented is that it will never use the fonts installed on the users machine for all text fields where you specify "embed font outlines". You can retrieve a list of all fonts installed on a users machine via TextField.getFontList but you cannot make Flash use them in a text field (via TextFormat.font ) unless you specify "No Characters"in the Character Options for the text field.

 

When you embed fonts though, you get better quality font rendering. Where you want to retain the ability to swap between a number of fonts whilst retaining the ability display better quality fonts, you have to embed all the font outlines you will be using in at least one other text field per font. A good trick is to embed all the font outlines you will be using dynamically (but which Flash may think it does not need to embed at SWF generation because they do not yet appear on any timeline), in text fields that are off screen.

 

Because this is a text formatting issue, we have included a fuller discussion of this in the entry for TextFormat . This includes short examples to show you the issues. Getting the ability to change fonts when you also embed fonts right is initially confusing until you see a working example (which is what this book is all about!).

Flash text rendering performance

One of the most computationally intensive things you can make Flash do is constantly update a text field every frame. For example, simply updating a text field that shows the players score during a Flash video game can eat up to half of your available performance if that text field has complex font outlines embedded and is being updated at 18 fps (a normal frame rate for a game). Consider deriving alternative ways of updating text fields that avoid this, such as:

 

·       Updating your text fields on change rather than every frame.

·       Updating your text field at a slower interval, such as every second rather than at the frame rate (you could use the Flash 6 setInterval event to do this).

Creating text fields dynamically.

Those of you looking for a way to create text fields dynamically may be sorely disappointed to find that there is no method to create them in the TextField object. The method to do this is actually in the movie clip object, and is MovieClip.createTextField . We don't know why either; we only document the rules not define them!
