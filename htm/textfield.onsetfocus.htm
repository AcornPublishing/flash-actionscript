TextField.onSetFocus

myTextField .onSetFocus(oldFocus)

 

Where myTextField is a TextField instance and oldFocus is the object that lost focus

 

Compatible with Flash Player 6 and later .

 

The TextField.onSetFocus event handler is invoked when a text field gets keyboard input focus. The object that loses the focus is specified inside the event handler function.

Description

The TextField.onSetFocus event handler is invoked when a text field is given input focus. The function associated with the TextField.onSetFocus event handler is launched when this happens and that function specifies a new object to receive the focus.

Examples and practical uses

The TextField.onSetFocus event handler is invoked when a text field gains keyboard input focus from another object that had focus. The function defined as part of the event handler definition is launched on receipt of data and contains the new object to receive focus. To define the TextField.onSetFocus event handler, the function is written like this:

 

myTextField .onSetFocus = function () {

  // code for the focus change

}

 

The TextField.onSetFocus event handler function will be sent an argument that equals the object that has just lost focus (i.e. the text field / movie clip / button that was selected before the current text field). If there was no last selected object (which would occur if the current text field was the first thing selected, or the last thing selected was a button, and this was done via a click rather than using the Tab key

 

NB – A button or clip that is selected by a click does not gain input focus; it can only gain input focus by being selected via the TAB key. If you exclude all except text fields from the tab order, the oldFocus will always be a text field, and this can sometimes make life much easier when creating the onSetFocus event handler. See also the Tips and precautions section.

 

To use this argument you would use the following syntax:

 

myTextField .onSetFocus = function (oldFocus) {

  // code for the focus change

}

 

Within the event handler you can refer to the current text field with the this keyword and the object that last had focus with oldfocus (or whatever you call your argument). The following code would trace the name of the currently selected text field and last selected object:

 

myTextField .onSetFocus = function (oldFocus) {

message = "last selected text field: "+oldfocus;

trace(message);

message = "currently selected text field: "+this+"\n";

trace(message);

}

Tips and precautions

When using text fields, you may want to alter the tab order (it is set to the order in which you placed the text fields, movie clips and buttons onto the stage by default). You can do this via the MovieClip.tabIndex , Button.tabIndex and TextField.tabIndex methods. You may also want to prevent movie clips and certain buttons and text fields from being in the tab order. To do this, see the entries for MovieClip.tabEnabled , Button.tabEnabled and TextField.tabEnabled .
