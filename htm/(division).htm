/ Division

a = b/c;

 

Where b and c are expressions that can be resolved to numbers. a will be equated to a number that is the result of the division of b into c. Anything that cannot be resolved to a number (such as a string) will be undefined, and the expression will be assumed to be zero.

 

Compatible with Flash 4 and above. There are no known issues with this operator.

Description

This operator will divide two numbers.

Strings and Booleans will be converted to a number if possible. Boolean true /false will be converted to 1 /0 and strings will be converted to numbers as per the Number() function. Note that an "e " in a string will be taken to be the decimal point in the exponential representation of a number, so "6e1" = 6x101 = 60.

Any number divided by zero will return zero.

 

Examples and practical uses

/ is used as per the normal arithmetic division operator.

Tips and precautions

The value of anything divided by zero is undefined mathematically. Theoretical maths defines 1/0 as infinity, but since that’s also undefined, it doesn’t really make a difference to coders. The bottom line is that any number divided by zero will return zero, and technically, this is the wrong answer, but since the correct answer (infinity) is unusable by computers because it is a "symbolic" value rather than a real number (where symbolicis mathematical speak for "we don’t know what it is, but its useful to give it a symbolic value because that allows us to cancel it out in our equations via infinity/infinity = 1 and infinity*0 = 0"). In most cases, a divide by zero result is caused by either incorrect data or a logic error in your code, and the zero is just the error propagating through your code.
