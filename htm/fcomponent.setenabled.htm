FComponent.setEnabled

(This method is applicable to FCheckBox.setEnabled, FPushButton.setEnabled, FRadioButton.setEnabled, FScrollBar.setEnabled and FListBox.setEnabled. )

 

myComponent.setEnabled(Boolean);

 

Where myComponent is a component instance or radio button group. When Boolean is set to true, the component is enabled and will respond to user interaction. When it is set to false, the component will be displayed grayed out and will not respond to user inputs.

 

This method is compatible with Flash 6.

Description

This method allows you to enable or disable components. A disabled component will not respond to user interaction, but will respond to all relevant methods that do not rely on user interaction to work.

Notes

 

This shows the effect of disabling one radio button in a group. The remaining buttons are still active.

 

The following code disables a radio button:

 

one.setEnabled(false);

 

Where one is the instance name of the top button in the group.

 

 

This shows the effect of disabling a radio button group. All buttons in the group become disabled.

 

The following code disables the group:

 

group.setEnabled(false);

 

Where group is the group name that all three buttons are members of.

 

 

This shows the effect of disabling a list box. The list becomes grayed out and its scrollbar becomes disabled.

 

The following code disables the list box:

 

myList.setEnabled(false);

 

Where myList is the name of the list box.

 

Examples and practical uses

The ability to disable components allows you to:

 

·       Stop user interaction before you are ready for it. For example, you may not want to enable user interaction with your user interface until all the content has loaded in.

·       Create context sensitive controls. By disabling controls that are not relevant to the current user task, you can tailor the UI to the current user task and avoid user confusion.

·       Stop user interaction when the user has finished. Disabling further user interaction once the user has entered their data allows you to prevent any further interaction, particularly if it is possible for the current user to leave the screen after entering their data (allowing someone else to change the data) or if there may be a download pause whilst the next set of controls appear.

Tips and precautions

Components always load in before frame 1 of a SWF (unless you change the linkage properties of a component whilst it is in the Library (something that is not recommended). This means that all components will be ready as soon as the _root timeline starts, but this doesn’t mean that any other content that your page may include will also be ready. Disabling all components until all your content and relevant keyframes are loaded in completely is a wise precaution to take. If you are designing content that may be accessed by low end modems as well as broadband connections, you don’t want your wonderful broadband e-commerce front end compromised by a customer who finds that they can order stuff without paying because on a 14.4k modem it is possible to process an order before the order validation functions are loaded in!
