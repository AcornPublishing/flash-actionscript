MovieClip.focusEnabled

myMovieClip.focusEnabled

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip.focusEnabled property indicates whether a movie clip can have input focus.

Description

A movie clip that has input focus will be able to see MovieClip.onKeyDown and MovieClip.onKeyUp events.

 

The MovieClip.focusEnabled property returns a Boolean that indicates whether a movie clip can receive input focus or not. If a movie clip has this property set to true, it can receive input focus even if the clip is not a ‘button movie clip’ (a movie clip that has button events). The MovieClip.focusEnabled property sets the ability to receive focus.

 

 

Code

 

 

Notes

 

 

myFocusEnabled = myMovieClip.focusEnabled;

 

 

 

 

 

 

myMovieClip.focusEnabled = false;

 

Stores the current setting of the movie clip focus enable status to the variable myEnabled.

 

myFocusEnabled must be a Boolean.

 

Prevents the movie clip from receiving input focus.

 

Examples and practical uses

By default, a symbol has input focus if:

 

·       It is a selected text field.

·       It is a button or movie clip that is included in the Tab order, and is currently the selected item in the Tab order. The movie clip/button will have a visible rectangle around it (which is the same size as the symbols bounding box).

 

It is important to realize that a movie clip or does not gain input focus if you simply click on it, a fact that makes the MovieClip.onKeyDown and MovieClip.onKeyUp events not normally very useful in any case.

 

The MovieClip.focusEnabled method applies to movie clips that are not ‘movie clip buttons’ (any movie clip that has no button events attached to it). Normally such movie clips will not respond to any movie clip events, but you can allow them to respond to MovieClip.onKeyUp and MovieClip.onKeyDown if you set MovieClip.focusEnabled to true.

Tips and precautions

Input focus is very difficult to maintain for anything other than a text field, and you are advised not to rely on it for keyboard inputs. Getting the user to specifically use the Tab key to give a movie clip is hard enough, but they only have to click on something else for the focus to be lost. Focus has also been seen to be lost simply by the user moving the mouse!

 

Input focus is difficult to reproduce in the Flash environment. You are advised to test for input focus in a browser.
