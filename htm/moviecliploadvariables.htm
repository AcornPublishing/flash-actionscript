MovieClip.loadVariables

myMovieClip.loadVariables (URL [, method])

 

Where myMovieClip is a movie clip instance, URL is the absolute or relative URL where the variables are located, and GET/POST indicates the method used to send the variables.

 

Compatible with Flash 5 and later.

 

The MovieClip.loadVariables() method is used to load variables into a movie from an external file.

Description

The MovieClip.loadVariables() action allows you to transfer variables and their values to a movie or movie clip already loaded. The variables can come from a file (either static or created by a server-side script). The MovieClip.loadVariables() action can be used to update existing values as a movie plays. The file to be read must be maintained in application/x-www-urlform-encoded MIME format.

 

The MovieClip.loadVariables() action requires a URL for the location of the variable file. This file and the movie itself must be in the same subdomain.

 

The optional GET and POST methods indicate how the variables and their values will be sent to the movie. The GET method uses a URL descriptor while the POST method uses an HTTP header. If there are a lot of variables to be sent, POST is the better method.

 

The MovieClip.loadVariables() action is similar in purpose to the method loadVariables(), but the syntax is slightly different.

Examples and practical uses

Once a movie is loaded (using a loadMovie() method, for example), variables can be passed to the movie using the MovieClip.loadVariables() method. The external file from which to read variables may have been created by a CGI script or some other method. The only requirement is that the file is in the standard application/x-www-urlform-encoded MIME format:

 

varname=varvalue&…

 

Where varname if the variable name and varvalue is the value assigned. Each variable name and value pair is separated by an ampersand. For example, a data file may look like this:

 

firstname=Tim&lastname=Parker&city=Ottawa&country=Canada

 

This data is transferred to a movie using the MovieClip.loadVariables() action referencing the file’s name. The easiest way to use MovieClip.loadVariables() is to specify the file name the data resides in:

 

myMovieClip.loadVariables(“timdata.txt”);

 

This can be placed inside an event manager, such as a button click to send the values to the movie:

 

on(release){

 loadVariables(“timdata.txt”);

}

Tips and precautions

All movies loaded into level 0 set the properties for the higher levels (including frame rate, size, and background color). The MovieClip.loadVariables() action does not remove any properties set in level 0 but may override existing values.
