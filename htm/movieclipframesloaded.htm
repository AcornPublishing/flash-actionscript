MovieClip._framesloaded

myMovieClip._framesloaded

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 4 and later.

 

The MovieClip._framesloadedproperty returns a number indicating how many frames have been loaded in a streaming movie

Description

The MovieClip._framesloaded property is used to determine the number of frames that have been loaded in a streaming movie. The MovieClip._framesloaded property can be checked at intervals to allow you to gauge the progress of a clip’s loading process.

 

You can compare this with the MovieClip._totalframes method to see what fraction of the movie clip has loaded in:

 

loaded = _100*(root.framesloaded/_root.totalframes);

 

… will give you the percentage of the SWF loaded (in terms of frames loaded).

Examples and practical uses

This method can be used to see if a certain amount of a movie clip has loaded in yet:

 

_root.onEnterFrame = function() {

    // do not start the timeline until at least 30% of

    // the frames have loaded in...

    if ((this._framesloaded/this._totalframes)>0.3) {

        // now that at least 30% of _root has loaded in,

        // start playing _root and remove this script...

        this.play();

        this.onEnterFrame = undefined;

    }

};

_root.stop();

 

This script will stop the _root timeline until at least 30% of the frames on _root have loaded in.

 

You can also change the script so that is checks for a certain frame. Changing line 4 of the above script to:

 

if (this._framesloaded>5) {

 

… will halt the _root timeline until frame 5 has loaded in. Jumping to a frame that has not loaded yet is a common early mistake that beginners make, and one of the fixes that even professionals use is to simply wait until the full site has loaded up. That ignores the potential for streaming content, particularly now that Flash has video capabilities.

 

The trick with streaming content (such as video) is to preload enough of the content (‘the buffer’) so that you can begin playing your SWF before it is fully loaded in, knowing that you have enough content buffered to ensure the timeline will not stutter (because it is waiting for content to load up) until your SWF is fully loaded.

Tips and precautions

The discussion above assumes that all your frames contain roughly the same file size of content, but this is not always the case. Two things can break the code above:

 

·       Most of your site is loaded on a single frame (or a small number of them). This means that _framesloaded/_totalframes will give a misleading value, and this value may appear to ‘stick’ at frames with large amounts of content.

·       Your main timeline has only one frame. In this case, the code example above will not run until the entire site has loaded.

 

The second problem cannot be fixed easily (you need to restructure your site for streaming), but if you have the first problem, you can use the MovieClip.getBytesLoaded and MovieClip.getBytesTotal methods to give you actual file sizes rather than base the preloading progress on frames loaded
