Color.setRGB

myColor.setRGB(a);

 

Where myColor is a color object, and a is the color you want to use.

 

This method is compatible with Flash 5 or later. In Flash 5, the Color object can be applied to movie clips only. In Flash 6, it can be applied to movie clips, buttons, or dynamic/input text fields. See the entry for Color to see how you can work around this via a wrapper.

Description

This method allows you to color a movie clip, button, or text field. See the entry for Color for a fuller discussion on the terms color, mix, filter,and tint, which is the full set of available color transforms using the color object.

 

The argument for this method is best expressed in hexadecimal. See also the entry for Color for a fuller discussion and links that explain the use of hexadecimal with color representation.

 

You must construct a color object before using this (or any other) method. See also the entry for Color constructor. The examples below also show you haw to create a color object in the first line of each example listing.

 

 

Code

 

 

Original clip

 

Clip after transformation

 

 

my_color = new Color(my_mc);

my_color.setRGB(0x000000);

 

The original clip is a filled gray circle.

 

The transformed clip is a filled black circle because 0x000000 = black.

 

 

my_color = new Color(my_mc);

my_color.setRGB(0xAAAAAA);

 

The original clip is multi colored

 

The transformed clip is a mid gray, as defined by the RGB color value 0xAAAAAA.

 

Note that the original color information is lost.

 

 

my_color = new Color(my_mc);

my_color.setRGB(0xFF0000);

 

The original clip has a color gradient applied to it.

 

The transformed clip has lost the color gradient, and is now a solid color.

 

 

my_color = new Color(my_mc);

my_color.setRGB(0xFF0000);

 

The original clip has a color gradient applied to it that uses alpha (the white area is "black with zero alpha".

 

The transformed clip is still solid, but retains the alpha. This allows the gradient to retain its appearance.

Although using alpha can give more pleasing effects with setRGB, you are better off using mix or tint effects via Color.setTransform because this allows more control.

 

Examples and practical uses

The setRGB method is useful for simple color changes, where you want to change a clip, button or text field that is one solid color, into another solid color. See the example color.setrgb.fla and color.setrgb.swf . This changes a buttons color appearance per button state via scripting.

Tips and precautions

As noted in the examples, you should be aware that the setColor method will result in the loss of any color information you have on the original clip/button/text-field. In particular you will lose:

 

·       Any multi-coloring on the original, which will become a single solid color.

·       Any color gradients.

 

Once you have colored something using setCol, it is impossible to "un-color" it again, so although the separate color areas are still in there for multi colored objects, it is impossible to get them back.

 

The alpha information on the original is however retained.

 

The setColor is actually a simplified Color.getTransform. The two scripts below are equivalent, and both will color your clip with the solid color 0xRRGGBB :

 

my_color = new Color(my_mc);

my_color.setRGB(0xRRGGBB);

 

 

my_color = new Color(my_mc);

myColorTrans = {ra:100, rb:0xRR, ga:100, gb:0xGG, ba:100, bb:0xBB, aa:100, ab:0};

my_color.setTransform(myColorTrans);

 

Of course, setTransform has many more controls that are not being used here, and it is these controls that allow a greater level of fine-tuning than the rather unsubtle setColor effect.
