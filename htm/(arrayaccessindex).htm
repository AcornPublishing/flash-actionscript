[ ] Array access index

a = [];

b = [c, d, e];

b[f] = g;

h = [ [ i0, i1, i2 ], [ j0, j1, j2 ] ];

 

Where a is a new empty array. c, d, e, and g are expressions that can be evaluated to a string, Boolean or number, and f is an integer.

 

b is a new single dimensional array with elements:

b[0] = c,

b[1] = d,

b[2] = e

b[f] = g.

 

h is a multi dimensional array with 2 3 dimensional elements as follows:

h[0] = [ i0, i1, i2 ]

h[i] = [ j0, j1, j2 ]

 

Compatible with Flash 5 or later. Arrays are not typed by array but by element, so that you can mix different types within the same array.

 

See also section 2.7.

Description

Arrays allow you to store related multi-element data in an object that is structured for easy storage and retrieval via an index based system. The array access index operator allows you to access, set or define particular elements of the array. This entry only discusses the array access index. To find out more information on the array object itself, see the entry for Array.

 

 

Code

 

 

Notes

 

myArray = [];

 

myArray = new Array();

 

myArray = [true, "ball", 3.14];

 

 

Initializing an array object. The first two lines shown create an empty array object called myArray. You can use either of these two syntaxes to create an empty array. The third line creates and populates the array. myArray is created with the following elements:

myArray[0] = true

myArray[1] = "ball"

myArray[2] = 3.14

 

 

myArray = [2, 4, 6, 7];

myArray[4] = myArray[0]+myArray[2];

 

 

 

Using the Array access index. myArray is defined in line 1, and consists of 4 elements myArray[0] to myArray[3]. Line 2 creates a new element, myArray[4] with value 2+6 = 8.

 

 

myArray = [[23, 45], [0, 54]];

myArray[0][1] = 30;

 

Creating and using complex array indexes. myArray is defined as two embedded arrays, each with two elements. This index system can be used to represent 2 dimensional data.

 

The second line accesses the element [0][1], which means look in entry zero (i.e. [23, 45]) element 1 (i.e. 45) and equate it to 30. The array structure following this code is as follows:

 

 

 

myArray = [23, 45, 56];

myArray[1.1] = 78;

x = myArray[1.1];

y = myArray[1.2];

 

 

This code will create an array element myArray[1.1] = 78. y will be undefined because there is no element myArray[1.2]. Although using non-integer arrays is allowed, it is not normally recommended.

 

Examples and practical uses

The uses of arrays are described as follows.

Programming

Arrays are traditionally used with looping structures, because loops can quickly go through every element of an array by incrementing an index value. The following code will print every element in array myArray into the output window:

 

myArray = [1.234, 2.345, 3.456, 4.567, "Flash", true, false];

for (i=0; i<myArray.length; i++) {

   message = "element "+i+" is "+myArray[i];

   trace(message);

}

 

See also Array.length to see how this code knows the number of elements in the array.

 

Arrays are useful in defining look-up tables. Computers can only work in numbers, and cannot calculate things like "What is two months on from March?". By converting the months into numbers via a look-up, we can write code that can answer this question:

 

months = ["january", "february", "march", "april", "may", "june", "july", "august", "september", "october", "november", "december"];

startMonth = "march";

// look-up "january" - "december" to convert them into numbers...

for (i=0; i<12; i++) {

   if (months[i] == startMonth) {

      startMonthAsNumber = i;

      break;

   }

}

// now find what two months on from march is...

endMonth = startMonthAsNumber+2;

trace(months[endMonth]);

Motion Graphics

The use of Array indexes that form two and three-dimensional arrays is a very efficient way of storing the data for 2D and 3D drawing engines. See the example (arrayaccessindex).fla and (arrayaccessindex).swf .

Tips and precautions

The use of non-integer indexes is allowed in Flash (as shown by the examples in the description section) but all the methods of the Array object assume integer indexes. You will therefore get odd results if you mix array object methods with arrays that contain non-integer indexes.

 

Creating large multidimensional arrays will quickly eat up memory, so be careful when creating large 3 dimensional arrays.
