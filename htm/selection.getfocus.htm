Selection.getFocus

Selection.getFocus()

 

This method is compatible with Flash 6 and later.

 

The Selection.getFocus() method returns the variable name of the field that has focus.

Description

The Selection.getFocus() method returns variable name assigned to the text field that has focus. If no text field has focus  null is returned. The Selection.getFocus() method returns the target path as a string if a button or text field instance has focus.

 

 

Code

 

 

Additional explanation

 

Notes

 

curr = Selection.getFocus();

 

Stores the variable name of the field with focus in curr.

 

 

If no field has focus a null is returned.

 

Examples and practical uses

The Selection.getFocus() method is used to return the variable name of the text field that has input focus. If there is no text field with input focus, a null is returned by the Selection.getFocus() method. The variable name is returned as a string.

 

If the object with input focus is a TextField object instance, the Selection.getFocus() method returns the path to the instance as a string. Similarly, if the object with input focus is a button movie clip, Selection.getFocus() returns the path to the button movie clip.

 

To use the Selection.getFocus() method you can assign the result to a variable:

 

curr = Selection.getFocus();

 

You can see the use of the Selection.getFocus() method in the code below, which checks to see if you have entered a Control+Q sequence and either a button or text field has input focus. If focus exists, the identity of the field is returned as a string:

 

if ( Key.isDown(Key.CONTROL) and Key.isDown(81) ) {

           if ( Selection.getFocus() != null) {

               myField = Selection.getFocus();

           }

   }

 

In this case, when the frame is entered and both the Control and Q keys are pressed (detected with the Key.isDown() method), the condition checking for focus is executed. If focus does exist, the name of the field is stored in myField. The check for focus is to prevent null being assigned to myField (you could as easily have left this check off the code here, and check for null later when using the contents of myField ). The contents of myField will include the level, so if the field is textfield1 the contents of myField will be _level0.textfield1. This type of approach of checking for specific key presses and input focus is handy for several purposes. See the Selection.setFocus() method for more examples.

 

The same approach applies to buttons. For example, to capture the identity of a button that currently has focus (and before the user moves the cursor elsewhere and loses the focus), you can use the code:

 

on (rollover) {

 myButton = Selection.getFocus();

}

 

… which will record the button name in the variable myButton.

Tips and precautions

The Selection.getFocus() method is used to determine the variable name or path of an object that currently has input focus. To convert the variable name into the value contained therein, use the eval() method. You should always check to make sure that a field has focus before using any of the Selection methods on that field. The easiest way is to check for a null value (since a field without focus returns null to the Selection.getFocus() method):

 

if ( Selection.getFocus() == null) {

   trace(“No field has focus.”);

}
