-- Decrement (post/pre decrement)

a--;

b = --a;

b = a--;

 

Where a is a variable, array element, or property that holds a number variable. b will be returned as a number value. If a is not a number the result is NaN ("Not a Number", see NaN entry).

 

Compatible with Flash 4 and above. It is can be used as a shorthand way of decrementing counter variables, and it is faster (in terms of code performance) than if you used the longer versions of the same code.

 

Related actions include for... loops (where -- is often used to decrement the loop counter) and ++ (which is similar to --, except that it increments).

Examples and practical uses

The decrement operator is often used as a shorthand form of the code sections noted in the table above.

 

You will often need to continually subtract 1 from a numeric value to create counters or loop variables, and -- is useful in doing this. It can also be used in motion graphics to reduce a property (such as _alpha or _width ) to make a movie clip fade or grow smaller with time.

 

Motion Graphics

The following event code will move a movie clip myClip from screen right to screen left (i.e. the direction of reducing x ) by using -- to subtract 1 from the _x property of myClip every frame. The motion is stopped by removing the event as soon as _x is less than 40.

 

See the example (decrement).fla and (decrement).swf .

 

myClip.onEnterFrame = function() {

   this._x--;

   if (this._x<40) {

      this.onEnterFrame = undefined;

   }

};

Looping structures

Refer to the entry for the for... loop. You will see here how the decrement (and increment) operator is used to update the loop counter per loop iteration. This is the most common use of the decrement (and increment) operator.

Tips and precautions

There is very little final performance gain in using the short hand version a-- ; over a = a – 1 ;, so it should be seen purely as a scripting feature, not an optimization of the code itself.

 

Note that using the pre-decrement will increment a first then assign its value to b, whereas the post decrement will assign a to b, and then decrement a. The difference really only becomes important when assigning a to b. If you are merely incrementing a without assigning the value, then the two are quite interchangeable.

 

ActionScript is an interpreted language, which means that each line of code is converted to executable code every time it is encountered (rather than a compiled language, where the code is all converted to executable code once). The time to convert code is partly based on the time to read each line of code, and because the short hand versions are shorter, they will run more quickly than the longer versions (usually about 10% or so faster). The short hand versions should therefore be used in critical code loops where performance is important.

 

Note that using -- with a numeric literal will raise a syntax error. The following is not allowed for example:

 

x = 2--;

 

Also, note that subtracting 1 from certain properties will give unexpected results because of the nature of the property. For an example of this, change the code in the motion graphics example above to

 

myClip.onEnterFrame = function() {

   this._alpha--;

   trace(this._alpha);

   if (this._alpha<0) {

      this.onEnterFrame = undefined;

   }

};

 

The first few iterations will give you:

 

Actual                 Expected

98.828125           99.000

97.65625             98.000

96.484375           97.000

95.3125               96.000

94.140625           95.000

92.96875             94.000

91.796875           93.000

 

This occurs because alpha goes between 0 and 100 in 256 discrete steps (alpha goes from 255 to 0 internal to the Flash player), and this property cannot exist in 'in-between' values such as 99.000. This means that you cannot assume that it will take 100 decrement subtractions to get from 100% to 0% for some properties. A better way would be to set the final target value (in this case 0% alpha) explicitly;

 

myClip.onEnterFrame = function() {

   this._alpha--;

   if (this._alpha<0) {

      this.onEnterFrame = undefined;

      this._alpha = 0;

   }

};

 

Finally, any expression that resolves to the following will give a syntax error:

 

b= (a--)--;
