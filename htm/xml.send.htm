XML.send

myXML.send (a, b)

 

Where myXML is an instance of the XML object, a is the location (URL) to which the data is sent (generally a server script or middleware application), and b is the browser window or frame in which the returned data is displayed.

 

Compatible with Flash Player 5 and later. This method is used to send XML data to a location, but it is not returned to the Flash movie. Refer to related entries XML.sendAndLoad, XML.loaded, XML.getBytesLoaded, XML.getBytesTotal, and XML.onLoad.

Description

The XML.send() method encodes the specified XML object into an XML document and sends the instance to a specific location specified in the arguments. All documents are sent using HTTP transfer protocols. If a second parameter is specified, it refers to the window that will contain any data returned from the server. The default value for this window is _self.

Examples and practical uses

The XML.send() method lets you encode an XML object as an XML document and send it to a server. Usually, it is sent to a server-side script that can handle XML data. At this location, further manipulation, and processing can occur. You might want to take user-input information, and then store it in a database for later retrieval. You can output the data in an HTML format: either in a pop-up window, or even in a frame on the same window as your Flash movie.

 

In the following code block, a URL is sent as an XML packet to ColdFusion. ColdFusion parses the XML information and then sends it to a blank window.

 

myXML_xml = new XML("<location url='http://www.macromedia.com/desdev/resources/macromedia_resources.xml'>");

myXML_xml.send("redirect.cfm", "_blank");

myXML_xml.ignoreWhite = true;

trace(myXML_xml.toString());

 

You will need ColdFusion MX installed on your server or local Web server in order to use this code. Refer to the example xml.send.fla and xml.send.swf, which uses redirect.cfm, to illustrate this method.

Tips and precautions

There are several different windows, which can be targeted:

 

·       _blank – a new window

·       _parent – the parent window

·       _self – the current window

·       _top – the top-level frame in the current window

 

If a target parameter is not supplied, the _self setting is the default window that is used.

 

The XML.send() method allows you to send XML documents to a file. It is the opposite of the XML.load() method. The URL must be in the same subdomain as the movie, for security reasons. Refer to the Macromedia security white paper from the following location for more information: www.macromedia.com/software/flash/whitepapers

 

If you require data to be sent back to the Flash movie, look into XML.sendAndLoad. If you instead need your information sent and received in small amounts quickly, look into the XMLSocket object and socket servers. Remember that this method must be attached to a top level XML class node.
