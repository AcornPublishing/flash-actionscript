XMLSocket.onData

myXMLSocket.onData()

 

Where myXMLSocket is an instance of the XMLSocket object.

 

Compatible with the Flash 5 Player 5 and above. The XMLSocket.onData() event handler is invoked when an XML message has been downloaded from a server. Refer to the closely related entry on XMLSocket.onXML().

Description

The XMLSocket.onData() event handler is a "callback" function that is invoked when a message has been downloaded. Each data transmission is terminated with a zero byte, and this is what causes the onData handler to execute.

thesocket = new XMLSocket();

thesocket.connect("localhost", 12345);

thesocket.send("<myname>jepo</myname>");

thesocket.onData = function(thedata) {

   trace("data arrived");

   trace (thedata);

};

 

 

 

When data arrives to the movie, the content is traced, as well as an alert that it has arrived.

 

This data is not parsed by Flash. To parse this information, you would enter functions within the inline function.

 

Examples and practical uses

The XMLSocket.onData() event handler is used as a "callback" function whenever a server downloads data from the server. A properly formatted data transmission has a zero byte at the end. The XMLSocket.onData() handler reads a stream from the server, and whenever a zero byte is received it parses everything between that zero byte and the last zero byte. This is then passed to the XMLSocket.onXML() handler, by default, for processing. If you override this with a custom "callback", you can intercept the data and it is not sent to the onXML() handler for processing. Therefore, you can use, parse and/or process the raw data yourself. This could mean better parsing times, depending on the functions you pass.

 

We have overridden the onData in the above example, thereby avoiding the onXML handler and any parsing performed by the Flash player. You could perform your processing within the inline function if you want to parse the data manually. Refer to the example xmlsocket.ondata.fla and xmlsocket.ondata.swf for more information on this handler.
