MovieClip.clear

myMovieClip.clear()

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip.clear() method removes all graphics within a movie clip.

Description

The MovieClip.clear() method removes any and all drawing commands associated with the movie clip instance calling the method. Any established settings for the current line style are also cleared.

myClip.clear()

 

 

Clears all graphics that were created in the movie clip myClipvia the drawing API methods.

 

Examples and practical uses

my_mc.clear() erases all graphics in the movie clip my_mc.

 

You need to clear all drawing API graphics created before redrawing them to create animation. The example file movieclipclear.fla / movieclipclear.swf shows this. It creates two animated morphing triangles with the following core code:

 

 

 

    clip.drawshape = function() {

        this.clear();

        this.lineStyle(0, black, 100);

        this.beginFill(orange, 100);

        this.moveTo(this.x[0], this.y[0]);

        this.lineTo(this.x[1], this.y[1]);

        this.lineTo(this.x[2], this.y[2]);

        this.endFill();

    };

 

Notice that we have to repeat the lineStyle definition following every clear(), because the clear resets it.

 

If you delete the line this.clear(), you will end up with this;

 

 

 

Although equally pretty, you will notice that the effect slows down very quickly.

 

Flash will slow down considerably if you make it draw more that a couple of hundred unique shapes, so you should formulate some method of deleting all old clips over a certain age (or simply clear them all after a certain number of draws and start again) if you want to create effects like this. One route to doing this is noted in the next section.

Tips and precautions

You can also erase the results of the drawing API simply by deleting the movie clips you drew into. An example of this can be seen in the file (arrayaccessindex).fla / (arrayaccessindex).swf, and this code allows multiple shapes on the stage, but deletes all but the last 20…

 

 

This is not actually using clear at all, but is using depth. It creates each shape (called a petalin the script) in its own movie clip.Creating a clip over a depth that is already occupied deletes the previous occupant, so if we simply keep increment depth but limit between 0 and 20 with modulo:

 

petalCount = (petalCount+1)%20;

 

… we automatically limit the number of shapes to 20, and don’t have to bother with clear() at all.
