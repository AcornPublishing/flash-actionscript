Where a is an initialization, b is a condition, and c is an increment or decrement used to update the value of the Loop Control Variable (LCV), which eventually causes the condition to fail, thus ending the loop.

 

for is compatible with Flash 5 and above. The for loop performs the same actions as the while loop, although can be written in fewer lines of code. You should be aware of a known issue with the for loop: if a parameter is missing from the for loop (for example, if you are missing a, b, or c), Flash MX will become unresponsive or crash. Related actions include the ++ increment and decrement operators, and the for...in loop.

Examples and practical uses

The for loop is a commonly used iterative loop. for loops will execute until a false value is returned or unless a break action is used to stop the loop prematurely. The for loop is used when you need to loop over a known number of items.

Tips and precautions

A variable of i is commonly used by developers (although any legal identifier could be used) as a counter in a for loop. You can also put more than one statement in as long as they are separated by a comma operator. For example, you could write the following code:

 

for (i=0, j=10; i<10, j>=7; i++, j--) {

   trace("This is i: "+i+", and this is j: " +j)

}

 

It is important to make sure your end conditions can be met when constructing a for loop. For example, this for loop would enter an infinite loop and the Flash player will throw an error when executing the code.

 

for (var i=0; i<=5; i--) {

trace(i);

}

 

Since you would be decrementing i from 0, and your value of 5 would never be reached. When the processor reaches its limit, the program will throw an error.
