XML.hasChildNodes

myXMLNode.hasChildNodes()

 

Where myXMLnode is a reference to the node of an XML object instance.

 

This property is compatible the with Flash 5 Player and above. The XML.hasChildNodes property is used to check whether or not a specified node has any child nodes. Refer to the related entry XML.childNodes.

Description

The XML.hasChildNodes property returns a Boolean that indicates whether or not the specified node has child nodes. If it does, then true is returned. If there are no child nodes, false is returned. This is a read-only property.

Returned values

 

 

Notes

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age><fur>black</fur></cat>");

myNode = myXML_xml.firstChild;

trace(myNode.hasChildNodes());

 

 

true

 

This code checks whether or not the firstChild (<cat>) has any child nodes. It does, so true is returned.

 

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age><fur>black</fur></cat>");

myNode2 = myXML_xml.firstChild.childNodes[0].firstChild;

trace(myNode2.hasChildNodes());

 

 

false

 

This code checks whether or not a different node (the text node Whiskers) has any children (sub) nodes. It does not, so false is returned.

 

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age><fur>black</fur></cat>");

myNode = myXML_xml.firstChild;

trace(myNode.childNodes(myXML_xml));

 

 

undefined

 

Incorrect syntax.

 

Examples and practical uses

This property can be useful if you need to figure determine the need to process a particular node. You can ascertain whether the node has child nodes, and if so, then you can process the node. For example, you might structure your code in the following manner:

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age><fur>black</fur></cat>");

myNode = myXML_xml.firstChild;

//trace(myNode.hasChildNodes());

hasChildren = myNode.hasChildNodes();

if (hasChildren){

   _root.myXML.text = myXML_xml.toString()

}else{

   _root.myXML.text = myXML_xml.firstChild.nodeName();

}

 

As you can see, we are using the property as a condition in the lines:

 

hasChildren = myNode.hasChildNodes();

if (hasChildren){

   //do stuff

}

 

Therefore, you can control your XML processing based on what this property returns. This can be particularly useful if you are working with XML of an unknown structure. Refer to the example xml.haschildnodes.fla and xml.haschildnodes.swf for more information on this property.

Tips and precautions

You can also use myNode.childNodes.length > 0 in the same way as you use myNode.hasChildNodes():

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age><fur>black</fur></cat>");

myNode = myXML_xml.firstChild;

trace(myNode.childNodes.length > 0);

 

which of course returns true. Even though we are illustrating this property with known XML data structures, it will be of most use when you are working with newly created structures, or those from another source you are loading into your movie.
