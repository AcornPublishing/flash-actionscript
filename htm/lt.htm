lt

string1 lt string2

 

Where string1 and string2 are two strings.

 

Compatible with Flash Player 4 and later. Deprecated for < in Flash 5 and later.

 

Thelt operator is used for string "less than" comparisons. Flash 5 and later emphasizes the use of the more common < comparison operator (which can compare all data types) instead, but lt is still supported.

Description

The lt operator is used for "less than" comparisons between strings only. It can be used instead of the < operator in Flash 5 and higher, although the latter is the preferred format because of consistency with non-string operations. String comparisons are left to right, using ASCII values.

Notes

 

string1 lt string2

 

If string1 is less than string2 in an ASCII comparison from left to right, true is returned.

 

 

Deprecated for < in Flash 5 and later.

Examples and practical uses

When used in a conditional clause lt returns true if the first string is less than the second, based on a character-by-character comparison from left to right. For example, this code:

 

string1 = "ABC";

string2 = "CDE";

if (string1 lt string2) {

   trace ("The first string is less than the second string.");

}

Tips and precautions

lt was implemented in Flash 4, but since Flash 5 has only been maintained for backward compatibility reasons, and should only be used for legacy coding. Also note that le only compares strings, whereas < can compare all data types.
