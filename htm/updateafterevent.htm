updateAfterEvent

updateAfterEvent()

 

Compatible with Flash 5 and later.

 

The updateAfterEvent action forces the Flash Player to redraw the stage. This function is useful in situations when the global onEnterFrame timeline event isn't updated frequently enough. For example, when using the setInterval/clearInterval commands the updateAfterEvent function is very important, as the intervals defined using setInterval may not necessarily coincide with the global onEnterFrame event, causing a visible lag in the Flash file when the frame is re-drawn.

 

Calling the updateAfterEvent command invokes the same process as the global onEnterframeEvent, only it is independent of the movie's frame-per-second rate, which usually controls the frame re-draw. See also clearInterval and setInterval. See also setInterval and clearInterval.

Description

The updateAfterEvent action forces a frame redraw.

Examples and practical uses

The updateAfterEvent action is best used in situations when interactions can occur independent of the fps (frames per second) of a movie, including when a key is pressed, when a certain millisecond in time is reached or even when the mouse moves. See example updateAfterEvent.fla and updateAfterEvent.swf , to show the improvement that using updateAfterEvent provides.

 

The example clip, moves a clip up or down depending on whether the "up" or "down" arrow key was pressed. The movies fps is set to 1, very low, so the movie only gets updated every one second. Using the drop down menu, you can set the movie to use the updateAfterEvent command, which forces Flash to redraw the frame, before the onEnterFrame timeline event redraws the screen.

 

Using the updateAfterEvent is usually a must to take advantage of the powerful clearInterval and setInterval functions.

Tips and precautions

The updateAfterEvent function will be ignored by Flash unless it is called in an onEnterFrame event or within a function associated with an interval.
