XML.nodeType

myXMLNode.nodeType

 

Where myXMLNode is an instance of the XML object.

 

Compatible with Flash Player 5 and later. This property returns an integer of 1 or 3 based on the type of a specified node in an XML object instance. See related entries XML.nodeName and XML.nodeValue.

Description

The XML.nodeType property is used to return the node type of the XML object calling the property. If the object is an XML element the node type is 1. If the object is a text node, the node type is 3.

 

 

Code

 

 

Returned values

 

 

Notes

 

myXML_xml = new XML("<cat/>");

trace(myXML_xml.firstChild.nodeType);

 

 

1

 

Returns a value based on the type of node it is. In this case, <cat> is an element node.

 

 

myXML_xml = new XML("<cat>whiskers</cat>");

trace(myXML_xml.firstChild.childNodes[0].nodeType);

 

 

 

3

  Returns a value based on the type of node it is. In this case, whiskers is a text node.

 

 

myXML_xml = new XML("<cat fur='black'>whiskers</cat>");

trace(myXML_xml.firstChild.attributes.fur.nodeType);

 

 

undefined

 

Cannot return the value of an attribute.

 

 

myXML_xml = new XML("<cat/>");

trace(myXML_xml.firstChild.firstChild.nodeType);

 

 

undefined

 

There is no node at this location.

 

Examples and practical uses

The XML.nodeType property holds a code indicating the type of the XML node. If the property’s object is a text node the node type is 3. If the property’s object is an XML element the node type is 1. There are no other values which are supported by Flash and accessible as objects. However, other values do exist in the DOM, and there are 12 in total.  As you probably already know, an element is within a tag, and a text node is between two tags. In the following XML:

 <mycat>whiskers</mycat>

 The <mycat> tag is a type 1 element node, and whiskers is a type 3 text node. Using this property is helpful when you are processing XML documents of unknown structure or content. For example, you might need to test whether a particular node is an element, so you can extract the data using nodeName (which is used for elements). However, if it returns as a type 3 node, you would instead need to process the data using nodeValue since it is a text node. This is easily worked into an if... else statement. Let's look at an example of how you would extract these values:

myXML_xml = new XML("<cat><catname>whiskers</catname><age>5</age></cat>");catname = myXML_xml.firstChild.childNodes[0]trace("The <" + myXML_xml.firstChild.nodeName + "> node is a type " + myXML_xml.firstChild.nodeType+" node.");trace("The <" + catname.childNodes[0].nodeValue + "> node is a type " + catname.childNodes[0].nodeType + " node.");

 You could cycle through XML and return node types in the following way. Note that there are several different properties in action here:

 function debugNode(node) {   if (node.nodeName != null) {

      trace(node.nodeName+"{"+node.nodeType+"}");

}  

if (node.firstChild.nodeValue != null) {

       trace(node.firstChild.nodeValue);

}

if (node.hasChildNodes()) {

      debugNode(node.firstChild);

}

if (node.nextSibling != null) {

      debugNode(node.nextSibling);

}   trace("---");

}

var myXML = new XML("<cat><kitten>whiskers</kitten><kitten>kid</kitten></cat>");

myXML.ignoreWhite = true;debugNode(myXML.firstChild); 

This code would return the following cat {1}kitten {1}whiskers---kitten{1}kid------------

 Refer to the example xml.nodetype.fla and xml.nodetype.swf to see this code in action.

Tips and precautions

The XML.nodeType property is useful for checking a node’s type, but you will need to use the XML.nodeValue and XML.nodeName properties in order to extract their values.
