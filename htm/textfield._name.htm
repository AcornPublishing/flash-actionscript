TextField._name

myTextField ._name

 

Where myTextField is a TextField instance.

 

Compatible with Flash Player 6 and later .

 

The TextField._name property returns the name of the text field instance.

Description

The TextField._name property simply returns the name of the text field instance. This is often used in generic scripts to store the clip name in a variable.

Examples and practical uses

The TextField._name property returns the name of the text field and allows you to store it in a variable for future use:

 

myName = myTextField ._name;

 

See the example textfield._name.fla and textfield._name.swf .

Tips and precautions

The TextField._name property simply returns a text field instance name, but this is useful for developing versatile scripts that work with any movie. By storing the name in a variable, the script can be used for any movies. In many cases, you can also use the more versatile reference, this , in event handlers, and you do not normally use the _name property explicitly.

 

Unless you are creating text fields dynamically via MovieClip.createTextField , you should enter the instance name for every text field you place on stage. Failure to do this will mean that you will not know the text field instance name, and cannot control it via ActionScript (or read data from/to it).
