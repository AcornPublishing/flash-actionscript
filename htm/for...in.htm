for...in

for (a in b) {

statement

}

 

Where a is a property, and b is an object. The for...in loop will iterate through the properties of the object one time.

 

The for...in loop is compatible with Flash 5 and above. You should be aware that the returned values are not returned in any particular order. You therefore cannot rely on a predictable stack order in the array of returned properties.

Predefined objects are returned using a for...in loop.

 

 

var cat = new Object();

cat.age = 5;

cat.fur = "black";

cat.size = "big";

 

for (var prop in cat) {

trace("The cat is " + cat[prop]);

}

 

The cat is big

The cat is black

The cat is 5

 

The properties of an object are returned. The use of var is permitted.

 

 

for (x in System.ShowSettings) {

trace (x +"=="+ typeof(System.ShowSettings[x]));

}

 

n/a

Do not try to loop over something which is not an object or array. For example, you cannot loop over a function.

 

Examples and practical uses

The for...in loop is used to iterate through the properties of an object. Therefore, it is most useful when you need to determine which properties can be manipulated in your movie. You can also use the for...in loop to view the number of, name and level of movie clips on a timeline.

Programming

You can use the for...in loop to detect the number of movie clips currently on a timeline. The following script is placed on the main timeline, and returns the location and names of all movie clips located on this timeline.

 

var myClip = new Array();

var counter = 0;

for (i in _root) {

   thisItem = _root[i];

   if (typeof (thisItem) == "movieclip") {

      myClip[counter++] = thisItem;

   }

}

 

 

Tips and precautions

You can loop over objects and arrays, but you cannot loop over methods or functions. If you try to loop over methods or functions, you will either get null returned values, or nothing returned at all.

 

Also, the order in which the properties are returned is not always predictable. In the following example, using an alternate method for building an array, we enter the following code into a Flash document:

 

values = [2, 66, 7, 45, 22, 453, 33];

for (i in values){

   trace (values[i]);

}

 

When testing the movie, you will notice values will be returned in a different order, likely starting with 33 (arrays are often returned backwards, although this is not a steadfast rule).
