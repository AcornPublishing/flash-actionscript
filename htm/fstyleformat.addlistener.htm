FStyleFormat.addListener

a.addListener(c [d,e,f,n]);

 

Where a is the name of the custom style format, and c to n are the instance names of the components you are applying the style to.

 

Compatible with Flash MX. This method is used to apply custom style formats to component instances. You must provide at least one component instance to add a listener to. Refer to the related entry on FStyleFormat.removeListener.

Description

This method adds listeners to one or more component instances.

Examples and practical uses

Components need to listen to a style format in order to make changes to the default text or colors. You need at least one component in this method, although you can also have a series of components listening to the same style format. You do not need to use the FStyleFormat constructor to use the addListener method.

Programming

We can add instances of different kinds of components to one style format using the addListener method. Refer to the example fstyleformat.addlistener.fla and fstyleformat.addlistener.swf to see how this is done. A new style format myStyle is created:

 

myStyle = new FStyleFormat({textColor:0xFF0000, textFont:"Arial", textSize:10, textAlign:"center", face:0xFFFFCC, shadow:0xFFCC33, highlight:0xFF9933, highlight3D:0xFFCC99, darkshadow:0xFF6600});

 

Following the style format, we need to apply the changes we have made to the global style format.

 

myStyle.applyChanges();

 

We will make the changes to our movie when the user clicks a "push button" instance named button1. The "click handler" name addL is assigned to the button instance, and the following function is entered into frame 1 of the actions layer on the main timeline:

 

function addL() {

   myStyle.addListener(check1, radio1, combo1, list1, button1);

}

 

When the user clicks button1, the component instances check1,radio1,combo1,list1, and button1 one listen to the custom style format myStyle.

Tips and precautions

There are a few things to remember when using the addListener method. A component can only listen to one property value if it is assigned in more than one style format, and it is simultaneously listening to several. Also, you can add a listener to a radio button group instead of giving individual instance names. For example:

 

myStyle.addListener(radioGroup1); //a group of radio button instances

 

However, if you need to remove the listener, you will have to remove reach instance individually. For further information, refer to the FStyleFormat.removeListener entry.

 

When you are changing a textSize property in the ListBox and ComboBox components, the component will change its dimensions. In a ListBox this can cause problems. Each time a listener is added or removed, the component will get increasingly smaller no matter what size the text is changed to or from. You can fix this to some degree by using the setSize method after setting the textSize property.
