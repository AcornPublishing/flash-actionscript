++ increment

a = b++;  (post increment)

c = ++d;  (pre increment)

 

Where b and d are expressions that can be converted to numbers, a literal number or any data element holding a numeric value.

 

·       In post increment, a is equated to b, and then b in incremented by 1.

·       In pre increment, d is incremented by 1, and then c is equated to the new value of d, thus c and d end up with the same value.

 

 

This operator is compatible with Flash 4 or above. Because incrementing a value by 1 is a common programming requirement, ++ is a shorthand way of writing code that increments number values. The performance gains in doing this are not normally large, so if you are in any doubt, use the long hand versions (shown below) instead. See the entry for for.

Examples and practical uses

The simple a++ form is the most commonly used shorthand. It is used extensively with the for loop. Refer to that entry for typical examples.

Tips and precautions

The most common shorthand form, a++ or ++a results in no performance gain. It should therefore only be seen as a quick way of writing a = a + 1 for one finger typists and impatient programmers rather than a magic way of coaxing a bit of extra performance out of critical loops in your code. In fact, the only ActionScript programmers that tend to use it are JavaScript old timers. If you‘re not one of those, you may want to forget this entry completely and file under ‘bad habit that reduces legibility for no apparent gain.
