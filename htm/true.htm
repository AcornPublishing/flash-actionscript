Examples and practical uses

The true keyword Boolean indicates a positive result. Only Boolean values are allowed for some properties, and these are assigned as either true or false. For example:

 

myTextFormat.italic = true;

 

indicates italics are to be used for a text field.

 

Although not explicitly stated, expressions can evaluate to true. For example, the code:

 

var1 = 5;

var2 = 5;

if ( var1 == var2 ){

    trace("They are equal");

}

 

will perform the comparison in the if... statement and evaluate it to true. This value is not displayed anywhere, but the condition holds the true value for further use.

 

Using an if... statement, you can quite easily determine if two passed objects are of the same type, for example the String object or Number object. The following function will return true if both passed objects are of the same type:

 

a = 10;

b = new Number();

 

c = checkType(a, b); // equals false

 

function checkType(el1, el2) {

   if (typeof el1 == typeof el2) return true;

   return false;

}

 

As you can see, the checkType function relies on a true result to work. If the if... statement doesn't evaluate to true, the function will return a false value.

 

Using the trace() function, you can explicitly return a true or false statement. The following code blocks will all send a true statement to the Output window as the equations will always equal true. Using the trace statement is a particularly good method for debugging problems with if... statements, as it allows you to see exactly what the if... statement will return.

 

a = 2;

b = a;

trace(a == b); returns true, both a and b have a value of 2

 

a = {b: 2};

c = a;

trace(a.b == c.b); // returns true, both b of the a object and b of the c object have a value of 2

 

a = 10;

b = 11;

trace(a != c); // returns true because eleven and ten are not equal

 

Tips and precautions

The true keyword is the opposite of false, which indicates a negative result.
