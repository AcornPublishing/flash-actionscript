Button._y

instanceName._y = a;

b = instanceName._y;

 

Where instanceName is the instance name of a button, and a is the value you want to assign to the buttons _y value. b is a variable that is equated to the current value of the button’s_y property value.

 

This property is compatible with Flash 6 or later. When using previous versions of Flash, buttons are not objects and do not therefore have properties. The Button._y property defines the button y co-ordinate on the stage.

Description

The Button._yproperty defines the y co-ordinate of the button. This property can be used to dynamically position a button on the stage.

Normally however, you want to use the Button._y property to produce much smoother animation than sudden single frame movements. See the Examples and practical uses section.

 

Examples and practical uses

The _y property is more successfully used with the Movie Clip object. Here, MovieClip._y is typically used within a MovieClip.onEnterFrame event handler to form continuous animation. Although the Button object doesn’t have the onEnterFrame event handler (because it doesn’t have a timeline, and so doesn’t have any frames internal to it to cause the "enter new frame" event), you can use the onEnterFrame event of the timeline that the Button is on to provide the animation. The example below would animate the button smoothly:

 

my_btn.onRelease = function() {

   my_btn.startPosY = my_btn._y;

   _root.onEnterFrame = function() {

      my_btn._y += 1;

      if ((my_btn._y-my_btn.startPosY)>39) {

         this.onEnterFrame = undefined;

      }

   };

};

 

 

It uses the enter frame event handler of the timeline the button is on (_root in this case) to move the button by 40 pixels, at a rate of 1 pixel per frame. This method can be extended to provide button transition animations. See the example button._y.fla and button._y.swf .

Tips and precautions

The Button._y property cannot be used easily to produce per frame animation. As seen above, the button object has no onEnterFrame event handler to provide the ability to set up a per frame animation event handler, and you usually have to work around this issue by using the event handler of the nearest MovieClip to provide something to attach the button animation script to. In most cases, this would be seen as bad programming. It is better to use a movie clip set up as a button rather than a true button if you also want to animate the button. To do this, you would do the following:

 

·       Attach an onEnterFrame event to the Movie Clip. See also MovieClip.onEnterFrame.

·       Optionally, designate a separate movie clip to define the button hit area. See also MovieClip.hitArea.

·       Create keyframes in the movie clip timeline that correspond to the button up, over,and down states. You should label these keyframes _up, _over, and _down. See also the entries for _up, _over, and _down.

 

In general, the advanced ActionScript programmer will very quickly realize that the Button object is somewhat superceded by the Movie Clip object in Flash MX, and you will rarely use the Button object with anything more advanced than a simple Button event script. The Movie Clip has a much richer set of properties and timelines.
