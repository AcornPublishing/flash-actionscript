unescape

unescape(string)

 

where string is an URL-encoded string.

 

Compatible with Flash 5 and later.

 

The unescape function is a top-level function used to decode strings which are in an URL-encoded format. An URL-encoded format is used to pass information through a web browser or on the end of an URL. URL-encoded strings are safe to use in a web browser's address bar as certain characters from the ASCII table need to be encoded before they can be used in an URL. All of these special characters in a string are converted to ASCII sequences upon being encoded to the URL-encoded format. The special characters are all non alpha-numeric characters in the ASCII table and some examples of hexadecimal characters are {} and [] and a space.

 

You probably see URL-encoded strings quite regularly while surfing the web, it is common to see them after the name of a HTML page you're visiting along with the ? character. For example this is the URL-encoded URL from a Google search, http://www.google.com/search?hl=en&ie=UTF-8&oe=UTF-8&q=%7BFlash+MX+information%7D&btnG=Google+Search.

Notes

 

unescape ("hello%7B%7D")

 

Returns "hello{}" as an un-encoded URL string

 

The parameter must be a string.

 

Examples and practical uses

The top-level unescape function is primarily used to decode URL-safe strings. Here is a small list of ASCII characters and there URL-encoded equivalent:

 

Character

URL-encoded ASCII sequence

*

%2A

"a space, from the space bar"

%20

{

%7B

[

%5B

*This* string is {URL-encoded}

%2AThis%2A%20string%20is%20%7BURL%2Dencoded%7D

 

The example unescape.fla and unescape.swf , demonstrates using escape and unescape to decode and encode strings ready to be used in URLs and the like which are character sensitive.

Tips and precautions

Be careful not to use the escape function on already URL-encoded string as the string will be URL-encoded a second time, therefore needing to be passed through the unescape function twice to retrieve the initial value. However, "unescaping" an already "unescaped" string has no effect.
