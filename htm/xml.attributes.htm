XML.attributes

myXML.attributes

 

Where myXML is an XML instance, and attributes refers to the attributes of a specified node.

 

This property is compatible with the Flash 5 Player and above. It is used to extract attribute names and values, which can then be used in a Flash movie. You are able to extract either the name, value or both name and value as an array. See related entries on for, for...in, and the XML object.

Description

The XML.attributes collection can provide a complete list, or selected value or name of any attribute attached to an XML instance. The list of attributes can be returned as an associative array.

Examples and practical uses

The XML.attributes collection lets you examine an associative array of the attributes of an XML instance. For example, you can create an attribute for the first child (<person>) element, and output the value using the following code:

 

myXMLObject = new XML("<person/>");

myXMLObject.firstChild.attributes.myname = "Magoo";

trace (myXMLObject.firstChild.attributes.myname);

 

Which displays the output:

 

Magoo

 

This is because we create the mynameattribute with the value of Magoo in the second line of code, using the XML.attributes method. Then, we can also extract (trace) it using the same method. If you want to view the entire XML document, trace it:

 

trace(myXMLObject);

 

As you can see in the Output window, an attribute has been created in the <person> node.

 

<person myname="Magoo" />

 

You can trace an exisiting attribute, and it is done in the same way. In the following code, we trace the myname attribute, which already exists in the myXMLObject instance:

 

myXMLObject = new XML("<person myname='Magoo'/>");

trace (myXMLObject.firstChild.attributes.myname);

 

The output window will also display Magoo. The example xml.attributes.fla and xml.attributes.swf further illustrates this concept.

Tips and precautions

Attributes are advantageous in several ways. First of all, using attributes as opposed to text nodes often speeds up parsing and processing by the Flash player. Secondly, some people find that reading attributes in a structure is more intuitive. By creating a single array with all the properties, you can examine and compare objects easily.
