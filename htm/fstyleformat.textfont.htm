FStyleFormat.textFont

a.textFont = "b";

 

Where a is the name of your custom style format, and b is the name of a system font or the Linkage identifier of a Font embedded into your movie.

 

Compatible with Flash MX. You should be aware that if you use a system font, it will only be used by the component if the computer you are playing your movie on contains the font face. If you use an embedded font, the same text font will be seen by all users, but your movie will have a higher file size.

Description

This property sets the foreground regions of the disabled component instance(s) to a specified RGB value.

myEmbedStyle = new FStyleFormat();

myEmbedStyle.textFont = "myFont";

myEmbedStyle.embedFonts = true;

myEmbedStyle.applyChanges();

myEmbedStyle.addListener (myButton);

 

 

 

myEmbedStyle = new FStyleFormat({textFont:"myFont", embedFonts:true});

myEmbedStyle.applyChanges();

myEmbedStyle.addListener (myButton);

 

 

In order to make sure your font is visible on all machines, you can embed a font in the Library, set a Linkage name (myFont ) and set embedFonts to true.

 

Examples and practical uses

Changing the font in your components is perhaps one of the most noticeable ways of changing the look of your movie. You can either use the system fonts of your end user, by providing the name of a system font in your code, which the component will use for display, or you can embed a font into your movie, and use this for display within your components. This will increase your file size, but guarantees the end user will see what you intended your movie to look like.

Programming

The example fstyleformat.textfont.fla and fstyleformat.textfont.swf uses the two different methods of changing the font face of our components. Component instances list1 and button1 both use the system font Courier as a new font face by using the following code:

 

fontFormat = new FStyleFormat({textFont:"Courier"});

fontFormat.applyChanges();

fontFormat.addListener(list1, button1);

 

However, if you want to use an embedded font in your movie instead, you will need to add FStyleFormat.embedFonts = true; into the code. An example from our FLA is as follows:

 

embedFormat = new FStyleFormat({embedFonts:true, textFont:"arialN"});

embedFormat.applyChanges();

embedFormat.addListener(list2, button2);

 

A custom style format of embedFormat is created, and embedFonts is set to true. The style applies an embedded font to our component instances list2 and button2. The "linkage identifier" of this font is called arialN.

Tips and precautions

When working with fonts it is important to consider the two different ways of setting a custom font face to your style format. If you choose to use system fonts, remember to name your textFont value exactly the same as the font itself. This font will be recognized on an end users computer that has the same font installed.

 

If you are using unusual fonts in your components, it is a good idea to embed the font into your movie. Forgetting to set linkage in the library is one of the most common errors when using FStyleFormat.embedFonts and an embedded font. Remember to right-click or control-click on the font in the Library, and select Linkage.... Then select Export for ActionScript and Export on first frame, if they are not already selected. Note the identifier name provided, or assign a new one. Your identifier name is used instead of a system font name in the FStyleFormat.textFont property.
