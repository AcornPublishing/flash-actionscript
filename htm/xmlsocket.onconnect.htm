XMLSocket.onConnect

myXMLSocket.onConnect(a)

 

Where myXMLSocket is an instance of the XMLSocket object, and a is a Boolean indicating a successful or failed connection attempt.

 

Compatible with the Flash 5 Player and above. The XMLSocket.onConnect() event handler is invoked when a connection request is completed. See related entries: XMLSocket.connect and XMLSocket.onClose.

Description

The XMLSocket.onConnect() event handler is used with a "callback" function. The handler is invoked when a connection request is completed, and a true (success) or false (fail) result is sent back to the movie. Your "callback" function can execute custom tasks based on whether or not the connection is successful.

thesocket = new XMLSocket();

thesocket.connect("localhost", 12345);

thesocket.onConnect = function(success) {

   if (success) {

      trace("connection made");

   } else {

      trace("no connection");

   }

};

 

 

connection made

 

If the server makes a connection, the output window will trace "connection made", and if not, "no connection".

 

Examples and practical uses

The XMLSocket.onConnect() event handler is used as a "callback" function whenever a server tries to establish a socket connection and the attempt has either succeeded or failed. If the attempt was successful, then true is used as the parameter. Otherwise, false is returned and indicates a failure to connect. Unless you override the default XMLSocket.onConnect() event handler with a custom "callback" function, no actions are performed by the handler. Refer to the example xmlsocket.onconnect.fla and xmlsocket.onconnect.swf for more information on this handler.
