loadVariables

loadVariables (URL, target [,GET|POST]);

 

Where URL is the absolute or relative URL where the variables are located, target is the movie clip into which the variables are to be loaded, and GET|POST indicates the method used to send the variables

 

Compatible with Flash Player 4 and later. Flash 5 and later will automatically convert Flash 4 files to the proper format.

 

The loadVariables() action is used to load variables for a movie, either to set initial values or to update them.

Description

The loadVariables() action allows you to transfer variables and their values to a movie or movie clip already loaded with loadMovie or loadMovieNum. The variables can come from a file (either static or created by a server-side script). The loadVariables() action can be used to update existing values as a movie plays.

 

The loadVariables() action requires a URL for the location of the variable file. This file and the movie itself must be in the same sub-domain.

 

The optional GET and POST methods indicate how the variables and their values will be sent to the movie. The GET method uses a URL descriptor while the POST method uses an HTTP header. If there are a lot of variables to be sent, POST is the better method.

 

The loadVariables() action is similar in purpose to the method MovieClip.loadVariables, but the syntax is different. The LoadVars object has methods that can accomplish the same task.

Examples and practical uses

Once a movie is loaded (using a loadMovie or loadMovieNum action), variables can be passed to the movie using the loadVariables() action. The loadVariables() action reads from a file, in which the variables are in the standard MIME format:

 

varname=varvalue&...

 

where varname is the variable name, and varvalue is the value assigned. Each variable name and value pair is separated by an ampersand. For example, a data file may look like this:

 

firstname=Tim&lastname=Parker&city=Ottawa&country=Canada

 

This data is transferred to a movie using the loadVariables() action referencing the file’s name. The easiest way to use loadVariables() is to specify the file name and the movie name that the target movie resides in:

 

loadVariables("timdata.txt", "newMovie.swf");

 

This can be placed inside an event manager, such as a button click to send the values to the movie:

 

on(release){

   loadVariables("timdata.txt", "newMovie.swf");

}

 

If you want to load variables by referring to the movie level in theplayer, use the loadVariablesNum() action instead. The MovieClip.loadVariables method does a similar task to loadVariables().

Tips and precautions

You can call the loadVariables()method from a movie instance, like this:

 

_root.myMovie.loadVariables("timdata.txt");

 

All movies loaded into level 0 set the properties for the higher levels (including frame rate, size, and background color). The loadVariables() action does not remove any properties set in level 0 but may override existing values.
