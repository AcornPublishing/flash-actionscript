MovieClip._name

myMovieClip._name

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 4 and later.

 

The MovieClip._nameproperty returns the name of the movie clip instance.

Description

The MovieClip._nameproperty simply returns the name of the movie clip instance. This is often used in generic scripts to store the clip name in a variable.

 

 

Code

 

 

Notes

 

 

myName = myMovieClip._name;

 

Assigns the name of the movie clip instance to MyName.

 

 

myName = myMovieClip._name(_parent);

 

 

No parameters are specified.

Examples and practical uses

The MovieClip._nameproperty returns the name of the movie clip and allows you to store it in a variable for future use:

 

myName = myMovieClip._name;

Tips and precautions

The use of general identifiers (this, and _parent) means that you rarely need to use the _name property because it is usually implicitly known via scope; an event handler will use this for example:

 

function mover() {

    this._x += 5;

}

myClip.onEnterFrame = mover;

 

A good trick is to use the name property to signify something else as well as the name itself. For example, supposing you had a number of navigation buttons that were identical except that each one required the main timeline to jump to a different label. Instead of writing a separate script for each clip, you can simply do this:

 

function navButton() {

    gotoAndPlay(this._name.toString());

}

label1.onRelease = navButton;

stop();

 

Assuming a particular button requires you to jump to a frame “label1”, you can simply call the button label1,and convert the name to the label. This means that your script for all such buttons will be the same general code; only the name of each clip will change and this is effectively being used as a parameter!
