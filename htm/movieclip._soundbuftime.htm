MovieClip._soundbuftime

myMovieClip._soundbuftime

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip._soundbuftimeproperty specifies the number of seconds of sound to load before starting to play the movie

Description

The MovieClip._soundbuftimeproperty is used to set the number of seconds of time to preload a movie (buffer it) prior to starting playback of the movie.

 

 

Code

 

 

Notes

 

 

myTotalFrames = myMovieClip._framesloaded;

 

 

Stores the number of frames loaded so far into myTotalFrames.

 

The result is always an integer and counts frames.

 

 

myMovieClip._framesloaded();

 

 

No method of this name exists.

Examples and practical uses

The MovieClip._soundbuftimeproperty lets you set a buffer size for sound (in seconds) prior to starting to play a streamed movie. To assign a value to the property, simply use a standard assignment:

 

MovieClip._soundbuftime = 10;

 

This will buffer 10 seconds of sound prior to starting playback. To use the MovieClip._soundbuftimeproperty in other commands, assign the result of the property to a variable:

 

myBuffer = MovieClip._soundbuftime;

Tips and precautions

The MovieClip._soundbuftimeproperty is usually used to ensure a specific number of second’s worth of sound have been buffered prior to playback to prevent interruptions or pauses in the playback of a movie. This is especially useful when streaming over low-speed connections where real-time updates are unlikely.
