Mouse.addListener

Mouse.addListener (listener_name)

 

Where listener_name is the name of an Object object that you want to register as a listener.

 

Compatible with Flash Player 6 and later.

 

Mouse.addListener() registers an Object object as a listener to the Mouse object. The listener object gains a number of methods that allow it to respond to mouse events.

Description

A Mouse listener gains onMouseDown(), onMouseMove() and onMouseUp() events.

The procedure for registering a Mouse listener object is as follows:

 

·       Create an Object object that you want to use as a listener (all listeners have to be Object objects so that they can inherit the appropriate listener methods; if you try to use normal variables, the listener will not gain any methods). If we wanted to register an object myListener, we would do this:

 

myListener = new Object();

 

·       Next, you register the Object as a listener to the Mouse object:

 

Mouse.addListener(myListener);

 

All the methods are now available through the object myListener. The following code sets up a function to respond to the onMouseMove event:

 

myListener.onMouseMove = function() {

    trace("I heard you move!");

};

Examples and practical uses

Given that most user interaction occurs through the mouse, the Mouse listener is pretty fundamental to user interaction. Mouse listeners allow you to respond to mouse interaction via an efficient event driven route.

Tips and precautions

Although there are no limits to the number of Mouse objects that can respond to button or movement activity after being registered using Mouse.addListener(), registering too many objects can cause slow response from Flash. This is not a fault of Flash, but a general issue of event driven code.

 

If you want to respond to mouse events to control some feature of a movie clip, you are better off simply using the MovieClip.onMouseUp, MovieClip.onMousedown and MovieClip,onMouseMove events, and you can do this without having to set up a listener. See also the entries for these events.

 

Using The onMouseMove event can lead to much smoother animations (when responding to user interaction) than simply increasing frame rates, although when creating motion graphics, your applications are usually more suited to the movie clip mouse methods than listeners.
