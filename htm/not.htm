The not operator performs a logical NOT operation.

Description

The not operator negates the expression, reversing the logic. It is usually used with an if statement to reverse the condition’s true and false conditions.

 

 

Code

 

 

Additional explanation

 

 

Notes

 

myNum = 5;

if (myNum == 5) {

 trace(“five”);

}

 

myNum = 5;

if (not(myNum == 5)) {

 trace(“not five”);

}

 

 

The first code will display the trace() message if the condition is true (which it is). The second code will not display the trace() message because the condition is negated. This is not the same as saying myNum != 5. Instead, the condition is true, but the not negates it to false (see next example).

 

 

Flash 5 and above should use the ! operator instead of not. The if statement shown left should be written as:

 

if (!(myNum == 5)) {

 trace(“not five”);

}

 

 

myNum = not false;

myNum = not true;

 

The not negates the value, so not false is true, and not true is false.

 

 

Examples and practical uses

The not operator has been deprecated in favor of the ! operator (see the ! operator for code and examples). As with the ! operator, you can use the not operator instead of an else condition in an if statement. This code:

 

myNum = 15;

if (not (myNum == 15)){

 trace(“not fifteen”);

}

 

… is the same as writing:

 

myNum = 15;

if (myNum == 15){

}

else {

 trace(“not fifteen”);

}

Tips and precautions

Although implemented in Flash 4 and maintained for backward compatibility reasons in all subsequent versions, the not operator was deprecated in Flash 5 and later for the more common ! operator. Unless you are writing legacy Flash 4 applications, use the ! operator instead.
