Boolean.valueOf

a = myBool.valueOf()

 

Where myBool is a Boolean. a will be returned as the literal Boolean value (true or false ) of myBool.

 

This method is compatible with Flash 5 or later.

Description

This is a simple and little used method. It is the only way to retrieve the internal and current literal value of a Boolean object directly, but you will find that you never actually have to do this, assuming you only create Boolean variables (see the entry for Boolean (as a constructor and initializer) for an explanation of why this is the case).

a = New Boolean(true);

b = a.valueOf();

 

 

b = "true"

Note that the value of the Boolean object a cannot be accessed directly, but the value of b can.

 

 

a = 3;

b = a.valueOf();

 

b = "3"

 

Because the String and Number objects also have a valueOf method, Boolean.valueOf will not automatically force the conversion to assume thata is a Boolean if it turns out to be anything else. Boolean.valueOf cannot therefore convert non-Booleans to the corresponding Boolean value as part of the conversion.

 

Examples and practical uses

Because ActionScript is an untyped language, and data types are converted to the appropriate form implicitly, there is usually little of no reason to use Boolean.valueOf, because there is no reason to use Boolean objects (use Boolean variables instead, which do not have inaccessible internal values as does the Boolean object). The Boolean.valueOf method only really exists to provide core ActionScript functionality in handling Booleans when they are mixed with other types (Strings and Numbers), and any conversion is usually done automatically, so you don’t have to worry about it.

Tips and precautions

Because there is also a Number.valueOf() and String.valueOf() method, you cannot assume that using valuOf() will always return a Boolean literal. Depending on the type of the object (or variable), you may see a number or string returned.
