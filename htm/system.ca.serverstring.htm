System.capabilities.serverString

a = System.capabilities.serverString

 

a will be returned as a string value encoded for server side applications

 

This property is compatible with Flash 6 or later. This property is read only.

Description

This property will return a long string containing all available System.capabilities information regarding the user environment. A typical server string is shown below. This is for the same machine, " models author's Windows laptop ", as used in all the System.capabilities examples so far.

 

A=t&MP3=t&AE=t&VE=t&ACC=f&DEB=t&V=WIN%206%2C0%2C21%2C0&M=Macromedia Windows&R=1024x768&DP=72&COL=color&AR=1.0&I=point&OS=Windows 98/ME&L=en-UK

Examples and practical uses

A typical URL that uses the server string (it's the output from the undocumented System.product 's download() method) is shown below, and it can be seen that this is of the form: http://URL/whatever.cgi?product=whatever&serverString

 

http://www.macromedia.com/bin/flashdownload.cgi?product=mydownload&A=t&MP3=t&AE=t&VE=t&ACC=f&DEB=t&V=WIN%206%2C0%2C21%2C0&M=Macromedia Windows&R=1024x768&DP=72&COL=color&AR=1.0&I=point&OS=Windows 98/ME&L=en-UK

 

See the example system.capabilities.fla and system.capabilities.swf .

Tips and precautions

Note that:

 

·       The language string descriptor appears to include the region/dialect (i.e. "en-UK ", "en-US ", "zh-CN ", "zh-TW ", whereas the System.capabilities property has been seen to only show the language only on some operating systems (i.e. "en ", "zh ").

·       Not all the System.capabilities properties are included in the server string, and some appear to be combined with others. This is different to the descriptions in the Macromedia documentation.

 

More generally, the server string is only partially documented by the Macromedia literature, and the server strings that are listed in the literature do not all tie up with practical results. The documentation provided here is based on practical results, plus reverse engineering to reveal undocumented features.
