FStyleFormat.highlight

a.highlight = b;

 

Where a is the name of your custom style format, and b is a RGB value assigned the highlight regions of components.

 

Compatible with Flash MX. This property changes the color value for the highlight regions specified in several components. The RGB color value is assigned to the highlight property using the 0xRRGGBB format. Related entries include those for FStyleFormat.shadow, FStyleFormat.highlight3D, andFStyleFormat.darkshadow.

Description

This property sets the highlight regions of components to the specified RGB value.

 

 

Code

 

 

Shortcut

 

 

Notes

 

myHighlight = new FStyleFormat();

myHighlight.highlight = 0x99CCFF;

myHighlight.applyChanges();

myHighlight.addListener (myComponent);

 

 

myHighlight = new FStyleFormat({highlight:0x99CCFF});

myHighlight.applyChanges();

myHighlight.addListener (myComponent);

 

 

This code will change the highlight color of the myComponent instance to light blue.

 

Examples and practical uses

This property is used with FStyleFormat.shadow, FStyleFormat.highlight3D, andFStyleFormat.darkshadow. Together these propertiescreate a boarder around the edges of your components to create the appearance of depth and movement when a button is pressed.

Programming

In the example fstyleformat.highlight.fla and fstyleformat.highlight.swf , a series of components are displayed on the stage. These instances illustrate which area of each component is altered using this property. This is expanded upon with a style format using each of the properties, affecting the outlines in varying shades of blue. This creates the appearance of a raised edge, and a button being depressed when it is clicked on. The following set of colors is applied to the style format:

 

myStyle = new FStyleFormat();

myStyle.highlight3D = 0x00CCFF; //light shadow

myStyle.shadow = 0x0099CC; //outer border, light shadow

myStyle.highlight = 0x006699; //darker shadow

myStyle.darkshadow = 0x003366; //darkest

myStyle.face = 0x99CCFF;

myStyle.applyChanges();

myStyle.addListener(button3);

Tips and precautions

If you are designing a style format using these properties, you should try to choose colors that correspond in such a way that it appears as though the button is being pressed. In the previous example, you may have noticed the colors were chosen with this in mind. Refer to the example FLA for an example of such a combination.

 

Disabled components will retain this color scheme chosen with keep the same color value in the highlight3D, shadow, highlight, and darkshadow properties. However, the text color on the button will change, and the button will not be clickable.

 

You should also beware that there is a slight coloration of the outline of the round button area radio button component, and outline of the selection area of a check box component.
