Key.isDOWN

Key.isDOWN(code);

 

Where code is the key code value or a key object property of a particular key.

 

Compatible with Flash Player 5 and later. There are no known issues with any versions of Flash that support this method.

 

Key.isDOWN returns true if the specified key code is pressed (and held down).

Description

Key.isDOWN returns true if the key code value parameter matches the key code of a pressed key.

Notes

 

Key.isDOWN(key);

 

If key is pressed, returns true.

 

Matches key code values as well as key object properties assigned to specific keys.

 

Macintosh Caps Lock and Num Lock keys have identical key codes.

 

Examples and practical uses

The Key.isDOWN() method is used to determine if a particular key has been pressed. For example, to check if the Insert key has been pressed, check for key code 45:

 

if( Key.isDOWN(45)){

   // code

}

Tips and precautions

Key codes for all operating systems and platforms are summarized in the Flash documentation.

 

The key.isDown method is the only way to accurately tell which keys are pressed for all conditions. In particular, it is the only method that will:

 

·       Allow you to identify all keys that are currently being pressed when multiple keys are involved, and they are being held down. This is a usual situation for Flash video games or real time complex user interaction using a keyboard.

·       Allow direct interrogation of the current keyboard state without going through typomatic filtering or the keyboard buffer/operating system keyboard buffer.

 

This accuracy comes as a cost though; you have to test all keys you want to know about every frame you want to know about them.

The Macintosh maps the Caps Lock and Num Lock keys to identical key codes. There is no workaround for this.
