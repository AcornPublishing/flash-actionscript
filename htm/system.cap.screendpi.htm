System.capabilities.screenDPI

a = System.capabilities.screenDPI;

 

a will be returned as a number

 

This property is compatible with Flash 6 or later. This property is read only.

Description

This property is used to find out the dots per inch of the users screen. For a standard desktop, or laptop screen, it is 72dpi. For other screens, the DPI setting may vary.

Examples and practical uses

For a traditional cathode ray tube (CRT) screen, the DPI setting is a very rough (and somewhat outdated) measure of how big an image will appear on the screen. As a rule of thumb, if you scan an image at 72DPI, it will appear on a monitor at a size that is consistent with the original hardcopy.

 

On older machines, the available resolutions were not as wide as they are today, and they approximated 72 (or 75) DPI. In particular, an 800x600 size screen on a typical 15-inch monitor will show consistent image sizes if you scan at 72DPI.

 

When using Flash, the DPI is an indication of the density of pixels on the screen. A screen with a high DPI will display a smaller image for the same resolution, and this may affect the readability of your page.

 

Newer monitors can handle a wide range of resolutions because both their pitch (the density of phosphor dots across the face of the screen) and maximum scan frequencies (a measure of the screen refresh rate) have improved dramatically.

 

For example, a 21-inch monitor can display any screen from about 400x300 up to 1600x1200, and each one has a different corresponding DPI setting. A 21-inch screen displaying a 1600x1200 screen has a corresponding DPI of approximately 100. If you use the same monitor to display a 600x400 screen, the DPI scan rate for an image is less than 40.

 

In all these cases, Flash will return the "default value" of 72DPI. It can be seen that with today's technology, the default DPI setting is a very broad-brush figure! This all means that all you can assume from the default value of 72 is "this is a standard monitor".

 

For devices that do not return the default value, you can start to infer some meaning; it gives you a measure of how big pixels on the screen will appear compared to the "classic" 800x600 screen/15-inch CRT computer screen. A DPI setting larger than 72 will mean that the pixels will be denser and therefore smaller and text may have to be bigger to make up for this.

 

See the example system.capabilities.fla and system.capabilities.swf .

Tips and precautions

As seen in the discussion above, this property is only useful for non-standard devices. You cannot use it to detect different sized "standard monitors" for example. This is a shame, because as most designers know, a browser looks somewhat different on a 1600x1200 screen than it does on a 640x480 one! There is another set of properties that will allow you to do something in this situation however, and they are the properties: System.capabilities.screenResolutionX and System.capabilities.screenResolutionY. In general, it is a good idea to look at the screenDPI, screenResolutionX, and screenResolutionY properties together when forming a view on how a site will look on a particular screen.
