FStyleFormat.backgroundDisabled

a.backgroundDisabled = b;

 

Where ais the name of your style format, and b is a RGB value being assigned to the background regions of disabled component instances.

 

Compatible with Flash MX. You should be aware that this will affect the color of the background of disabled list box, combo box, check box, and radio button components. It will affect any other disabled custom or third party components supporting this property. The RGB color value is expressed using the 0xRRGGBB format. Also see the entries on FStyleFormat.foregroundDisabled andFStyleFormat.textDisabled.

Description

This propertyassigns a new RGB color value to the background property when you create a new custom style format.

This code will change the background region of myComponent when it is in a disabled state.

 

Examples and practical uses

You can use this property as part of a custom color scheme for your disabled components. Such components may be used when a certain part of your movie should be inactive. In the example fstyleformat.backgroundDisabled.fla and fstyleformat.backgroundDisabled.swf a set of active components, and a set of disabled components are on the stage. These two sets illustrate a typical color setup for an enabled component and a disabled component. Typically, a disabled component color scheme will be lighter or appear "grayed out". Our component sets in the example reflect this scheme with the following style format:

 

myStyle = new FStyleFormat();

myStyle.backgroundDisabled = 0xCC9999;

myStyle.background = 0xCC3300;

myStyle.textColor = 0xCCCCCC;

myStyle.applyChanges();

myStyle.addListener(listbox1,listbox2,comboBox,comboBox2);

Tips and precautions

This property will only alter the background of a disabled component instance. Make sure that you check the text color value of the FStyleFormat.textDisabled property is compatible the color value given to backgroundDisabled. You will also want to make sure your FStyleFormat.selectionDisabled is a compatible color as well.
