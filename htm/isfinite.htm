isFinite

isFinite(a)

 

Where a is an expression. This expression may include a variable or Boolean value that returns a numeric value.

 

Compatible with Flash 5. This function is typically used to evaluate mathematical errors, which may include division by 0. Related entries include Infinity, isNaN, Number.POSITIVE_INFINITY, and Number.NEGATIVE_INFINITY.

Description

This function evaluates the expression, and will return a Boolean value of true or false depending on whether or not it is positive infinity or negative infinity.

 

 

Code

 

 

Returned value

 

 

Notes

 

isFinite(56);

 

 

true

 

This code checks if 56 is a finite integer. The returned value is true.

 

 

isFinite(Number.MAX_VALUE * 6);

 

 

false

 

This code returns false, as it is a multiple of the max number, which returns Infinity.

 

 

isFinite(Math.infinity/Math.infinity);

 

 

false

 

NaN is returned by the expression, so a false value is returned. Refer to the entry on Infinity.

 

Examples and practical uses

This function is typically used to ascertain if a number is greater than infinity or less than infinity. isFinite is used in ActionScript to check if a number falls within the range Flash can understand. This number must be between Number.MAX_VALUE and -Number.MAX_VALUE. Therefore, isFinite is used when your code requires a number to run properly, but there is a chance an infinite number could be returned.

Programming

The function isFinite is commonly used in situations where the Math object is being used. If a possible outcome is an infinite number, this function will check whether or not it is an infinite number or one that Flash understands.

 

These numbers can be traced in the following way, as is detailed and expanded upon in the example file isfinite.fla and isfintine.swf:

 

trace(isFinite(Number.MAX_VALUE*4));

//returns false

 

trace(isFinite(Number.MAX_VALUE-Math.infinity));

//returns true

Tips and precautions

There are several methods that do not work with positive or negative Infinity or Number.POSITIVE_INFINIT. For more information on this issue, refer to the entries for Infinity, Number.POSITIVE_INFINITY, or NEGATIVE_INFINITY.
