MovieClip.createEmptyMovieClip

myMovieClip.createEmptyMovieClip (instance, depth);

 

Where myMovieClip is a movie clip instance, instance is a string identifying the new empty movie clip, and depth is the level at which to place the new clip.

 

Compatible with Flash 6 and later.

 

The MovieClip.createEmptyMovieClip() method creates a new, empty movie clip and places it at a specified level. The new clip is a child of the existing clip.

Description

The MovieClip.createEmptyMovieClip() method creates a new movie clip which has no contents, and assigns it as a child of the calling movie clip instance.

 

The following line will create an empty movie clip on the _root timeline:

 

_root.createEmptyMovieClip called bob on the timeline _root;

 

 

_root.createEmptyMovieClip("bob", 0);

 

The following lines will create both boband bill. Note that the depth has been made unique so that billdoesn’t overwrite bob:

 

 

_root.createEmptyMovieClip("bob", 0);

_root.createEmptyMovieClip("bill", 1);

 

The following code will create two movie clips bob and bill,and also create two new movie clips boband billinside bob. Notice that the depth numbers start at zero again – the depth levels are unique per timeline:

 

 

_root.createEmptyMovieClip("bob", 0);

_root.createEmptyMovieClip("bill", 1);

bob.createEmptyMovieClip("bob", 0);

bob.createEmptyMovieClip("bill", 1);

 

Finally, the following code wlll create all of the above, but also adds a new clip sueat depth 0 of _root. This will have the effect of not only overwriting bob,but also the embedded clips boband billinside it:

 

 

_root.createEmptyMovieClip("bob", 0);

_root.createEmptyMovieClip("bill", 1);

bob.createEmptyMovieClip("bob", 0);

bob.createEmptyMovieClip("bill", 1);

_root.createEmptyMovieClip("sue", 0);

 

Examples and practical uses

There are two common uses of creating empty movie clips:

 

·       When you are using the drawing API. The creation of empty movie clips is strongly associated with the drawing API. See also the entries for the drawing API for further examples.

 

·       When you want to create multiple event scripts. For example, supposing you wanted two separate onEnterFrame scripts running on _root. This is one way of doing it:

 

_root.createEmptyMovieClip("codeHolder01", 0);

_root.createEmptyMovieClip("codeHolder02", 1);

//

codeHolder01.onEnterFrame = function() {

    x++;

};

codeHolder02.onEnterFrame = function() {

    y++;

};

 

The advantage of doing this over the more obvious route (shown below):

 

function dostuff1() {

    x++;

}

function doStuff2() {

    y++;

}

_root.onEnterFrame = function() {

    doStuff1();

    doStuff2();

};

 

… is that because you can delete created clips, you can stop one of your functions without stopping the other:

 

_root.createEmptyMovieClip("codeHolder01", 0);

_root.createEmptyMovieClip("codeHolder02", 1);

//

codeHolder01.onEnterFrame = function() {

    x++;

};

codeHolder02.onEnterFrame = function() {

    y++;

};

codeHolder01.removeMovieClip();

 

The last line will stop x incrementing permanently. You can also do this with a conditionalifin a single event handler, but that would require the if to run every frame. Using empty movie clips as code carriers has the advantage that you can forget about a piece of code once you have removed the movie clip associated with it.

Tips and precautions

The method will not work if you omit any of the arguments, and is another one of those strange methods that doesn’t raise an error when this happens:

 

_root.createEmptyMovieClip("bob");

 

… will not create anything because the depth is missing, but will not raise a syntax error!

 

Also, creating two movie clips with the same instance names on the same timeline raises no error:

 

_root.createEmptyMovieClip("bob",0);

_root.createEmptyMovieClip("bob", 1);

 

This will only create one version of bob(the second one), but raise no error.
