XML.childNodes

myXML.childNodes

 

Where myXML is an instance of the XML object.

 

This method is compatible with Flash 5 Player and above. The XML.childNodes collection is an array of the nodes’ children. This method can be used in conjunction with other methods in the XML object to navigate to a particular node, and then return a part of it as usable data to the Flash movie. Related entries include XML.nodeName, XML.nodeValue, and XML.nodeType.

Description

The XML.childNodes collection provides a complete list of all the node’s children attached to an XML instance. The list of children is returned as an array. You can return a portion of the array by indicating which node you want returned.

Examples and practical uses

The XML.childNodes collection provides a list of an XML object’s children. The list is returned as an array when calling for all childNodes of a given element. Each element in the array is a reference to the XML object that represents the child node. For example, you can return all child nodes of the firstChild using the following ActionScript:

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age></cat>");

trace(myXML_xml.firstChild.childNodes);

 

Since we are asking for the childNodes of the firstChild called <cat>, we will have an array returned to the Output window in the following format:

 

<catname>Whiskers</catname>,<age>5</age>

 

This method is commonly used to navigate a structure, and return a particular name or value of a node. This will naturally depend on the nodeType of a node (type 1 is an element, and type 3 is a text node.) You will need to ask for a nodeName if it is an element node, and nodeValue if it is a text node.

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age></cat>");

trace(myXML_xml.firstChild.childNodes[0].nodeName);

 

Returns the node name:

 

catname

 

However, if you want to extract the value Whiskers of this child node, you will need to use nodeValue. This is the first child of this particular node, so you need to add firstChild, and then call for nodeValue (since it is a text node).

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age></cat>");

trace(myXML_xml.firstChild.childNodes[0].childNodes[0].nodeValue);

 

Which will output the text node:

 

Whiskers

 

You might want to use the childNodes property to change the value of a particular node as well. For example you could change the catname from Whiskers to Spooky in the following way:

 

myXML_xml = new XML("<cat><catname>Whiskers</catname><age>5</age></cat>");

myXML_xml.firstChild.childNodes[0].childNodes[0].nodeValue = "Spooky";

trace(myXML_xml.firstChild.childNodes[0].childNodes[0].nodeValue);

 

Refer to the example xml.childnodes.fla and xml.childnodes.swf for an example of this property in use.

Tips and precautions

Remember that childNode arrays are 0 based, so the first node will always be [0]. You might also want to note that firstChild and childNodes[0] reference exactly the same node.
