MovieClip.getBytesLoaded

myMovieClip.getBytesLoaded();

 

Where myMovieClip is a movie clip instance.

 

Compatible with Flash 6 and later.

 

The MovieClip.getBytesLoaded()method returns the number of bytes that have been loaded.

Description

The MovieClip.getBytesLoaded()method returns an integer indicating the number of bytes that have been loaded from a streaming clip. If you apply it to _root, it can be used to tell you the total bytes loaded in the current SWF.

Notes

 

 

myBytes = myMovieClip.getBytesLoaded();

 

 

 

Returns the number of bytes loaded in the myMovieClip clip and stores the result in myBytes.

 

The return from the method is an integer.

 

Examples and practical uses

You can use the time elapsed since loading started, the bytes loaded and the total bytes in the swf to find out when a swf will be fully loaded in, as well as all other useful parameters that describe the loading timescales:

 

_root.onEnterFrame = function() {

    fractionLoaded = _root.getBytesLoaded()/_root.getBytesTotal();

    elapsedTime = (getTimer()-startTime)/1000;

    timeToLoad = elapsedTime/fractionLoaded;

    timetoFinish = Math.ceil(timeToLoad-elapsedTime);

    percentLoaded = Math.floor(fractionLoaded*100);

};

startTime = getTimer();

 

If you place this on frame1 of _root of the loading SWF, this script will give you the following values in real time during the loading process:

 

 

PercentLoaded

 

 

The percentage of the SWF that has loaded in so far.

 

ElapsedTime

 

 

The time in seconds since frame 1 was loaded.

 

TimeToLoad

 

 

The current best guess to how long the SWF will take to load in total.

 

TimeToFinish

 

 

The current best guess to how much longer the SWF will take to finish loading from now.

 

 

As any parent will tell you, the worstthing is not letting your kids out of sight, but not knowing what they are up to.

 

The same applies for waiting for a swf to load; the user doesn’t want to know about what percentage of the swf has loaded (as most pre-loaders tend to tell us), but how much longer the user has to wait from now until something cool starts happening, wherenow is always this instant in time.

 

As well as allowing you to display the estimated time for download to complete on the users connection, thisscript will:

 

·       Modify the estimated time constantly to allow for changes in bandwidth.

·       Start off with optimistic values and get progressively more pessimistic (this is because most connections start off slightly slow and get better as they settle.

·       Allow you to detect a slow connection rather than ask the user if they want to see a high bandwidth site or a low bandwidth one.

As with all preloading schemes, you are going to get nowhere fast if your splash screens and preloader animations are as large in file size as the final site itself (mentioning no names). Using a basic script like the one shown plus some minimalist graphics are really all you need.
