Button._visible

instanceName._visible = a;

b = instanceName._visible;

 

Where instanceName is the instance name of a button, and a is the value you want to assign to the buttons viability (true for visible, and false for not visible). b is a variable that is equated to the current value of the button’svisibility property value (true or false ).

 

This property is compatible with Flash 6 or later. When using previous versions of Flash, buttons are not objects and do not therefore have properties.

Description

The Button._visibilityproperty is somewhat misnamed since it does more than just decide whether a Button can be seen or not. Setting Button._visibility = false is different to setting the _alpha property to 0, because the _visibility property doesn’t just draw the button at "zero visibility". Rather, it doesn’t draw the button at all! See the tips and precautions section to see the implications of this.

Notes

 

my_btn._visible = false;

myOther_btn._visible = true;

 

 

The button instance my_btn will no longer be drawn on stage. It will not be able to respond to most mouse based events, and you can assume that it is effectively disabled as well as invisible. You can still make changes to it (such as changing its position via its _x and _y properties), and these will be remembered and reflected when you make the button visible again.

 

The button instance myOther_btn has been made visible, and will operate as a normal button.

 

Examples and practical uses

A normal beginner site consists of navigation buttons that each take you to a different keyframe. For example, the home button would consist of a button that has a script that does a gotoAndPlay("home"), and so on for all four buttons on the menu strip.

 

 

In most cases, you may jump to a new keyframe with a different set of buttons. Rather than jump to new keyframes, you can now stay at the same keyframe, but selectively show/hide the buttons with _visibility.

 

For example, hitting the products button may appear to take you to a new keyframe that contains some new submenu buttons as shown below:

 

 

What you could do, however, is have the submenu options there on the initial page, but hide them with _visibility = false until they need to be unhidden, and stay on the same keyframe. This allows you wait around on the main keyframe for longer, and fine tune where exactly it is the user wants to go before you make the jump to the next keyframe:

 

whatWeDo_btn._visible = false;

investorDetails_btn._visible = false;

history_btn._visible = false;

...

...

about.onRelease = function() {

   // unhide the submenu...

   whatWeDo_btn._visible = true;

   investorDetails_btn._visible = true;

   history_btn._visible = true;

};

Tips and precautions

Setting Button._visibility to false is a more processor friendly way to temporarily hide a button without actually deleting it (which is not always desirable, because when you bring it back, you have to reassign all your scripts back onto it). Other, less efficient ways of hiding a button that are sometimes used include:

 

·       Moving the button to an off-stage location via the Button._x, Button._y position properties.

·       Making the button transparent by setting Button._alpha to zero. This is not the same as simply not drawing it. Setting Button _visibility to zero disables the ability of the button to respond to the mouse. This is because the button hit state doesn’t get processed in addition to the fact that the visible appearance of the button doesn’t get put onto the stage. If you set the alpha property to zero, the hit state does get processed, so the button will still work; alpha doesn’t hide everything, rather only hides the things you can see!

·       Setting the size to zero pixels with Button._height = Button._width = 0.

·       Setting the scale to zero percent with Button._xscale = Button._yscale = 0.

 

If you want to leave a button visible, but instead want to disable it, you should use Button.enabled.
