FStyleFormat.foregroundDisabled

a.foregroundDisabled = b;

 

Where ais the name of your custom style format, and b is the RGB value assigned to the foreground area of a disabled component instance.

 

Compatible with Flash MX. You should be aware that this will affect the color of the arrows used in disabled component instance scrollbars. It will also modify any other installed components supporting this property. The RGB color value is written using the 0xRRGGBB format. Related entries include FStyleFormat.backgroundDisabled and FStyleFormat.selectionDisabled.

Description

This property sets the foreground regions of the disabled component instance(s) to a specified RGB value.

This code changes the background color of myBG to a light blue shade when it is in a disabled state.

 

Examples and practical uses

This property works together with values given to the backgroundDisabled, selectionDisabled, and textDisabled properties to create a unified look when a component is in a disabled state.

Programming

In our example fstyleformat.foregroundDisabled.fla and fstyleformat.foregroundDisabled.swf , there is an example of the specific region affected by this property. Foreground disabled affects component scrollbar arrows. It does not affect push button, radio button, or check box instances. In the FLA, there are two sets of components: one shows the global settings given to a regular and disabled component, and then a component with this property set to red.

Tips and precautions

When designing your custom style format, it is recommended that you consider usability. This is especially important when assigning values to disabled components. Disabled instances are typically a dulled, lighter, or grayed out version of the other instances using the style format. You should also consider your scroll bars track, face, and push button face color values when including these properties in a style format. The color values of this property remain the same when the component is in a disabled state.
