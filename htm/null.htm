null

 

This keyword is compatible with Flash 5 and later.

 

The null keyword indicates values that are missing or undefined.

Description

The null keyword is a special value usually assigned to variables when a value is missing or there is no defined type. Numerically, it evaluates to zero. This is not the same as NaN (Not a Number).

Notes

 

if (myVar == null) {...}

 

 

The condition is true if myVar does not have a value assigned or is an invalid data type (both result in a null ).

 

null values should be tested for if there is a critical step in a movie depending on a variable’s value.

 

 

if (myVar != null) {...}

 

 

The condition is false if myVar does not have a value assigned or is an invalid data type.

 

 

Examples and practical uses

The null keyword indicates no value has been set or there is no valid data type for a value. This is often the case through the failure of a method or a reference to a variable that does not have a value set. For example, the code:

 

if (myVar == “A”){

 myFlag = 1;

}

 

… will compare the value of myVar to the value of “A”. If myVar has a value assigned to it (either directly using an assignment operation or as the result of a function or method), the comparison will be performed correctly. However, if myVar does not have a value assigned to it, the comparison is meaningless because you are trying to compare no value to something. In most cases this will cause the condition to be false, and the code inside the curly braces would never be executed. To prevent this problem, you can test for the value null using a conditional:

 

if (myVar != null ){

 myFlag = 1;

}

else {

 // code to handle null value

}

 

You can perform a test for a null value and a specific value at the same time:

 

if (myVar != null && myVar == “A”){

 myFlag = 1;

}

else {

 // code to handle other values

}

Tips and precautions

The difference between null and zero is often misunderstood. Zero is a number, although it represents nothing. A null value is not a number – it represents no value or type of value. A Flash-oriented example of this is if we have a variable called CurrentPage which points to the page a user is currently seeing. If they are not actually looking at a page, setting a value of zero would be misleading because they are not really at page zero, they’re not at any page. So, setting CurrentPage to null would indicate that there is no value for the variable.

 

The null keyword is almost always used with a condition to test a value. Testing to make sure a valid value has been assigned can prevent unwanted results from occurring, and many developers write null condition tests for all variables used to control critical parts of a script or movie.

 

A null value is often returned by methods that would normally return a value, but can’t for some reason. For example, an XML method such as XML.parentNode() will return a null if there is no parent. You should test for null values with returns from these methods. See also the keyword NaN.
