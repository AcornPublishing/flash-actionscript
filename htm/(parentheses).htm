() Parentheses

(expression)

action(parameters)

 

Where expression is any expression that can be evaluated by ActionScript, action is any ActionScript action or method that requires parameters, and parameters is the required parameter list.

 

Compatible with Flash 4 and later. Parentheses are used to define the evaluation order of an algorithm, or to define the set of parameters associated with a function, action, or method.

 

Description

Parentheses are the first thing to be addressed in a mathematical equation; a programming algorithm is the same.

 

Parentheses are also required in the syntax of many actions or methods. Refer to the specific entries for more information.

Notes

 

a = 10 +4/2

b = (10+4)/2;

 

 

a = 12

b= 7

 

The order of mathematical execution is:

Brackets Open

Evaluate Divisions

Evaluate Multiplications

Evaluate Additions

Evaluate Subtractions

 

For a, the 4/2 is evaluated first, followed by 10 + 2.

 

For b, the brackets are opened first, giving 10+4 = 14. This is followed by 14/2, giving 7.

 

 

a = !false || true;

b = !(false ||true);

 

 

 

a = true

b = false

 

The order of logic execution is:

Brackets Open

Evaluate Logic

 

The evaluation for b makes the ! act on the result false || true, which is evaluated first. In a, the ! acts on false only.

 

Examples and practical uses

Parentheses are usually used (where they are not required by the syntax anyway) to define an order of execution for mathematical evaluation. Refer to particular entries for examples of this.

Tips and precautions

As seen by the examples above, the order of execution and therefore final result can be changed drastically by grouping particular elements in the evaluation in brackets. You should use parenthesis liberally when there is any doubt that the default order of evaluation is different to what you expect. Adding parentheses also tends to make algorithms more readable, and tends to make for more readable code generally.
