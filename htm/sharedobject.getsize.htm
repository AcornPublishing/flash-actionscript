SharedObject.getSize

var sizeOfSO = mySO.getSize();

 

Where the variable sizeOfSO is defined and its value is the size, in bytes, of the shared object referenced as mySO.

 

This method is compatible with Flash 6 and above. There are no known issues with any version of Flash that supports this object.

 

This method is used to evaluate the size of a shared object. This method can be used before the flush method to determine if an increase in the allocated disk space for the current domain is needed. If so, you could warn the user with an explanation of what may happen and why, before you execute the flush method. See also SharedObject.flush.

Description

This method is used to retrieve the size of a locally persistent shared object, in bytes.

Examples and practical uses

As the user can limit the amount of space that a particular domain is allowed to use (or even not allow it altogether), the getSize method is useful for making sure you always have enough allocated disk space. Using this method in conjunction with the flush method, you can be assured that your data will be saved – user permitting of course. The example file, sharedobjectgetsize.fla (sharedobjectgetsize.swf), uses a function that is used to save the data from an object of the type Object object (which is passed to the function as a parameter, obj ) to a shared object. The saveData function retrieves the size of a shared object and passes that value plus the total value of the extra bytes (the byte size of the obj ) to the flush method – increasing the amount of allocated storage space needed as you go.

Tips and precautions

The getSize method retrieves the size of the shared object by stepping through each attribute of the data property and calculating the bytes, much like the example above. If your shared object has many attributes of the data property the getSize method can take significant time and cause quite a drain on your CPU, especially on the slower machines out there.

Use it wisely

In all honesty the getSize method isn’t very useful at all. Taking into account the processing power needed to execute, use it wisely and only when specifically needed. To alleviate your code from using this method, pass a number value representing the required bytes, when using the flush method as an alternative to using the getSize method. If you estimate properly the amount of bytes you’ll need to store your data, you’ll hopefully never have to use it.
