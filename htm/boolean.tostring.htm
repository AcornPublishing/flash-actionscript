Boolean.toString

a = myBool.toString()

 

Where myBool is a Boolean. a will be returned as a string containing the Boolean value ("true " or "false ") of myBool.

 

This method is compatible with Flash 5 or later.

Description

This is a simple and little used method. It is the only way to retrieve the internal value of a Boolean object directly as a string, but you will find that you never actually have to do this, assuming you only create Boolean variables (see the entry for Boolean (as a constructor and initializer) for an explanation of why this is the case).

 

 

Code

 

 

Notes

 

a = new Boolean(true);

b = a.toString();

 

 

b = "true"

Note that the value of the Boolean object a cannot be accessed directly, but the value of b can.

 

 

a = 3;

b = a.toString();

 

b = "3"

 

Because the String and Number objects also have a toString method, Boolean.toString will not automatically force the conversion to assume thata is a Boolean if it turns out to be anything else. Boolean.toString cannot therefore convert non-Booleans to the corresponding Boolean value as part of the conversion.

 

Examples and practical uses

Because ActionScript is an untyped language, and data types are converted to the appropriate form implicitly, there is usually little of no reason to use Boolean.toString. The Boolean.toString method only really exists to provide core ActionScript functionality in handling Booleans when they are mixed with other types (Strings and Numbers), and any conversion is usually done automatically, so you don’t have to worry about it. The only toString method with useful conversion possibilities other than those that are performed automatically anyway is Number.toString, which converts to a given number base. See Number.toString().

 

Additionally, you don’t have to convert a Boolean to a string before you can display it in a textfield.

 

The only reason you might have to use it is if you were using Flash as an application front end, and the server side portion required values as strings, but this is not likely.

Tips and precautions

As mentioned in the examples, you cannot force Boolean.toString to convert a value to a string whilst assuming it is a Boolean value (because there is also a Number.toString and a String.toString ).
