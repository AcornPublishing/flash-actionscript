TextFormat

Compatible with Flash Player 6 and later.

 

The TextFormat object handles character appearance of one or more TextField objects. You apply a TextFormat object to a TextField using the TextField.setTextFormat method.

Description

The TextFormat object has many properties attached to it that to control the appearances of text. A constructor must be used to create a new TextFormat instance.

 

The TextFormat object contains a single method:

 

·       TextFormat.getTextExtent – gets the width and height of a text field.

 

The TextFormat object has several properties associated with it:

 

·       TextFormat.align – sets the alignment of a paragraph.

·       TextFormat.blockIndent – controls block indenting.

·       TextFormat.bold – sets whether a text field is boldface.

·       TextFormat.bullet – sets whether the text field has bulleted lists.

·       TextFormat.color – sets the color of the text.

·       TextFormat.font – sets the font used in text.

·       TextFormat.indent – sets the indentation value.

·       TextFormat.italic – sets whether the text is italicized.

·       TextFormat.leading – sets the leading vertical space between lines.

·       TextFormat.leftMargin – the left margin in points.

·       TextFormat.rightMargin – the right margin in points.

·       TextFormat.tabStops – sets the tab stops.

·       TextFormat.target – returns the window in a browser where a hyperlink is displayed.

·       TextFormat.size – sets the point size of text.

·       TextFormat.underline – sets whether text is underlined.

·       TextFormat.url – sets the URL to which the text links.

Examples and practical uses

The TextFormat object contains many properties that have to be specified in a constructor. Any property that is not specified is set to null and not used. The properties (and their data types) are listed in the order:

 

1.       font – string

2.       size – number

3.       color – hexadecimal value

4.       bold – Boolean

5.       italic – Boolean

6.       underline – Boolean

7.       url – string

8.       target – string

9.       align – string

10.   leftMargin – number

11.   rightMargin – number

12.   indent – integer

13.   leading – number

 

as parameters to the constructor:

 

myTextFormat = new TextFormat(font, size, color, bold, italic, underline, url, target, align, leftMargin, rightMargin, indent, leading);

 

You should not miss any fields out if you use this form (set them to null instead), although it is possible to miss out values from the end of the list because they would become null anyway (although this is not recommended because it may confuse).

 

The following constructor:

 

myTextFormat = new TextFormat("Times New Roman", 20, 0x000000, null, false, true, null, null, null, null, null, null, null);

 

will define a format of 20 point Times New Roman font in black, with the text underlined. The bold setting (and any other property set to null) of any text field this text format is applied to will be unchanged.

 

To apply this text format to a text field, you use the TextField.setTextFormat method. Assuming we wanted to apply it to a text field called my_txt ,you would use the following:

 

my_txt.setTextFormat(myTextFormat);

 

You can see this example : textformat.fla and textformat.swf;

There are two issues that may stop this working:

 

·       The font you specify does not exist on a particular users machine. You can check for this with the TextField.getFontList method.

·       You are trying to apply the font change to "Times New Roman" to a text field that you have also specified embedded font outlines for. Flash cannot retrieve font outlines from a users machine. See the Tips and precautions section for workarounds.

 

The full constructor definition above can be a little cumbersome if you only want to define one or two text format properties. If this is the case, you can define the properties individually. The following code will define a text format object that only sets bold to true:

 

myTextFormat = new TextFormat();

myTextFormat.bold = true;

Tips and Precautions

You should note that Flash has two ways of rendering text, and these affect the way the way that the text is treated:

 

·       If you use the Character… button on the property inspector to embed font outlines (or use the TextField.embedFonts property), then Flash will treat the text as a vector graphic, and cannot apply many style changes associated with fonts (such as bold) unless you also embed all the fonts you will be changing between.

·       If you select No Characters , Flash will not embed font outlines, and will treat the text field as a "system" font. You can then apply all font style changes, and can also use fonts available on the users machine that are correctly registered with their operating system.

 

 

An understanding of the way font outlines change the way Flash handles text will prevent most of the common problems seen with the TextFormat object, particularly the "why does the TextFormat object not work with some of my text, but does work with other text?" sort of question so often seen on the newsgroups regarding this object.

 

You do not usually need to use a text format object if you only have a few fields to change, or when you will not change the formatting per text field after content creation. In such cases, consider simply using the property inspector to manually change your text fields individually.

 

Text fields come into their own when you are either:

 

·       Dynamically changing from one text format to another.

·       Need to apply the same formatting to many text fields.

 

See the example textformat.fla and textformat.swf . The first retrieves a font form the users machine and uses it in a text field, whereas the second embeds the font in the SWF itself and uses that dynamically. Both apply the text formatting noted in the example seen in the Examples and practical uses section.
