FStyleFormat

a = new FStyleFormat([b,c,n])

 

Where a is the name of a new formStyleFormat, and b, c, n areoptional properties assigned to the custom style format.

 

Compatible with Flash MX. This object is used to change the color and style of the text in Flash UI components, custom components, and third party components. You cannot use this object to change the graphics of components themselves. You wouldn't use this object to change a single instance of a component. Instead of using FStyleFormat,it is best to use the setStyleProperty method. Refer to related entries on the FStyleProperty.addListener, FStyleProperty.applyChanges, and FComponent.setStyleProperty methods.

Description

This object is used to create a custom style format for Flash UI components.

var myStyle = new FStyleFormat();

myStyle.textSize = 8;

myStyle.textColor= 0xFF0000;

myStyle.applyChanges();

myStyle.addListener (myListBox);

 

 

myStyle = new FStyleFormat ({textSize:8, textColor: 0xFF0000});

myStyle.applyChanges();

myStyle.addListener(myListBox);

 

 

Using var in the FStyleFormat object is optional. There are two ways to use the constructor, as seen in the shortcut.

 

myNewStyle = new FStyleFormat();

myNewStyle.textColor = #99CCFF;

myNewStyle.applyChanges();

 

 

myNewStyle = new FStyleFormat({textColor:#99CCFF});

myNewStyle.applyChanges();

 

 

Colors must be written using the 0xRRGGBB format. You must also add a listener using the myNewStyle.addListener(); method.

 

Examples and practical uses

Flash MX assigns a global style format to all UI Components, which is an instance of the FStyleFormat object. You can use the FStyleFormat object to create your own style formats to use within any components supporting the properties. These properties can be used with third party components, or your own custom components. The FStyleFormat object allows you to change a single instance of a component away from properties set in the globalStyleFormat.

Programming

In the example fstyleformat.fla and fstyleformat.swf we create a new instance of the FStyleFormat object using its constructor. Several properties are formatted within a few style formats, and then assigned to components using the FStyleFormat.addListener method. Two components are on the stage. Each has its own style format, and one style format dealing with text is shared between the two.

 

The shared style format is as follows:

 

textStyle = new FStyleFormat({textSize:10, textItalic:true, textColor:0x333366});

textStyle.applyChanges();

textStyle.addListener(myListBox, myButton);

 

The example file will elaborate on how to remove one of the style formats from the stage.

Tips and precautions

When you are changing style properties within your movie, you must make sure that you use the FStyleFormat.applyChanges method. This will update the style format applied to the component instances listening to the format. In order to apply a custom style format to a component, you must use the FStyleFormat.addListener method. Third party components and custom components can listen to the properties of the FStyleFormat object.

 

Sometimes it is best to keep.as file libraries of custom style formats, and use #include for adding to a Flash movie. You can keep several style formats in one library. Also, you can apply more than one format within the same move, and even onto the same component. Using setStyleProperty is the best way to modify single FStyleFormat properties. This method will override what style format the component instance is listening to.

 

When custom style formats are applied to instances, they take on the new properties in the format. All other properties will listen to the globalStyleFormat, or the previous custom style format.
