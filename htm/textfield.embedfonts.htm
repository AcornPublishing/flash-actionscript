TextField.embedFonts

myTextField .embedFonts

 

Where myTextField is a text field instance.

 

Compatible with Flash Player 6 and later .

 

The TextField.embedFonts indicates whether embedded fonts are allowed.

Description

The TextField.embedFonts property is a Boolean that indicates whether embedded font outlines are to be rendered. If true , embedded fonts are allowed: if false embedded fonts are not allowed.

Examples and practical uses

The TextField.embedFonts property is a Boolean that indicates whether the text field can be rendered with embedded font outlines. If true, embedded font outlines are rendered. If false , only device fonts are allowed. The command:

 

myTextField .embedFonts = true;

 

means the text field can render embedded outline fonts.

Tips and precautions

The TextField.embedFonts property allows you to use more flexible and varied embedded font outlines, at the cost of more processing time. If turned off (set to false ) only device fonts are allowed in the text field.

 

This method is related to the Character Options window discussed in the Tips and precautions section of the entry for TextField . See also the discussion there regarding embedded fonts and text fields created with MovieClip.createTextField .

 

Because there is little need to change the value once set (the fonts are embedded during SWF creation, so changing the value at run time will not save you bandwidth, and will not allow you to change the default), you are unlikely to need to change the embedFonts property for text fields. In fact, changing this setting during run time can cause problems such as:

 

·       If you change embedFont to true for a text field, and the font outlines have not been embedded in the SWF (i.e. by another text field) you will see no text whatsoever.

·       If you change embedFont to False , Flash will stop treating the text field as a graphic, and property based animation is no longer possible. This side effect is not immediately obvious or expected. See also the discussion in the Tips and precautions section of the entry for TextField .

 

For these reasons, you are advised not to change the embedFont property, unless you want to stop Flash anti-aliasing text (something you may want to do for small font sizes to maintain legibility). If you pixel snap text, anti-aliasing will not occur in any case, and you might want to try that instead

 

For more information about pixel snapping and its effect on anti aliasing, see the Button entries.
